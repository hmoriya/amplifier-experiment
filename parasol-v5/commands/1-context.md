# /parasol:1-context - プロジェクトコンテキスト確立

## コマンド: `/parasol:1-context`

プロジェクトの基礎となるコンテキストを確立します。組織分析、市場評価、制約条件を明確化します。

## 実行時間
約1-2時間

## 前提条件
- プロジェクトの基本情報
- 組織の概要情報
- 市場/業界の基本知識

## 実行内容

### Task 1: 組織分析

#### 組織の現状評価
```yaml
組織プロファイル:
  名称: [組織名]
  業界: [業界分類]
  規模: [従業員数、売上規模]
  事業領域: [主要事業の説明]

組織構造:
  階層: [組織階層の数]
  部門: [主要部門のリスト]
  意思決定: [意思決定プロセス]

技術成熟度:
  現在のIT活用レベル: [1-5の5段階評価]
  デジタル化の進捗: [%表示]
  主要システム: [使用中の主要システム]

組織文化:
  変革への姿勢: [保守的/中立/積極的]
  リスク許容度: [低/中/高]
  学習志向: [1-5の5段階評価]
```

### Task 2: 市場評価

#### 市場環境分析
```yaml
市場規模:
  総市場規模: [金額]
  成長率: [年率%]
  主要トレンド: [3-5個のトレンド]

競合状況:
  主要競合: [上位3-5社]
  市場シェア: [自社のポジション]
  差別化要因: [独自の強み]

顧客セグメント:
  主要顧客層: [セグメント定義]
  顧客ニーズ: [主要なニーズ3-5個]
  顧客期待: [期待値の変化]

技術動向:
  業界標準技術: [主流技術]
  新興技術: [注目すべき新技術]
  技術的脅威: [ディスラプション要因]
```

### Task 3: 制約条件の特定

#### プロジェクト制約
```yaml
リソース制約:
  予算: [利用可能予算]
  人員: [投入可能人員]
  期間: [プロジェクト期限]

技術制約:
  既存システム: [統合が必要なシステム]
  技術スタック: [使用可能な技術]
  セキュリティ: [セキュリティ要件]

組織制約:
  承認プロセス: [必要な承認レベル]
  コンプライアンス: [準拠すべき規制]
  変更管理: [変更の許容範囲]

外部制約:
  規制要件: [業界規制]
  市場要因: [市場からの制約]
  パートナー: [外部依存関係]
```

### Task 4: ステークホルダー分析

#### ステークホルダーマップ
```yaml
内部ステークホルダー:
  経営層:
    期待: [経営層の期待]
    影響力: [高/中/低]
    関与度: [高/中/低]

  事業部門:
    期待: [事業部門の期待]
    影響力: [高/中/低]
    関与度: [高/中/低]

  IT部門:
    期待: [IT部門の期待]
    影響力: [高/中/低]
    関与度: [高/中/低]

外部ステークホルダー:
  顧客:
    期待: [顧客の期待]
    影響力: [高/中/低]

  パートナー:
    期待: [パートナーの期待]
    影響力: [高/中/低]
```

## 成果物

以下のファイルが`outputs/1-context/`に生成されます：

1. **organization-analysis.md**
   - 組織の現状分析
   - 強みと弱みの評価
   - 改善機会の特定

2. **market-assessment.md**
   - 市場環境の評価
   - 競合分析
   - 市場機会の特定

3. **constraints.md**
   - 制約条件の一覧
   - リスク評価
   - 対応策の提案

4. **stakeholder-map.md**
   - ステークホルダー分析
   - 影響力/関心度マトリクス
   - エンゲージメント戦略

## 価値トレーサビリティ（Phase 1）

> **参照**: `_value-traceability-system/` - 完全な価値追跡可能性保証システム

### 事実ベース分析確認

**組織分析の価値的根拠**：
```yaml
価値発見プロセス:
  現在価値の定量測定:
    - 顧客満足度: [具体的な数値・調査結果]
    - 売上成長率: [過去3年の実績データ]
    - 市場シェア: [公的データソース]
    - 従業員満足度: [社内調査結果]
  
  価値創造能力:
    - 技術力: [特許数、技術評価レポート]
    - 人材力: [スキル評価、専門資格]
    - 財務力: [ROI、キャッシュフロー]
    - ブランド力: [ブランド価値調査]
```

**市場分析の事実確認**：
```yaml
市場機会の客観的根拠:
  市場規模データ: [信頼できる市場調査会社のレポート]
  成長トレンド: [公的統計、業界団体データ]
  顧客ニーズ: [直接調査、インタビュー結果]
  競合状況: [公開財務データ、市場シェア]
```

### 想像表現検出・排除

**自動チェック対象**：
- ❌ "組織は〜だろう" → ✅ "組織データによると〜"
- ❌ "市場では〜と思われる" → ✅ "市場調査結果では〜"
- ❌ "ステークホルダーは〜を期待しているはず" → ✅ "ステークホルダーインタビューで〜と確認"

### 構造的必然性チェック

**分析の論理的根拠**：
```yaml
組織現状評価:
  強み特定根拠: [具体的な競合比較データ]
  弱み特定根拠: [客観的な評価指標]
  機会特定根拠: [市場データと組織能力の論理的接続]
  脅威特定根拠: [環境分析の具体的証拠]

ステークホルダー分析:
  影響力評価: [意思決定プロセス分析結果]
  期待値特定: [直接確認またはプロキシデータ]
  エンゲージメント戦略: [過去の成功事例分析]
```

### MS1品質ゲート準備

**Phase 1完了基準**：
- [ ] 想像表現完全排除（検出率0%）
- [ ] 全分析データの出典明記
- [ ] 価値発見の定量的根拠確立
- [ ] ステークホルダー情報の事実確認完了

**価値トレーサビリティ確保**：
```yaml
価値発見トレーサビリティ:
  現在価値 → 改善機会:
    - 定量的ギャップ分析
    - 具体的な価値向上ポテンシャル
    - 実現可能性の客観的評価

  制約条件 → 価値実現への影響:
    - 各制約が価値創造に与える具体的影響
    - 制約解除による価値向上可能性
    - 制約内での最大価値実現戦略
```
