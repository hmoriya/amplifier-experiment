## 1. エグゼクティブサマリー

### 1.1 V5.7の核心原則

**価値継承構造 = VStream構成**

V5.7では、**価値定義の継承構造と同じ構造で、VStreamも構成される**という原則を確立します。

**重要**: VStreamの本数は固定ではなく、**組織の価値継承構造から導出**されます。

```
【V5.7 核心コンセプト】

価値は継承される → VStreamも継承される

例：カルピス（HD → OpCo → Brand の3階層構造）の場合

  HD価値定義        →   HD VStream（全消費者向け）
      ↓ 継承
  OpCo価値定義      →   OpCo VStream（飲料消費者向け）
      ↓ 継承
  Brand価値定義     →   Brand VStream（乳酸菌飲料価値層向け）★詳細定義

→ 結果として3本のVStream（継承構造から導出）
```

### 1.2 継承構造によるVStream構成の例

| 継承構造 | VStream構成 |
|---------|-------------|
| HD のみ | HD VStream（1本） |
| HD → OpCo | HD + OpCo VStream（2本） |
| HD → OpCo → Brand | HD + OpCo + Brand VStream（3本） |
| HD → OpCo → Brand → Product | 4本のVStream |

### 1.2 V5.6からの進化

| 観点 | V5.6 | V5.7 |
|------|------|------|
| 分析起点 | 対象から直接開始 | **全体組織構造から開始** |
| 価値の根拠 | 暗黙的 | **継承チェーンで明示** |
| VStreamの定義 | プロセス中心 | **価値定義単位と1:1** |
| 複数VStreamの扱い | 顧客セグメント別 | **コンテキスト階層別** |

## 2. コンテキスト階層と価値定義

### 2.1 コンテキスト階層の定義

```
┌─────────────────────────────────────────────────────────────┐
│ Level 0: Holding Company (HD)                               │
│                                                             │
│   定義: 複数事業会社を統括する持株会社                        │
│   価値: グループ全体の理念・ビジョン                         │
│   顧客: グループが対象とする全消費者                         │
│   VStream: グループ全体として顧客に価値を届ける流れ          │
└──────────────────────────────┬──────────────────────────────┘
                               │ 継承
┌──────────────────────────────▼──────────────────────────────┐
│ Level 1: Operating Company (OpCo)                           │
│                                                             │
│   定義: 特定事業ドメインを担当する事業会社                   │
│   価値: 事業ドメインに特殊化されたHD価値                     │
│   顧客: 事業ドメインの消費者（HD顧客の部分集合）             │
│   VStream: 事業会社として顧客に価値を届ける流れ              │
└──────────────────────────────┬──────────────────────────────┘
                               │ 継承
┌──────────────────────────────▼──────────────────────────────┐
│ Level 2: Brand                                              │
│                                                             │
│   定義: 特定顧客セグメント向けのブランド                     │
│   価値: 顧客セグメントに特化されたOpCo価値                   │
│   顧客: ブランドのターゲット層（OpCo顧客の部分集合）          │
│   VStream: ブランドとして顧客に価値を届ける流れ              │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 顧客セグメントの包含関係

**必須検証**: 下位コンテキストの顧客は、上位コンテキストの顧客の部分集合でなければなりません。

```
HD顧客 ⊇ OpCo顧客 ⊇ Brand顧客

例（アサヒグループ）:
「全消費者」⊇「清涼飲料消費者」⊇「乳酸菌飲料価値層」
```

## 3. 価値継承の原則

### 3.1 継承タイプ

| タイプ | 定義 | 典型的な適用 |
|--------|------|-------------|
| **特殊化** | 上位価値を特定ドメインに適用 | HD → OpCo |
| **精緻化** | 上位価値をより具体的に表現 | HD → OpCo, OpCo → Brand |
| **セグメント化** | 上位価値を特定顧客層に特化 | OpCo → Brand |
| **増幅** | 上位価値の一部を強調 | Brand固有 |
| **補完** | 上位にない独自価値を追加 | 全階層 |

### 3.2 継承チェーンの記録形式

```yaml
# 継承チェーン定義
value_inheritance:
  source:
    context_level: "HD"
    context_name: "アサヒグループHD"
    value_id: "HV-01"
    value_name: "期待を超えるおいしさ"

  target:
    context_level: "OpCo"
    context_name: "アサヒ飲料"
    value_id: "OV-01"
    value_name: "飲料でのおいしさ追求"

  inheritance:
    type: "特殊化"
    rationale: "HDの「おいしさ」を清涼飲料という事業ドメインに特化"
    evidence:
      - "のどごし・爽快感・香りといった飲料特有の要素で具体化"
```

## 4. VStream設計原則

### 4.1 核心原則: 価値継承構造 = VStream構成

**定義**: ターゲットコンテキストを選択すると、そのレベルまでの**価値継承構造と同じ構造**でVStreamが構成されます。

**原則**:
- ✅ 「価値は継承される → VStreamも継承される」
- ✅ 「ターゲット階層までの全VStreamを定義」
- ✅ 「ターゲット階層を詳細に、上位は概要」

**誤解の排除**:
- ❌ 「1コンテキスト = 1 VStream」→ 継承構造を無視している
- ❌ 「顧客セグメントごとにVStreamを分ける」→ それはVS内の分岐
- ❌ 「上位VStreamは不要」→ 継承関係の説明ができなくなる

### 4.2 VStreamの構成

```
VStream（1本）
│
├── VS0→VS1: 認知段階
│   └── OS群: 認知を実現するOperational Stage
│
├── VS1→VS2: 選好段階
│   └── OS群: 選好を実現するOperational Stage
│
├── VS2→VS3: 初回体験段階
│   └── OS群: 初回体験を実現するOperational Stage
│
├── VS3→VS4: 継続段階
│   └── OS群: 継続を実現するOperational Stage
│
├── VS4→VS5: 定着段階
│   └── OS群: 定着を実現するOperational Stage
│
├── VS5→VS6: 発展段階
│   └── OS群: 発展を実現するOperational Stage
│
└── VS6→VS7: 推奨段階
    └── OS群: 推奨を実現するOperational Stage
```

### 4.3 VStreamと価値の対応

VStreamは、**そのコンテキストで定義されたすべての価値**を実現します。

```
例: カルピスコンテキストのVStream

カルピスVStream
│
├── VS段階全体を通じて実現する価値:
│   ├── VL1-1: 心身の健やかさを支える
│   └── VL1-2: 世代を超えた幸福な記憶を紡ぐ
│
├── VS0→VS2（認知〜選好）で特に重要な価値:
│   └── VL2-2-2: 家族のコミュニケーション触媒（世代間伝播）
│
├── VS2→VS4（初回〜継続）で特に重要な価値:
│   └── VL2-1-1: 乳酸菌による健康価値（おいしさ体験）
│
├── VS4→VS5（定着）で特に重要な価値:
│   └── VL2-1-2: 安心・信頼の品質（健康価値認識）
│
└── VS5→VS7（発展〜推奨）で特に重要な価値:
    └── VL2-2-1: 懐かしさと新しさの融合（ロイヤルティ）
```

│   ├── 三ツ矢サイダー（Brand）
│   │   価値定義: 「伝統・爽快感・ピュア」

### 5.3 カルピスターゲットのVStream構成

**ターゲット: Brand（カルピス）の場合**

カルピスの価値継承構造は「HD → OpCo → Brand」の3階層なので、**結果として3本のVStreamで構成**されます。

```
┌─────────────────────────────────────────────────────────────────────┐
│ VStream 1: HD VStream（アサヒグループHD）                            │
│ 顧客: 全消費者                                                      │
│ 価値コンポーネント: HV-01, HV-02, HV-03                             │
│ 詳細度: 概要レベル（上位なので簡略）                                │
│                                                                     │
│   VS0 → VS1 → VS2 → VS3 → VS4 → VS5 → VS6 → VS7                    │
│   （グループとして全消費者に価値を届ける流れ）                       │
└──────────────────────────────┬──────────────────────────────────────┘
                               │ 継承（特殊化）
┌──────────────────────────────▼──────────────────────────────────────┐
│ VStream 2: OpCo VStream（アサヒ飲料）                                │
│ 顧客: 清涼飲料消費者                                                │
│ 価値コンポーネント: OV-01, OV-02, OV-03                             │
│ 詳細度: 中間レベル                                                  │
│                                                                     │
│   VS0 → VS1 → VS2 → VS3 → VS4 → VS5 → VS6 → VS7                    │
│   （飲料会社として清涼飲料消費者に価値を届ける流れ）                 │
└──────────────────────────────┬──────────────────────────────────────┘
                               │ 継承（セグメント化）
┌──────────────────────────────▼──────────────────────────────────────┐
│ VStream 3: Brand VStream（カルピス）★ターゲット詳細                 │
│ 顧客: 乳酸菌飲料価値層                                              │
│ 価値コンポーネント: BV-01, BV-02, BV-03 → VL1-1, VL1-2              │
│ 詳細度: 詳細レベル（ターゲットなので最も詳細）                       │
│                                                                     │
│   VS0         VS1         VS2         VS3                           │
│  [未認知]────[認知]──────[興味]──────[初回購入]                      │
│     │          │           │            │                           │
│     │ 世代間  │ 健康     │ おいしさ  │                            │
│     │ 伝播   │ 訴求     │ 期待     │                              │
│                                                                     │
│   VS3         VS4         VS5         VS6         VS7               │
│  [初回購入]──[体験]──────[継続]──────[定着]──────[推奨]             │
│     │          │           │            │           │               │
│     │ おいし │ 健康価値 │ 習慣化   │ 世代間   │                  │
│     │ さ体験 │ 認識     │          │ 共有     │                    │
│                                                                     │
│  価値実現: VL1-1（健やかさ）、VL1-2（世代記憶）を全段階で実現       │
└─────────────────────────────────────────────────────────────────────┘
```

**上位VStreamの価値コンポーネント活用**:
- Brand VStreamは、上位（HD/OpCo）の価値コンポーネントを継承して利用
- 継承元の価値がなぜBrand価値になったかが構造的に説明可能
- 上位VStreamを参照することでグループ戦略との整合性を確認

| Value Component | 完全互換 | 変更なし |
| Operational Stage | 完全互換 | 変更なし |

### 8.2 拡張部分

| 概念 | V5.6 | V5.7 |
|------|------|------|
| コンテキスト階層 | 暗黙的 | **明示的定義** |
| 価値継承 | 暗黙的 | **継承チェーン明記** |
| VStream構成 | 顧客セグメント別複数 | **継承構造と一致（階層数分）** |
| 分析起点 | 対象から直接 | **全体構造から** |
| 価値コンポーネント | 階層別に独立 | **上位から継承して利用** |

## 9. 用語集

| 用語 | 英語 | 定義 |

| **ターゲットコンテキスト** | Target Context | 分析対象となる組織階層のエンティティ |

| **セグメント化** | Segmentation | 上位価値を特定顧客層に特化する継承タイプ |
| **顧客包含関係** | Customer Subset Relation | 下位顧客⊆上位顧客の関係 |

## 付録B: 従来（V5.6以前）との比較

### B.1 アプローチの違い

**V5.6以前**:
```
対象（カルピス）の分析開始
    ↓
VL1-VL3定義
    ↓
VStream設計（顧客セグメント別に複数本）
```

**V5.7**:
```
全体組織構造の把握
    ↓
コンテキスト選択（カルピス = Brand）
    ↓
価値継承分析（HD→OpCo→Brand）
    ↓
VL1-VL3定義（継承を基に）
    ↓
VStream設計（継承構造と同じ構成 = 3本）
  ├── HD VStream（概要）
  ├── OpCo VStream（中間）
  └── Brand VStream（詳細）★ターゲット
```

### B.2 得られる効果

| 効果 | V5.6以前 | V5.7 |
|------|---------|------|
| 価値の根拠 | 「分析の結果」 | 「企業構造から導出」 |
| 戦略整合性 | 個別確認が必要 | 構造的に担保 |
| オンボーディング | 各価値を個別説明 | 継承チェーンで説明 |
| VStream管理 | 顧客セグメント別で複雑 | 継承構造に従い明確（階層数分） |
| 価値コンポーネント | 階層間の関係不明確 | 継承により関係が構造化 |
