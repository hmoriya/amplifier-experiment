# Phase 2: Value Definition - 価値定義

企業の価値創造を定義し、**業種・セグメント特性に応じた**Value Streams (VS0-VS7) を確立します。

## 使用方法

```bash
/parasol:2-value              # インタラクティブ選択（業種→セグメント→VS）
/parasol:2-value industry     # 業種・セグメント設定
/parasol:2-value VS0          # VS0を直接指定
/parasol:2-value VS2          # VS2を直接指定
/parasol:2-value all          # 全VS概要を生成
```

## 目的

- **業種・セグメントを特定する** - 適切なVSパターンの選択
- **Value Proposition（価値提案）を明確化する** - 誰に、何の価値を、どのように提供するか
- **バリューストリーム (VS0-VS7) を定義する** - 業種特性に応じた価値創造の全体像
- **Phase 3 Capabilitiesへの入力を準備する** - 必要なCapabilitiesの特定

## ⚠️ 重要: 業種によるVSパターンの違い

業種によってステークホルダー構造が異なり、VSの設計パターンが変わります。

### パターン分類

| パターン | 説明 | 業種例 |
|----------|------|--------|
| **標準型** | 単一顧客視点 | 製造業、SaaS、小売、メディア |
| **両面市場型** | 2つの顧客セグメント | 人材派遣、不動産仲介、マッチングサービス |
| **多面ステークホルダー型** | 3つ以上のセグメント | 地方銀行（個人/法人/行員）、病院（患者/医師/保険） |

### 設計アプローチ

| パターン | Layer 1（分離型） | Layer 2（統合型） |
|----------|-------------------|-------------------|
| 標準型 | 不要 | 標準VSテンプレート |
| 両面市場型 | オプション | **統合型VS**（2面価値定義） |
| 多面ステークホルダー型 | **必須**（セグメント別VS） | **統合型VS**（N面価値定義） |

**参照**: `parasol/patterns/value/industry-value-stream-patterns.md`

## Value Proposition 発見質問

VS個別定義時に使用する発見質問：

### Business Context 質問
1. このVSが必要な事業上の理由は何ですか？
2. 現在の課題や問題点は何ですか？
3. 市場や競合の状況はどうですか？

### Stakeholder Value 質問
1. 顧客にとっての価値は何ですか？（具体的なメリット）
2. 従業員にとっての価値は何ですか？
3. 株主・投資家にとっての価値は何ですか？
4. 社会にとっての価値は何ですか？

### Business Metrics 質問
1. 成功を測る主要指標（KPI）は何ですか？
2. 現在のベースライン値はありますか？
3. 目標値と達成期限はありますか？

### Strategic Importance 質問
1. このVSは競争優位性にどう貢献しますか？
2. コア/サポート/ジェネリックのどれに分類されますか？
3. 投資優先度はどの程度ですか？

### Capabilities 質問
1. このVSを実現するために必要な能力は何ですか？
2. 現在その能力はどの程度成熟していますか？
3. Build/Buy/Partnerのどのアプローチが適切ですか？

## Value Streams (VS0-VS7) について

**VS0: Vision & Strategy** - 組織のビジョン・戦略の策定と整合
**VS1: Market Analysis** - 市場動向・競合・機会の分析
**VS2: Customer Experience** - 顧客体験の設計・提供・改善
**VS3: Operations Excellence** - オペレーションの効率化・最適化
**VS4: Innovation & Growth** - イノベーション推進と成長戦略
**VS5: Risk & Compliance** - リスク管理・法令遵守・ガバナンス
**VS6: Partnership & Ecosystem** - パートナーシップ・エコシステム構築
**VS7: Sustainability & Impact** - サステナビリティ・社会的インパクト

これらは業種を問わず適用可能な汎用的なバリューストリームです。
プロジェクトの特性に応じて、関連するVSを選択してください。
