# VS-CL階層図パターン集

**バージョン**: V5.4
**最終更新**: 2025-01-06

---

## 概要

このドキュメントは、Value Stage（VS）とCapability Level（CL）の関係を視覚化するための図パターンを収録しています。

### ⚠️ 重要：V5のデフォルトパターン

**Parasol V5では「バリューストリーム航海図」がデフォルトパターンです。**

- **VS毎にスイムレーン（縦列）で完全分離**が基本
- 3層スタック図などの代替パターンは使用しない
- 全てのケイパビリティはVS（WHEN軸）に紐付けて配置

---

## 図の構成要素：4軸構造

バリューストリーム航海図は **WHEN / WHAT / HOW + WHY** の4軸で構成されます。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  WHEN (いつ)     VS0 ──▶ VS1 ──▶ VS2 ──▶ VS3 ──▶ VS4 ──▶ VS5 ──▶ VS6 ──▶ VS7 │
│  顧客の航路      顧客が価値を体験するまでの状態変化を時系列で示す              │    WHY (なぜ)
│  (横軸)                                                                     │    航海の目的地
│                                                                             │    ┌─────────┐
├─────────────────────────────────────────────────────────────────────────────┤    │         │
│                                                                             │    │ 価値定義 │
│  WHAT (何を)     CL1: コア活動領域（顧客基点）                                │    │         │
│  各ステージの    CL2: サポート活動領域（バックオフィス）                       │───▶│すべての  │
│  活動            ★ コア領域: 顧客との接点、注意を集中すべき活動               │    │活動が   │
│  (縦軸の                                                                    │    │目指す   │
│   スイムレーン)  顧客が次のステージへ進むために、                             │    │最終ゴール│
│                  組織が実行すべき活動や獲得すべき能力を定義                   │    │         │
│                                                                             │    └─────────┘
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  HOW (どうやって) 活動の実現方法                                             │
│                   「WHAT」で定義された能力を、具体的なソフトウェアや           │
│                   システムとして実装する方法                                 │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### WHEN軸: Value Stage（顧客の航路）

顧客ライフサイクルの段階を表すシェブロンで表現（横軸）：

| Stage | 名称 | 説明 |
|-------|------|------|
| VS0 | 潜在ニーズ | 潜在的な課題・ニーズの認識 |
| VS1 | 認知到達 | 製品・サービスを知る |
| VS2 | 選好形成 | 興味を持ち比較検討 |
| VS3 | 意思決定 | 購入・契約を決定 |
| VS4 | 価値獲得 | 製品・サービスを受け取る |
| VS5 | 価値実感 | 価値を体験・実感 |
| VS6 | 価値継続 | 継続利用・リピート |
| VS7 | 価値共創 | 推奨・コミュニティ参加 |

### WHAT軸: Capability Level（各ステージの活動）

VS毎のスイムレーン内で活動領域とケイパビリティを表現（縦軸）：

| Level | 名称 | 説明 |
|-------|------|------|
| CL1 | コア活動領域 | 顧客基点の活動領域（価値を生む活動の単位） |
| CL2 | サポート活動領域 | バックオフィス/ケイパビリティ（活動を実現する能力） |
| CL3 | Business Operation | 業務操作（システムが提供する具体的操作） |

**★ コア領域**: 顧客との接点であり、注意を集中すべき活動

**包含関係**: CL1 ⊃ CL2 ⊃ CL3

### HOW軸: 活動の実現方法

「WHAT」で定義された能力を、具体的なソフトウェアやシステムとして実装する方法を示します。

### WHY: 航海の目的地（価値定義）

この航海（ビジネス活動）で顧客に届けるべき最終的な「価値」を定義します。すべての活動が目指す最終ゴールです。

## デフォルトパターン：バリューストリーム航海図

**これがV5の標準パターンです。** 全てのVS-CL図はこの形式を基本とします。

### 特徴

- **3層構造**: WHEN（横軸）+ WHAT（縦軸スイムレーン）+ HOW（実装層）
- **VS毎の完全分離**: 縦列（スイムレーン）でVSごとに完全分離
- **右端の目的地**: WHY（価値定義）が全ての活動のゴール
- **色分け**: Core領域（★）は強調色で表示

### この図が示すもの

| 軸 | 要素 | 意味 | 役割 |
|----|------|------|------|
| **WHEN** | 横軸（VS0→VS7） | 顧客の状態変化 | 顧客がどのような旅を経て価値を手にするか |
| **WHAT** | 縦軸スイムレーン | 各ステージの活動 | 顧客が次のステージへ進むために組織が実行すべき活動 |
| **WHAT** | CL1 | コア活動領域 | 顧客基点の活動領域（価値を生む活動の単位） |
| **WHAT** | CL2 | サポート活動領域 | バックオフィス/ケイパビリティ（活動を実現する能力） |
| **HOW** | 実装層 | 活動の実現方法 | 具体的なソフトウェアやシステムとして実装 |
| **WHY** | 右端の価値定義 | 航海の目的地 | この航海で到達すべき唯一のゴール |

### プロンプト例

```
バリューストリーム航海図を作成してください。

【形式】
- VS0〜VS7を横軸にダイヤモンド型シェブロンで配置
- 右端に価値定義（目的地）のボックス
- VS毎に縦列（スイムレーン）で完全分離
- 各列内にCL1/CL2をネスト表示

【内容】
- 価値定義: [顧客にとっての価値を記載]
- VS0: [ステージ名] → CL1: [活動領域] → CL2: [ケイパビリティ1]/[ケイパビリティ2]/...
- VS1: [ステージ名] → CL1: [活動領域] → CL2: [ケイパビリティ1]/[ケイパビリティ2]/...
- （以下同様）

【オプション】
- Core領域（★）の指定
- CL3（業務操作）の追加
```


## カスタマイズ用プロンプトテンプレート

### 基本テンプレート

```
バリューストリーム航海図を作成してください。

【形式】
- VSの範囲: VS{開始}〜VS{終了}
- CLの深さ: CL1のみ / CL1+CL2 / CL1+CL2+CL3
- スイムレーン: VS毎に完全分離（デフォルト）
- 凡例: あり/なし

【内容】
- 価値定義: {顧客価値の内容}
- VS{N}: {ステージ名}
  - CL1: {活動領域名}
    - CL2: {ケイパビリティ名}
      - CL3: {操作1}/{操作2}/{操作3}（必要な場合）

【追加要素】（オプション）
- Core/Supporting/Generic分類
- タイムライン表示: あり/なし
```

## 使用上の注意

### デフォルトパターンの原則

1. **VS毎のスイムレーン完全分離が必須**
   - ケイパビリティは必ずVS（WHEN軸）に紐付ける
   - VSを跨いで配置することは避ける

2. **価値定義が目的地**
   - 全ての活動は価値定義に向かう
   - 右端に明示的に配置

3. **CL階層の明確化**
   - CL1 ⊃ CL2 ⊃ CL3の包含関係を視覚化
   - 各列内でネスト表現

### やってはいけないこと

- ❌ VS毎の分離を無視した配置
- ❌ 3層スタック図などの代替パターンの使用
- ❌ 価値定義なしの図の作成
- ❌ CL階層を無視した平坦な配置
