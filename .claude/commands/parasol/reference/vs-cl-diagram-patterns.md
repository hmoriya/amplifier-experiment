# VS-CL階層図パターン集

**バージョン**: V5.4
**最終更新**: 2025-01-06

---

## 概要

このドキュメントは、Value Stage（VS）とCapability Level（CL）の関係を視覚化するための図パターンを収録しています。

### ⚠️ 重要：V5のデフォルトパターン

**Parasol V5では「バリューストリーム航海図」がデフォルトパターンです。**

- **VS毎にスイムレーン（縦列）で完全分離**が基本
- 3層スタック図などの代替パターンは使用しない
- 全てのケイパビリティはVS（WHEN軸）に紐付けて配置

---

## 図の構成要素

### 横軸: Value Stage（WHEN軸）

顧客ライフサイクルの段階を表すシェブロン（ダイヤモンド型）で表現：

| Stage | 名称 | 説明 |
|-------|------|------|
| VS0 | 潜在ニーズ | 潜在的な課題・ニーズの認識 |
| VS1 | 認知到達 | 製品・サービスを知る |
| VS2 | 選好形成 | 興味を持ち比較検討 |
| VS3 | 意思決定 | 購入・契約を決定 |
| VS4 | 価値獲得 | 製品・サービスを受け取る |
| VS5 | 価値実感 | 価値を体験・実感 |
| VS6 | 価値継続 | 継続利用・リピート |
| VS7 | 価値共創 | 推奨・コミュニティ参加 |

### 縦軸: Capability Level（包含関係）

スイムレーン内でネストしたボックスで包含関係を表現：

| Level | 名称 | 説明 |
|-------|------|------|
| CL1 | Activity Area | 活動領域（価値を生む活動の単位） |
| CL2 | Capability | ケイパビリティ（活動を実現する能力） |
| CL3 | Business Operation | 業務操作（システムが提供する具体的操作） |

**包含関係**: CL1 ⊃ CL2 ⊃ CL3

## デフォルトパターン：バリューストリーム航海図

**これがV5の標準パターンです。** 全てのVS-CL図はこの形式を基本とします。

### 特徴

- **VS毎の完全分離**: 縦列（スイムレーン）でVSごとに完全分離
- **右端の目的地**: 価値定義が全ての活動のゴール
- **CL階層の視覚化**: 各列内でCL1→CL2→CL3がネスト

### この図が示すもの

| 要素 | 意味 | 役割 |
|------|------|------|
| **横軸（VS0→VS7）** | 顧客の状態変化 | 顧客がどのような旅を経て価値を手にするか |
| **右端の価値定義** | 目的地 | この航海で到達すべき唯一の目的地 |
| **スイムレーン（縦列）** | VS毎の完全分離 | 各ステージの活動を明確に分離 |
| **CL1** | 活動領域 | 各ステージで「誰が何をするか」 |
| **CL2** | ケイパビリティ | 各領域で「具体的に何ができるか」 |

### 業界別カスタマイズ例

**EC/小売業向け:**
```
VS0:潜在ニーズ → VS1:認知 → VS2:検討 → VS3:購入 → VS4:配送 → VS5:利用 → VS6:推奨 → VS7:継続
価値定義: 「お客様の生活を豊かにする商品体験」
```

**B2B/SaaS向け:**
```
VS0:課題認識 → VS1:情報収集 → VS2:評価 → VS3:契約 → VS4:導入 → VS5:活用 → VS6:拡大 → VS7:更新

```

**金融/保険向け:**
```
VS0:潜在リスク → VS1:認知 → VS2:相談 → VS3:申込 → VS4:審査 → VS5:契約 → VS6:保全 → VS7:更新
価値定義: 「将来の安心を今ここで」
```

## 使用上の注意

### デフォルトパターンの原則

1. **VS毎のスイムレーン完全分離が必須**
   - ケイパビリティは必ずVS（WHEN軸）に紐付ける
   - VSを跨いで配置することは避ける

2. **価値定義が目的地**
   - 全ての活動は価値定義に向かう
   - 右端に明示的に配置

3. **CL階層の明確化**
   - CL1 ⊃ CL2 ⊃ CL3の包含関係を視覚化
   - 各列内でネスト表現

### やってはいけないこと

- ❌ VS毎の分離を無視した配置
- ❌ 3層スタック図などの代替パターンの使用
- ❌ 価値定義なしの図の作成
- ❌ CL階層を無視した平坦な配置
