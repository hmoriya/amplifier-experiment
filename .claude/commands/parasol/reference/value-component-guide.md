## 1. 概要

### 1.1 Value Componentとは

**Value Component（VC）** は、顧客に提供する価値を構成する要素です。顧客視点で「何を受け取るか」を記述する**外向きWHAT**として機能します。

```
WHATの二面性
│
├── 外向きWHAT: Value Component (VC)
│   「顧客に何を提供するか」
│   → 顧客視点・価値提案
│
└── 内向きWHAT: Capability
    「自社に何ができるか」
    → 組織視点・能力定義
```

### 1.2 なぜVCが必要か

| 課題 | VCによる解決 |
|------|-------------|
| 価値分析が組織視点に偏る | 顧客視点を体系的に組み込む |
| Capabilityと顧客価値の対応が不明確 | VC-Cap対応を明示化 |
| 投資判断の根拠が曖昧 | VC別の重要度で投資優先度を決定 |
| ステージ別の価値設計が不足 | VS-VCマトリクスで対応 |

## 2. 5つのValue Component

### 2.1 一覧

| ID | 要素 | 英語 | 定義 | 顧客の問い |
|----|------|------|------|-----------|
| **VC1** | 製品価値 | Product Value | 製品そのものが提供する価値 | 「何を買うのか」 |
| **VC2** | サービス価値 | Service Value | 利便性・アクセス性の価値 | 「どう手に入るか」 |
| **VC3** | 体験価値 | Experience Value | 購入・使用時の体験価値 | 「どう感じるか」 |
| **VC4** | 関係価値 | Relationship Value | ブランドとの関係性価値 | 「なぜ選び続けるか」 |
| **VC5** | 情報価値 | Information Value | 提供される情報の価値 | 「何を知れるか」 |

### 2.2 VC1: 製品価値（Product Value）

#### 定義
製品そのものが提供する価値。機能、品質、性能など製品の本質的な価値。

#### サブコンポーネント

| ID | 名称 | 説明 | 例 |
|----|------|------|-----|
| VC1-1 | 機能的価値 | 性能、機能、スペック | 処理速度、容量、精度 |
| VC1-2 | 品質価値 | 安全、信頼性、耐久性 | 品質保証、長寿命 |
| VC1-3 | 多様性価値 | 選択肢、カスタマイズ | バリエーション、オプション |

#### 顧客の問い
- 「この製品は何ができるのか？」
- 「品質は信頼できるか？」
- 「自分に合った選択肢はあるか？」

#### 対応Capability例
- 製品開発
- 品質管理
- R&D
- 製造技術

### 2.3 VC2: サービス価値（Service Value）

#### 定義
製品を入手・利用する際の利便性やアクセス性に関する価値。

#### サブコンポーネント

| ID | 名称 | 説明 | 例 |
|----|------|------|-----|
| VC2-1 | アクセス価値 | 入手しやすさ、チャネル | 店舗網、EC、配送 |
| VC2-2 | 価格価値 | 価格妥当性、コスパ | 価格設定、割引 |
| VC2-3 | 安定価値 | 安定供給、可用性 | 在庫、供給体制 |

#### 顧客の問い
- 「どこで買えるのか？」
- 「価格は妥当か？」
- 「いつでも手に入るか？」

#### 対応Capability例
- チャネル管理
- 在庫管理
- 物流
- 価格戦略

### 2.4 VC3: 体験価値（Experience Value）

#### 定義
購入・使用時に得られる体験や感情に関する価値。

#### サブコンポーネント

| ID | 名称 | 説明 | 例 |
|----|------|------|-----|
| VC3-1 | 購入体験 | 購買プロセス、接客 | 店舗体験、UI/UX |
| VC3-2 | 使用体験 | 使い心地、感動 | 操作性、満足感 |
| VC3-3 | 所有体験 | デザイン、所有感 | パッケージ、ブランド感 |

#### 顧客の問い
- 「買うときどう感じるか？」
- 「使うときどう感じるか？」
- 「持っていてどう感じるか？」

#### 対応Capability例
- 店頭体験設計
- UX/UIデザイン
- パッケージデザイン
- カスタマーサービス

### 2.5 VC4: 関係価値（Relationship Value）

#### 定義
ブランドや企業との継続的な関係性から得られる価値。

#### サブコンポーネント

| ID | 名称 | 説明 | 例 |
|----|------|------|-----|
| VC4-1 | 信頼価値 | ブランド信頼、安心感 | 企業イメージ、歴史 |
| VC4-2 | 帰属価値 | コミュニティ、所属感 | ファンクラブ、会員制度 |
| VC4-3 | 継続価値 | ロイヤリティ、習慣 | ポイント、特典 |

#### 顧客の問い
- 「このブランドを信頼できるか？」
- 「仲間とのつながりを感じるか？」
- 「続けることで得られるものは？」

#### 対応Capability例
- ブランド管理
- CRM
- ロイヤリティプログラム
- コミュニティ運営

### 2.6 VC5: 情報価値（Information Value）

#### 定義
製品やブランドに関して提供される情報の価値。

#### サブコンポーネント

| ID | 名称 | 説明 | 例 |
|----|------|------|-----|
| VC5-1 | 知識価値 | 製品知識、使い方 | マニュアル、How-to |
| VC5-2 | 発見価値 | 新商品、トレンド | ニュース、レコメンド |
| VC5-3 | 共有価値 | 口コミ、推奨 | レビュー、SNS |

#### 顧客の問い
- 「使い方がわかるか？」
- 「新しい発見はあるか？」
- 「他の人はどう思っているか？」

#### 対応Capability例
- コンテンツ制作
- マーケティング
- SNS運用
- カスタマーサポート

## 3. VCの活用方法

### 3.1 VL3へのVC構成分析

各VL3（詳細価値）に対して、どのVCで構成されるかを分析します。

#### 分析手順

```
Step 1: VL3の価値を顧客視点で言語化
        「この価値は顧客にとって何を意味するか？」

Step 2: 5つのVCそれぞれの貢献度を評価
        ★：低い / ★★：中程度 / ★★★：高い

Step 3: 主要VC（★★★）を特定
        「この価値の核心は何か？」

Step 4: VC構成パターンを識別
        「製品中心型」「体験中心型」など
```

#### 分析例

```yaml
VL3: "発酵技術による味の革新"

VC構成分析:
  VC1_製品価値: ★★★
    理由: 発酵技術は製品の味を直接決定する
    サブ: VC1-1（機能的価値）が核心

  VC2_サービス価値: ★
    理由: 流通は標準的で差別化要因ではない

  VC3_体験価値: ★★
    理由: 飲む瞬間の感動は重要だが二次的

  VC4_関係価値: ★
    理由: 技術訴求でブランド向上に寄与するが間接的

  VC5_情報価値: ★
    理由: 技術情報の発信は補助的

パターン: 製品中心型（VC1主導）
```

### 3.2 VS-VCマトリクス

各Value Stage（顧客状態）で重要なVCを特定します。

#### 標準マトリクス

```
           │  VC1    │  VC2    │  VC3    │  VC4    │  VC5   │
───────────┼─────────┼─────────┼─────────┼─────────┼────────┤
VS0→VS1   │    ★    │    ★    │   ★★   │    ★    │  ★★★  │
(認知)     │         │         │         │         │        │
───────────┼─────────┼─────────┼─────────┼─────────┼────────┤
VS1→VS2   │  ★★★   │   ★★   │  ★★★   │   ★★   │  ★★★  │
(選好形成) │         │         │         │         │        │
───────────┼─────────┼─────────┼─────────┼─────────┼────────┤
VS2→VS3   │  ★★★   │  ★★★   │  ★★★   │    ★    │   ★★  │
(初回体験) │         │         │         │         │        │
───────────┼─────────┼─────────┼─────────┼─────────┼────────┤
VS3→VS4   │  ★★★   │  ★★★   │   ★★   │   ★★   │    ★   │
(継続利用) │         │         │         │         │        │
───────────┼─────────┼─────────┼─────────┼─────────┼────────┤
VS4→VS5   │   ★★   │   ★★   │   ★★   │  ★★★   │    ★   │
(定着)     │         │         │         │         │        │
───────────┼─────────┼─────────┼─────────┼─────────┼────────┤
VS6→VS7   │    ★    │    ★    │  ★★★   │  ★★★   │  ★★★  │
(推奨)     │         │         │         │         │        │
───────────┴─────────┴─────────┴─────────┴─────────┴────────┘
```

#### 活用方法

| 活用シーン | 方法 |
|----------|------|
| **投資判断** | 現在のVS分布に応じたVC別投資配分 |
| **ギャップ分析** | 目標VSに必要なVC能力との乖離 |
| **マーケティング戦略** | VS遷移を促すVC強化ポイント |
| **Capability優先度** | VC-Cap対応から投資Capを決定 |

### 3.3 VC-Capability対応分析

#### 対応マトリクス

```
┌─────────────────────────────────────────────────────────────┐
│                  VC-Capability対応                          │
├──────────┬──────────────────────────────────────────────────┤
│   VC     │   対応Capability（例）                           │
├──────────┼──────────────────────────────────────────────────┤
│ VC1製品  │ 製品開発、品質管理、R&D、製造技術                 │
│ VC2サービス│ チャネル管理、在庫管理、物流、価格戦略           │
│ VC3体験  │ 店頭体験設計、UX/UI、パッケージ、CS               │
│ VC4関係  │ ブランド管理、CRM、ロイヤリティ、コミュニティ     │
│ VC5情報  │ コンテンツ、マーケティング、SNS、サポート         │
└──────────┴──────────────────────────────────────────────────┘
```

#### ギャップ分析

```yaml
VC-Capギャップ分析:
  VL3: "愛着のあるブランドとの関係"

  VC要求:
    VC4_関係価値: ★★★（最重要）
    VC3_体験価値: ★★★

  Cap充足度:
    VC4対応Cap:
      - ブランド管理: ★★★（充足）
      - CRM: ★★（やや不足）
      - コミュニティ運営: ★（大幅不足）

    VC3対応Cap:
      - 店頭体験設計: ★★（やや不足）
      - UX/UI: ★（大幅不足）

  ギャップ:
    - コミュニティ運営能力の強化が必要
    - UX/UI能力の獲得が必要

  アクション:
    - コミュニティ専任チームの設置
    - UX人材の採用/育成
```

## 4. VC構成パターン

### 4.1 代表的パターン

| パターン | 主導VC | 特徴 | 業種例 |
|---------|--------|------|--------|
| **製品主導型** | VC1 | 製品の機能・品質で差別化 | 製造業、テック |
| **サービス主導型** | VC2 | 利便性・アクセス性で差別化 | 小売、物流 |
| **体験主導型** | VC3 | 顧客体験で差別化 | ホスピタリティ、エンタメ |
| **関係主導型** | VC4 | ブランド・関係性で差別化 | 高級品、サブスク |
| **情報主導型** | VC5 | 情報・知識で差別化 | メディア、教育 |
| **バランス型** | 複数 | 複数VCで総合的に差別化 | 総合サービス |

### 4.2 パターン別戦略

#### 製品主導型（VC1中心）

```
強化ポイント:
  - R&D投資
  - 品質管理体制
  - 製造技術

補完ポイント:
  - VC5（製品情報発信）で認知促進
  - VC3（使用体験）で継続利用促進

注意点:
  - 機能だけでなく情緒的価値も意識
  - VS4以降は関係価値への移行が必要
```

#### 体験主導型（VC3中心）

```
強化ポイント:
  - 店頭/オンライン体験設計
  - UX/UIデザイン
  - カスタマーサービス

補完ポイント:
  - VC1（製品品質）は最低限必要
  - VC4（関係構築）で定着促進

注意点:
  - 体験の一貫性が重要
  - 全タッチポイントでの品質管理
```

## 7. FAQ

### Q1: VCとVLの違いは？

**A**: VL（Value Level）は価値の階層的分解（WHY軸）、VC（Value Component）は各価値の構成要素分類（WHAT軸の顧客側面）です。VL3の各価値がどのVC要素で構成されるかを分析します。

### Q2: すべてのVCを★★★にすべき？

**A**: いいえ。リソースは有限なので、自社の強みと顧客価値に合わせて重点VCを選択します。全部を★★★にしようとすると、どれも中途半端になります。

### Q3: VSごとにVCの重要度が変わる理由は？

**A**: 顧客状態によって求めるものが変わるからです。認知段階では情報価値、購入段階では製品・サービス価値、定着段階では関係価値が重要になります。

### Q4: VC-Cap対応のギャップが大きい場合は？

**A**: 以下の選択肢があります：
1. Capability強化への投資
2. パートナーシップによる補完
3. 重点VCの見直し（強みに集中）
4. 段階的な能力構築計画
