# Parasol [VERSION] スタイルガイド

このガイドは、Parasol書籍の一貫したスタイルと品質を保つための基準を定めます。

## 基本原則

### 1. 読者ファースト
- **明確性** > 簡潔性
- **実用性** > 理論的完璧性
- **段階的理解** > 包括的説明

### 2. コンテンツ比率
- **概念説明**: 70%以上
- **コード例**: 30%以下
- **ビジュアル**: 各章3つ以上

## 文体ガイドライン

### トーン
- **教育的**: 教師が生徒に説明するような親しみやすさ
- **権威的**: 専門知識に基づく自信
- **実践的**: 理論より実用を重視

### 人称
- **基本**: 「私たち」（読者を含む共同体）
- **説明**: 三人称
- **例示**: 「あなた」（読者への直接的な語りかけ）

### 時制
- **概念説明**: 現在形
- **事例紹介**: 過去形
- **手順説明**: 現在形/命令形

## 用語の統一

### 必須統一用語
| 日本語 | 英語 | 表記ルール |
|--------|------|------------|
| ケイパビリティ | Capability | カタカナ表記 |
| コンテキスト | Context | カタカナ表記 |
| ZIGZAG | ZIGZAG | 常に大文字 |
| フェーズ | Phase | Phase 0-7の形式 |
| 設計マトリクス | Design Matrix | 英語併記推奨 |

### 表記ルール
- **Phase番号**: `Phase 0`（スペースあり、アラビア数字）
- **バージョン**: `v5.4`（vは小文字）
- **章参照**: `Chapter 13`（Chapterは英語）

## コード例の扱い

### 基本ルール
1. **配置**: 概念説明の後に配置
2. **長さ**: 最大30行（それ以上は付録へ）
3. **言語**: TypeScript優先（型情報で理解促進）
4. **コメント**: 日本語で丁寧に

## ビジュアル要素

### 図表の種類と用途
1. **概念図**: PlantUMLで作成（概念関係の可視化）
2. **フロー図**: Mermaidで作成（プロセスの説明）
3. **比較表**: Markdownテーブル（選択肢の提示）
4. **スクリーンショット**: 実例の提示

### 図表配置ルール
- テキストで概念を導入
- 図表で視覚化
- 図表の後に詳細説明
