# 第6章　Phase 2：価値発見と価値設計 ― ビジネスの心臓部を見つける

## はじめに：価値とは何か？

ある経営会議での一コマです。

「我が社の価値は何ですか？」

コンサルタントの問いに、役員たちは口々に答えます。

「高品質な製品です」
「優れた技術力です」
「長年の信頼関係です」
「充実したサポート体制です」

コンサルタントは静かに首を振りました。

「それらは素晴らしい『特徴』です。でも『価値』ではありません。価値とは、顧客があなたに『お金を払う理由』です」

会議室に沈黙が流れました。

この章では、Parasol V5の心臓部である「価値」の発見と設計を学びます。技術やプロセスの前に、「なぜ顧客はお金を払うのか」「何が本当の競争力なのか」を明らかにすることが、成功への最短距離なのです。

## 価値の本質：特徴ではなく、顧客の変化

### Vision/Mission/Core Valuesとパラソル価値（VL）の違い

価値について考える前に、重要な区別を理解しましょう。多くの企業が持つ「ミッション」「ビジョン」「コアバリュー」と、Parasol V5で扱う「価値（VL）」は、根本的に異なる概念です。

**企業価値体系（Vision/Mission/Core Values）— HOWの規定**

| 概念 | 役割 | 特徴 |
|------|------|------|
| Vision | 目指す未来像 | 10年以上不変、「北極星」として機能 |
| Mission | 存在意義・使命 | 企業の根本的な存在理由 |
| Core Values | 行動指針・価値観 | 日々の判断基準、文化の土台 |

これらは**HOW（どのように振る舞うか）**を規定する「安定した錨」です。

**パラソル価値（VL1→VL2→VL3）— WHATの規定**

| レベル | 内容 | 特徴 |
|--------|------|------|
| VL1 | 最上位価値（企業が顧客に届ける根本的価値） | 3-5年で見直し |
| VL2 | 中位価値（VL1を実現する具体的価値） | 1-2年で深化 |
| VL3 | 詳細価値（測定可能な具体的成果） | 四半期で検証 |

これらは**WHAT（何を届けるか）**を規定し、顧客理解の深化により**進化（深化）**していきます。

```
┌────────────────────────────────────────────────────────────────┐
│  企業価値体系                 │  パラソル価値                  │
│  (HOW: どう振る舞うか)        │  (WHAT: 何を届けるか)          │
├────────────────────────────────────────────────────────────────┤
│  Vision: 「健康で豊かな社会」  │  VL1: 「健康価値の提供」       │
│  Mission: 「食を通じて幸せを」 │  VL2: 「低糖質商品群」         │
│  Values: 「品質へのこだわり」  │  VL3: 「糖質30%オフビール」    │
├────────────────────────────────────────────────────────────────┤
│  ★ 10年以上不変               │  ★ 顧客理解と共に深化          │
│  ★ 組織文化の土台             │  ★ 設計判断の直接的根拠        │
└────────────────────────────────────────────────────────────────┘
```

**なぜこの区別が重要か？**

Phase 2では**パラソル価値（VL）**を扱います。Vision/Missionは参照しますが、直接の設計対象ではありません。

- Vision/Mission → 「なぜこの会社が存在するか」（不変の指針）
- パラソル価値 → 「顧客に何を届けるか」（進化する設計対象）

この区別を誤ると、抽象的なビジョンをそのままシステム要件にしてしまう危険があります。

### 特徴と価値の決定的な違い

多くの企業が「特徴」を「価値」と混同しています。

| 業界 | 特徴 | 価値 | 顧客が払う理由 |
|------|------|------|----------------|
| **製造業** | 精度±0.001mmの加工技術 | 不良品ゼロで生産ラインが止まらない | ライン停止による損失（1時間1000万円）を防げる |
| **ITサービス** | 99.99%の稼働率 | ビジネスが24時間止まらない安心 | 機会損失と信用失墜のリスクがない |
| **コンサルティング** | MBA保有者10名のチーム | 3ヶ月で売上が20%向上する | 投資対効果が明確で確実 |

価値は常に「顧客の状態がどう変わるか」で定義されます。

### 価値の3層構造

価値は、表層から深層まで3つの層を持っています。

| 層 | 内容 | 例 | 測定指標 |
|----|------|-----|----------|
| **表層価値** | 見える成果 | 発注処理が5分→1分に短縮 | 時間、回数、量 |
| **ビジネス価値** | 経営への影響 | 業務効率化で人件費30%削減 | 売上、コスト、利益 |
| **戦略的価値** | 競争優位性 | 競合より3倍速い顧客対応で差別化 | 市場シェア、顧客満足度、ブランド価値 |

優れたシステムは、3層すべてで価値を生み出します。

## Phase 2の実践：価値ストリームマッピング

### ステップ1：現状の価値の流れを可視化する

まず、現在のビジネスで「価値がどう流れているか」を可視化します。

**事例：オンライン書店の価値ストリーム**

**価値ストリーム名**：書籍販売

- **トリガー**：顧客が本を探し始める
- **ゴール**：本を読み終えて満足する

| ステップ | アクション | 現状時間 | ペインポイント | 価値損失 |
|----------|------------|----------|----------------|----------|
| 1. 検索 | 欲しい本を探す | 平均15分 | 検索精度が低い / レコメンドが的外れ / 在庫状況が不明 | 50%が購入せずに離脱 |
| 2. 購入 | 購入を決定する | 平均5分 | レビューが少ない / 価格比較ができない / 送料が分かりにくい | カートの30%が放棄 |
| 3. 配送 | 商品を受け取る | 2-3日 | 配送状況が不明 / 受取時間の制約 / 梱包が過剰 | 5%が再配達 |
| 4. 読書 | 読書を楽しむ | 1-2週間 | 読後の共有場所がない / 次の本の推薦がない / 質問する相手がいない | 次回購入率30% |

この可視化により、価値が失われている「出血点」が明確になります。

### ステップ2：理想の価値ストリームを設計する

次に、「理想的にはどうあるべきか」を設計します。

**ビジョン**：本との完璧な出会いと深い読書体験

| ステップ | 新しいアプローチ | 目標時間 | 価値の追加 | 期待成果 |
|----------|------------------|----------|------------|----------|
| 1. 発見 | AI駆動のパーソナライズド検索 | 3分以内 | 読書履歴から好みを学習 / 今の気分に合った提案 / 立ち読み機能 | 離脱率を20%に削減 |
| 2. 決定 | 透明性の高い購入体験 | 1分以内 | 信頼できるレビュー集約 / 全ての費用を事前表示 / ワンクリック購入 | カート放棄を10%に |
| 3. 受取 | 柔軟な受取オプション | 即日〜選択可能 | 電子版の即時提供 / コンビニ受取 / エコ配送オプション | 再配達ゼロ |
| 4. 継続 | 読書コミュニティ | 読書中〜読後 | 読書会の開催 / 著者との対話 / 次の一冊の提案 | リピート率70% |

### ステップ3：価値指標の定義

価値は測定できなければ管理できません。

**顧客指標**

| 指標 | 現状 | 目標 | 測定方法 |
|------|------|------|----------|
| 満足度 | 3.5/5.0 | 4.5/5.0 | 購入後アンケート |
| 努力スコア | 本を見つけるまで15分 | 3分以内 | サイト分析 |
| ロイヤルティ | 年間購入回数2.5回 | 年間6回 | 顧客データベース |

**ビジネス指標**

| 指標 | 現状 | 目標 | 算出方法 |
|------|------|------|----------|
| 顧客単価 | 年間8,000円 | 年間20,000円 | 購入回数 × 平均単価 |
| コンバージョン率 | 訪問者の2% | 訪問者の5% | Google Analytics |
| 運営コスト | 売上の35% | 売上の25% | 在庫15%→10%、配送10%→8%、人件費10%→7% |

## 価値設計の極意：6軸システムの活用

### なぜ6軸が必要なのか

ビジネスの価値は多面的です。一つの軸だけでは、全体像を捉えきれません。

| 軸 | フォーカス | 例 |
|----|------------|-----|
| **ケイパビリティ軸** | このビジネスは、どんな能力を持つべきか？ | 24時間注文受付能力 |
| **ビジネスユニット軸** | 組織のどの部門が、何に責任を持つか？ | 法人営業部の効率化 |
| **価値軸** | どのような価値を、誰に、どう届けるか？ | 受注処理時間75%削減 |
| **プラットフォーム軸** | どのような共通基盤が、複数の価値創造を支えるか？ | 統合顧客データベース |
| **フュージョン軸** | 異なる要素を組み合わせることで、どんな新しい価値が生まれるか？ | 在庫×物流の最適化 |
| **多層価値ストリーム軸** | 価値はどのように階層的に流れ、積み上がるか？ | 現場→部門→全社への波及 |

### 産業別の軸選択パターン

業界特性により、重視すべき軸の組み合わせが変わります。

**製造業**
- **主要軸**：ケイパビリティ軸（生産能力の向上）、プラットフォーム軸（IoT基盤の構築）、フュージョン軸（設計×製造×品質の統合）
- **理由**：物理的な制約と統合が鍵

**小売業**
- **主要軸**：価値軸（顧客体験の向上）、ビジネスユニット軸（店舗別最適化）、多層価値ストリーム軸（店舗→地域→本部）
- **理由**：顧客接点と階層構造が重要

**金融業**
- **主要軸**：プラットフォーム軸（リスク管理基盤）、ケイパビリティ軸（即時処理能力）、価値軸（コンプライアンス価値）
- **理由**：信頼性と処理能力が生命線

### 価値マトリクスの作成

選択した3軸を使って、価値を立体的に分析します。

**小売業の例（3軸：価値×事業部×多層）**

**店舗レベル**
- 価値軸：レジ待ち時間削減、在庫切れ防止、スタッフ生産性向上
- 事業部軸：都市型店舗、郊外型店舗、ECサイト

**地域レベル**
- 価値軸：在庫の店舗間融通、スタッフの適正配置、地域特性への対応
- 統合効果：都市型の在庫を郊外型へ、繁忙店舗への応援体制

**本部レベル**
- 価値軸：全社在庫の最適化、需要予測の精度向上、新規出店の意思決定
- 経営インパクト：在庫回転率15%向上、機会損失50%削減

## Claude Codeとの協働：価値の深堀り

### AIが見つける隠れた価値

人間は既存の枠組みに囚われがちですが、AIは新しい視点を提供します。

```

4. 異常検知価値
   現状：クレーム後に対応
   発見：通常と異なる発注パターンから問題を事前検知
   価値：顧客満足度の低下を未然に防止
```

## 価値設計のアンチパターン

### 1. 機能の羅列を価値と勘違いする

**誤り**：
> システムの価値：100個の新機能、最新のAI搭載、マイクロサービス化、クラウドネイティブ

**正しい定義**：
> 顧客にとっての価値：商品選定時間が50%短縮、誤発注による損失がゼロに、24時間サポートで機会損失なし、導入3ヶ月で投資回収

### 2. 内部の都合を価値にすり替える

**誤り**：
> 「価値」：部門間の連携強化（→ で、顧客にどんないいことが？）、データの一元管理（→ それで何が改善される？）、業務の標準化（→ 顧客にとっての利点は？）

**正しい視点**：
> 真の価値：部門連携により即日回答、一元管理で履歴から最適提案、標準化により品質保証

### 3. 測定できない価値に逃げる

**誤り**：
> イノベーションの促進（どう測る？）、組織文化の改善（具体的には？）、将来への投資（いつ回収？）

**測定可能な形に**：
> 新商品開発期間：6ヶ月→3ヶ月、従業員エンゲージメント：3.2→4.0、3年後の新規事業売上：50億円

## Phase 2の成功基準

Phase 2を成功させるための重要ポイントをまとめます。

### 1. ステークホルダーの巻き込み

**巻き込むべき人**

| 区分 | 対象者 |
|------|--------|
| 必須 | 事業責任者、現場のキーパーソン、主要顧客（可能なら） |
| 推奨 | 財務責任者、営業トップ、IT責任者 |

**巻き込み方**
- 価値発見ワークショップ
- 個別インタビュー
- 価値仮説のレビュー会
- 定期的な進捗共有

### 2. 価値の優先順位付け

**評価基準**
- 影響度：ビジネスへの影響度
- 実現性：実現可能性
- 緊急度：緊急度

**優先順位マトリクス**

| 優先度 | 条件 | アクション |
|--------|------|------------|
| 第1優先 | 高インパクト × 容易 | Quick Win施策、すぐ着手 |
| 第2優先 | 高インパクト × 困難 | 戦略的投資、段階的実施 |
| 第3優先 | 低インパクト × 容易 | 余力があれば、自動化で対応 |
| 対象外 | 低インパクト × 困難 | 見送り、代替案を検討 |

### 3. 継続的な価値の検証

**検証手法**
- POCによる仮説検証
- A/Bテストでの効果測定
- パイロット運用での確認
- 顧客フィードバック

**PDCAサイクル**

| フェーズ | 内容 |
|----------|------|
| Plan | 価値仮説 |
| Do | 小規模実装 |
| Check | 効果測定 |
| Act | 仮説修正 |

**実施頻度**：2週間〜1ヶ月

### 4. VMS-OKR連携による戦略的目標管理

Parasol V5のVMS（Value Milestone）は、組織で広く採用されているOKR（Objectives and Key Results）フレームワークと統合できます。

**VMS-OKR連携の構造**

```
VMS（価値マイルストーン）
  │
  ├─ VMS1: 顧客が最初の価値を体験できる状態
  │     └─ Objective: 初期価値体験の実現
  │           ├─ KR1: 新規顧客100名が価値を体験
  │           ├─ KR2: 満足度スコア4.0以上
  │           └─ KR3: 再利用意向80%以上
  │
  ├─ VMS2: 顧客が価値を認識し選択できる状態
  │     └─ Objective: 価値認識と選択促進
  │           ├─ KR1: 価値認知率60%達成
  │           ├─ KR2: 競合比較での選択率40%
  │           └─ KR3: 有料転換率25%
  │
  └─ VMS3: 顧客が主要価値を日常的に体験できる状態
        └─ Objective: 日常的価値体験の定着
              ├─ KR1: DAU/MAU比率30%以上
              ├─ KR2: 継続利用率85%以上
              └─ KR3: NPS 50以上
```

**OKRがVMSを測定する理由**

| 観点 | VMS（単独） | VMS-OKR連携 |
|------|------------|-------------|
| 達成基準 | 定性的な状態定義 | 定量的なKRで測定可能 |
| 進捗管理 | 達成/未達成の二値 | KR達成率で段階的把握 |
| 組織連携 | 設計チーム内 | 全社OKRとの整合 |
| レビュー | フェーズ完了時 | 四半期ごとに継続的 |

**戦略レビューサイクル**

VMSとOKRを連携させることで、価値の進化を継続的に管理できます。

| サイクル | 対象 | 頻度 | 主な活動 |
|---------|------|------|----------|
| 四半期レビュー | KR進捗・VMS達成 | 3ヶ月 | OKR達成状況確認、次期目標調整 |
| 年次レビュー | VL2/VL3深化・OKR再設定 | 1年 | 価値理解の更新、年間OKR策定 |
| 中期レビュー | VL1・VS構造見直し | 3-5年 | Value Stream再構築、長期戦略整合 |
| 長期レビュー | Vision/Mission対応 | 5-10年 | 組織価値体系との再整合 |

> **注意**: 四半期・年次レビューでは**VL3の深化**（顧客理解の深まりによる価値の精緻化）が主な活動です。VL1やVision/Missionの変更は長期サイクルでのみ検討します。

## まとめ：価値なくして設計なし

Phase 2で学んだことを振り返りましょう。

### 価値発見の要諦

1. **価値は顧客の変化**
   - 機能や特徴ではない
   - 顧客の状態がどう変わるか

2. **価値は多面的**
   - 直接価値だけでなく間接価値も
   - 6軸システムで立体的に捉える

3. **価値は測定可能**
   - 定量化できない価値は管理できない
   - 投資対効果を明確に

### 価値設計の実践

1. **現状の可視化から始める**
   - 価値の流れを追う
   - 出血点を特定する

2. **理想との差分を設計**
   - あるべき姿を描く
   - 段階的な実現計画

3. **継続的な検証と改善**
   - 仮説は必ず検証
   - 柔軟に軌道修正

### 次章への橋渡し

価値が明確になったら、次は「その価値を実現するために必要な能力（ケイパビリティ）」を設計します。

Phase 3では、発見した価値を、具体的な「できること」に分解していきます。抽象的な価値を、実装可能な単位にまで具体化する—— これがケイパビリティ設計の真髄です。

価値という設計図ができました。次は、その設計図を実現する「部材」を準備する番です。
