# 「価値駆動設計」各章の概要とキーメッセージ

## 第I部：良い設計の再定義

### 第1章：なぜ「正しい設計」が失敗するのか
**キーメッセージ**: 技術的正しさ≠良い設計。価値なき設計は悪い設計。

**概要**:
- 技術的に完璧だが失敗したプロジェクトの実例
- 過剰な一般化、境界の過剰分離、抽象化の深淵
- 価値の不在がもたらす設計の失敗

**読者の獲得**: 自分の設計を振り返り、価値視点の重要性に気づく

### 第2章：Parasol V5が示す「良い設計」の定義
**キーメッセージ**: 良い設計は価値から逆算され、構造的必然性を持つ。

**概要**:
- V5における良い設計の3要素（価値実現性、構造的必然性、進化可能性）
- 価値トレーサビリティという革命的概念
- 設計の良し悪しを判断する新しい座標軸

**読者の獲得**: 良い設計を判断する明確な基準を得る

### 第3章：設計品質の多面的評価
**キーメッセージ**: 設計品質は多面的。コンテキストに応じた評価が必要。

**概要**:
- 6つの品質特性（価値適合性、実装容易性、運用可能性、進化可能性、理解可能性、経済合理性）
- トレードオフマトリクスの活用
- 品質特性の優先順位付け

**読者の獲得**: 設計を多角的に評価できるようになる

## 第II部：良い設計の判断基準

### 第4章：価値からの逆算思考
**キーメッセージ**: 全ての設計判断は価値から始まる。手段から考えない。

**概要**:
- 価値の3層構造（VL1→VL2→VL3）の深掘り方法
- 価値から能力（ケイパビリティ）への変換
- 実例：ECサイトのレコメンド機能設計

**読者の獲得**: 価値起点で設計を考える習慣が身につく

### 第5章：構造の自然な発見
**キーメッセージ**: 良い構造は発明するものではなく、発見するもの。

**概要**:
- ドメインに内在する自然な境界の見つけ方
- 恣意的な分割vs必然的な分割
- コンテキストマップの正しい描き方

**読者の獲得**: 無理のない自然な設計構造を見出せる

### 第6章：複雑性との正しい向き合い方
**キーメッセージ**: 複雑性を恐れず、適切に管理する。

**概要**:
- 本質的複雑性と偶有的複雑性の見分け方
- 複雑性の隠蔽vs複雑性の除去
- シンプルさと単純さの違い

**読者の獲得**: 必要な複雑性を受け入れ、不要な複雑性を排除できる

## 第III部：良い設計の実践技法

### 第7章：設計判断のチェックリスト
**キーメッセージ**: 良い設計は体系的なチェックで実現できる。

**概要**:
- Phase別チェックリスト（価値定義、境界設計、実装設計）
- レッドフラグ（警告サイン）の見分け方
- セルフレビューの実践方法

**読者の獲得**: 日々の設計作業で使える具体的なツールを得る

### 第8章：トレードオフの解決法
**キーメッセージ**: 完璧な設計はない。適切なトレードオフが良い設計を生む。

**概要**:
- 典型的な10のトレードオフパターン
- 意思決定マトリクスの使い方
- 後悔しない選択のための判断軸

**読者の獲得**: 難しい設計判断を下せるようになる

### 第9章：進化する設計
**キーメッセージ**: 良い設計は変化に適応し、成長する。

**概要**:
- 変更に強い構造の作り方
- 価値の変化を設計に反映する方法
- 計画的なリファクタリング戦略

**読者の獲得**: 長期的に価値を生み続ける設計ができる

## 第IV部：チームで実現する良い設計

### 第10章：設計の共通言語
**キーメッセージ**: 良い設計はチームの共通理解から生まれる。

**概要**:
- ステークホルダー別の翻訳技法
- 設計意図を正確に伝える図と文書
- ユビキタス言語の実践的構築法

**読者の獲得**: チーム全体で設計を共有できるようになる

### 第11章：設計レビューの実践
**キーメッセージ**: 建設的なレビューが設計を磨き上げる。

**概要**:
- 効果的なレビュープロセスの設計
- レビュアー視点別のチェックポイント
- フィードバックの与え方・受け方

**読者の獲得**: レビューを通じて設計品質を向上できる

### 第12章：設計文化の醸成
**キーメッセージ**: 良い設計は文化から生まれる。

**概要**:
- 設計品質を重視する組織の作り方
- 失敗から学ぶポストモーテムの実践
- 設計スキルの組織的向上策

**読者の獲得**: 組織全体の設計力を高められる

## 第V部：未来の良い設計

### 第13章：AIとの共創時代の設計
**キーメッセージ**: AIは設計のパートナー。人間は価値と判断に集中する。

**概要**:
- Claude Codeを使った設計プロセス
- 人間とAIの最適な役割分担
- AI時代に必要な設計スキル

**読者の獲得**: AIを活用した新しい設計手法を習得する

### 第14章：業界別の良い設計パターン
**キーメッセージ**: 業界特性を理解した設計が真の価値を生む。

**概要**:
- 製造業：リアルタイムとバッチの使い分け
- 金融業：一貫性とスケーラビリティの両立
- ヘルスケア：安全性と革新性のバランス

**読者の獲得**: 自分の業界に最適な設計パターンを理解する

### 第15章：良い設計の未来像
**キーメッセージ**: 技術は変わっても、良い設計の本質は変わらない。

**概要**:
- 10年後の設計風景予測
- 変化に備える設計者の心構え
- 永続する設計の原理原則

**読者の獲得**: 将来にわたって通用する設計力を身につける

## 各章を貫くテーマ

1. **価値中心**: すべての判断を価値から始める
2. **具体性**: 抽象論でなく実践的な内容
3. **バランス**: 理想と現実の適切な折り合い
4. **進化**: 固定的でなく成長する設計

2. **持続性**: 長期的に通用する設計思想
3. **波及性**: チーム全体への好影響
4. **確信度**: 設計判断への自信向上
