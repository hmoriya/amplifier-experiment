# 付録参照追加ガイドライン

## 目的

全38章の本文中に適切な付録参照を追加し、読者が詳細情報にアクセスしやすくする。

## 参照追加の原則

### 1. 参照タイミング

付録参照を追加すべき箇所：
- **技術的詳細を省略した箇所**：「実装の詳細は付録X.Xを参照」
- **ツール/テンプレート紹介時**：「テンプレートは付録X.Xで提供」
- **追加学習を促す箇所**：「さらなる学習は付録X.Xへ」

### 2. 参照フォーマット

統一フォーマット：
```markdown
詳細は[付録X.X: タイトル](../appendices/appendix-X.X-filename.md)を参照してください。
```

インライン参照：
```markdown
Design Matrixの完全なテンプレート（付録4.2参照）を使用して...
```

### 3. 付録の種類と参照パターン

#### A. 実装詳細型
```markdown
この概念の詳細な実装例は、付録X.1を参照してください。
そこでは、完全に動作するコード例と設定ファイルを提供しています。
```

#### B. ツール/テンプレート型
```markdown
組織アセスメントを実施する際は、付録X.2のチェックリストを
ダウンロードして使用することをお勧めします。
```

#### C. 参考資料型
```markdown
この理論の背景にある研究については、付録X.3の参考文献
リストを参照してください。
```

### 実装例

各章の適切な箇所に以下のような参照を追加：

```markdown
## 実践ガイダンス

組織アセスメントを開始する前に、[付録6.1: POMM評価ツール]
(../appendices/appendix-6.1-pomm-tool.md)をダウンロードし、
評価基準を理解しておくことが重要です。

### 成功の条件

1. 経営層のコミットメント
2. 十分な時間の確保（最低2週間）
3. 正確なデータ収集

実施の詳細な手順は[付録6.2: アセスメント実施ガイド]
(../appendices/appendix-6.2-assessment-guide.md)で
ステップバイステップで説明しています。
```

## 優先順位

### Phase 1（即座に実施）
1. 各章の設計書を確認
2. 計画された付録をリストアップ
3. 本文中の適切な箇所を特定

### Phase 2（1週間以内）
1. 付録参照を本文に追加
2. 付録ファイルの存在確認
3. リンクの動作確認

### Phase 3（2週間以内）
1. 読者テストの実施
2. フィードバックに基づく調整
3. 最終確認
