## 詳細構成

### Section 1: フック (500-800語)
**ストーリー候補**: Netflixの「視聴時間」から「顧客生涯価値」への進化
**選定理由**: 単一指標から階層的価値体系への移行を示せる
**導入する緊張感**: なぜ視聴時間が増えても解約が減らなかったのか？

### Section 2: 問題の本質 (300-500語)
**抽象化する課題**: 局所最適化による全体価値の毀損
**3層の関心事**:
- ビジネス課題: 部門KPIの総和≠企業価値
- アーキテクチャ課題: システム指標とビジネス指標の乖離
- 実装課題: メトリクスの意味が階層で異なる

### Section 3: 核心概念 (800-1200語)
**中心となる理論/フレームワーク**: 3層価値階層モデル
**導入順序**:
1. 戦略的価値 - 企業の競争優位性（3-5年スパン）
2. 戦術的価値 - 部門/プロダクトの成果（四半期-年次）
3. 運用的価値 - 日々のオペレーション効率（日次-月次）

**ビジュアル表現**: 
- 図11-1: 価値のピラミッド構造と相互関係
- 図11-2: 価値の上昇（ロールアップ）と分解（ドリルダウン）

### Section 4: 実世界例 (600-1000語)
**選定した例**: 製造業のスマートファクトリー導入
**段階的展開**:
1. 初期状態: 機械稼働率のみで管理
2. 課題認識: 稼働率は高いが収益性が低い
3. 解決適用: 3層価値体系（収益性→品質→稼働率）
4. 結果評価: 稼働率85%で収益性30%向上

**コードブロック計画**:
- Block 1 (5行): 価値階層の定義構造
- Block 2 (8行): 階層間のマッピングルール
- Block 3 (5行): 集約計算の基本パターン

### Section 5: 実践ガイダンス (400-600語)
**適用タイミング**: 経営計画策定、KPI体系見直し、PMO設置時
**成功条件**: 経営層の理解、中間管理層の協力、現場の納得
**よくある失敗**: 階層の過度な細分化、指標の形骸化
**チェックリスト**: 
- 各階層の目的は明確か
- 上位の価値に貢献しているか
- 測定可能で行動可能か

### Section 6: 技術統合 (300-500語)
**既存手法との関係**:
- Agile/Scrum: スプリントゴールと戦術的価値の連携
- マイクロサービス: サービスメトリクスの階層的集約
- DDD: 戦略的/戦術的/技術的境界との対応

**次章への流れ**: 定義した階層に対する具体的な測定手法
