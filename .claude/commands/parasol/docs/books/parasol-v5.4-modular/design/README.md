# Parasol V5.4 書籍設計ドキュメント

## 概要

このディレクトリには、Parasol V5.4書籍の設計関連ドキュメントを格納します。`parasol-book-architect`エージェントが作成・管理する設計書により、38章の一貫性と品質を保証します。

## ドキュメント構成

### 1. 全体設計書
**ファイル**: `../BOOK_DESIGN.md`
**内容**: 
- 書籍全体の構造設計
- 読者層プロファイル
- 学習フローとナラティブアーク
- 品質基準と成功指標

**使用タイミング**:
- 初期設計時（Phase 1）
- 大規模な構成変更時
- 全体レビュー時

### 2. 章別設計書
**ファイル**: `chapter[XX]_[title].md`
**テンプレート**: `CHAPTER_DESIGN_TEMPLATE.md`
**内容**:
- 章の詳細構成
- 読者層別価値提供
- 前後章との関係性
- コード配置計画

**使用タイミング**:
- 各章執筆開始前
- 章のレビュー時
- リファクタリング時

### 3. 関連ドキュメント

**スタイルガイド**: `../STYLE_GUIDE.md`
- 用語統一
- トーンと文体
- 章構成テンプレート

**コンテキストテンプレート**: `../CHAPTER_CONTEXT_TEMPLATE.md`
- セッション開始時の必須情報
- 前章要約の管理
- 読者層別ガイダンス

**変換ステータス**: `../TRANSFORMATION_STATUS.md`
- 各章の進捗管理
- コード比率の追跡
- 優先順位の可視化

## 設計プロセス

### Phase 1: 初期設計（ANALYZE Mode）

1. **全体構造の分析**
   ```
   エージェントへの指示:
   "ANALYZE Modeで書籍全体の構造を分析し、BOOK_DESIGN.mdを更新してください"
   ```

2. **章別設計の作成**
   ```
   エージェントへの指示:
   "Chapter [X]についてANALYZE Modeで詳細設計を作成してください"
   ```

### Phase 2: 設計レビュー

1. **整合性確認**
   - 前後章との論理的つながり
   - 読者層別価値の一貫性
   - 学習目標の達成可能性

2. **リスク評価**
   - コード比率の予測
   - 複雑性の管理
   - 依存関係の確認

## 設計書の活用方法

### 執筆者向け

1. **執筆開始前**
   - 該当章の設計書を確認
   - 前章の設計書で継続要素を把握
   - 次章の設計書で準備要素を確認

2. **執筆中**
   - 6セクション構成の遵守
   - コード配分計画の実行
   - 読者層別価値の実現

3. **執筆後**
   - 品質チェックリストの確認
   - 設計と実装の差異記録
   - 次章への申し送り更新

### レビュアー向け

1. **設計書との照合**
   - 学習目標の達成度
   - 構成の準拠性
   - 品質基準の満足度

2. **改善提案**
   - 設計書への反映
   - 他章への影響評価
   - 全体最適化の検討

## 設計書のメンテナンス

### 更新タイミング

- **定期更新**: 各パート完了時
- **随時更新**: 大きな変更発生時
- **最終更新**: 書籍完成時

### バージョン管理

- 設計書も本文と同様にGit管理
- 大きな変更時はコミットメッセージに明記
- レビュー履歴の保持

## ベストプラクティス

### DO
- ✅ 設計を先行させ、執筆を後続させる
- ✅ 読者視点で価値を定義する
- ✅ 前後関係を常に意識する
- ✅ 図表とコードのバランスを計画する

### DON'T
- ❌ 設計なしで執筆を開始する
- ❌ 単独章のみで完結させる
- ❌ コード比率を後から調整する
- ❌ 読者層を混同させる
