# V5アジャイル初心者向けガイド 企画案

## ガイドタイトル

**メインタイトル**: 「はじめてのParasol V5 × アジャイル実践ガイド」  
**サブタイトル**: 「価値駆動スクラムで変わるチーム開発」

## 企画趣旨

### なぜこのガイドが必要か

1. **既存の問題点**
   - アジャイル導入したが「なぜ作るのか」が不明確
   - ベロシティは上がるが価値実現が見えない
   - V5は理解したがアジャイルとの組み合わせ方が分からない

2. **ターゲット読者のペインポイント**
   - アジャイル経験1-3年の開発者・スクラムマスター
   - 「動くソフトウェア」は作れるが価値を実感できない
   - V5を学んだが実際のスプリント運営で活用できない

3. **提供価値**
   - 明日からのスプリントで実践できる具体的手法
   - 価値を中心としたアジャイル開発の体験
   - チーム全体での価値実現を実感

## ターゲット読者

### プライマリーターゲット
- **アジャイル開発者**（経験1-3年）
- **スクラムマスター**（初級〜中級）
- **プロダクトオーナー**（V5を知りたい）

### セカンダリーターゲット
- **アジャイルコーチ**（V5手法を学びたい）
- **エンジニアリングマネージャー**（価値重視に転換したい）
- **プロジェクトマネージャー**（従来手法から脱却したい）

### 前提スキル
- スクラムの基本的な流れを理解している
- ユーザーストーリーとスプリントの概念を知っている
- 開発チームでの作業経験がある

## ガイド構成案

### 第1部：基礎理解編（30分で読める）

#### 第1章：なぜ価値駆動アジャイルが必要なのか
- アジャイルの成功と失敗の分かれ道
- 「動くソフトウェア」だけでは不十分な理由
- V5が解決する3つの根本問題

#### 第2章：V5とアジャイルの美しい統合
- 従来アジャイル vs V5統合アジャイルの違い
- 価値ストリーム思考でスクラムを強化
- 15分で理解できるV5-スクラム統合モデル

#### 第3章：価値駆動スプリントの全体像
- 価値を中心としたスプリント設計
- VLからユーザーストーリーへの自然な展開
- 成功するチームの1日の流れ

### 第2部：実践準備編（1時間で準備完了）

#### 第4章：最初のスプリント準備
- 価値定義ワークショップ（VL1→VL2→VL3）
- 価値トレーサブルバックログの作り方
- チーム合意形成のコツ

#### 第5章：スプリント活動の価値化
- Daily Standupの価値フォーカス化
- Sprint Planningでの価値優先順位付け
- Sprint Reviewの価値検証

#### 第6章：測定と改善の仕組み
- 価値メトリクスの設定方法
- レトロスペクティブでの価値振り返り
- 継続的価値向上のサイクル

### 第3部：実践習得編（実際にやってみる）

#### 第7章：ハンズオン：初回スプリント実践
- 架空プロジェクト「タスク管理アプリ」での完全実践
- VL定義から実装完了までの全工程
- よくある失敗とその対処法

#### 第8章：チーム運営の実践テクニック
- ステークホルダーとの価値合意
- 技術的負債の価値判断
- スコープ変更の価値評価

#### 第9章：スケーリングと応用
- 複数チームでの価値整合
- 大規模プロジェクトへの適用
- 組織への展開戦略

### 第4部：継続発展編（チーム文化の変革）

#### 第10章：よくある課題と解決法
- 「価値が見えない」問題の解決
- ステークホルダーの巻き込み方
- 技術とビジネスの対話促進

#### 第11章：成功事例に学ぶ
- 3つの実際のチーム事例
- Before/Afterの定量的変化
- 成功要因の分析

#### 第12章：継続的進化のために
- チーム成熟度の自己診断
- 次の学習ステップ
- コミュニティとの連携

## 特徴的なコンテンツ

### 1. 実践重視の構成
- **15分ルール**: 各章15分以内で読める
- **即実践**: 明日から使えるテンプレート・チェックリスト
- **ハンズオン**: 架空プロジェクトでの完全実践

### 2. 視覚的な理解支援
- **Before/After図**: 従来 vs V5統合の違いを視覚化
- **フローチャート**: スプリント活動の価値統合プロセス
- **ワークシート**: そのまま使える実践フォーマット

### 3. 段階的学習
- **理解**: 概念の理解（第1部）
- **準備**: 実践準備（第2部）
- **実践**: 手を動かす（第3部）
- **発展**: 組織展開（第4部）

### 4. 実用的ツール群
- **価値定義テンプレート**
- **スプリント計画シート**
- **価値測定ダッシュボード**
- **レトロスペクティブガイド**

## 想定ページ数・読了時間

- **総ページ数**: 150-200ページ
- **読了時間**: 5-7時間
- **実践時間**: 10-15時間（ハンズオン込み）

## 提供形態

### 1. 基本ガイド
- **HTML版**: ブラウザで読める形式
- **PDF版**: 印刷・オフライン閲覧用
- **モバイル対応**: スマホ・タブレット最適化

### 2. 実践支援ツール
- **テンプレート集**: Word/Excel形式
- **チェックリスト**: 印刷可能PDF
- **動画ガイド**: 重要ポイントの説明動画（10分×5本）

### 3. コミュニティ
- **オンライン質問**: 実践中の質問対応
- **事例共有**: 読者の実践事例共有
- **アップデート**: 継続的な内容更新

## 成功指標

### 読了後の変化
1. **理解度**: V5アジャイルの基本概念を説明できる
2. **実践度**: 次のスプリントで価値フォーカスを実行できる
3. **測定度**: 価値実現度を数値で把握できる
4. **改善度**: 価値ギャップに基づく改善ができる

### チーム変化
1. **満足度向上**: チームの目的意識向上
2. **効率化**: 無駄な作業の削減
3. **品質向上**: 価値に直結する品質改善
4. **ステークホルダー満足**: より良い価値提供

## 差別化ポイント

### 既存ガイドとの違い
1. **統合性**: V5とアジャイルの自然な統合
2. **実践性**: 明日から使える具体性
3. **初心者配慮**: 専門用語を使わない分かりやすさ
4. **価値重視**: 技術ではなく価値にフォーカス

### 他の学習リソースとの補完
- **理論書**: 実践的な手順を提供
- **技術書**: ビジネス価値の観点を追加
- **アジャイル本**: V5の体系性を付加
