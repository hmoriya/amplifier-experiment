# Parasol 価値トレーサビリティシステム

**作成日**: 2025-12-13  
**バージョン**: V1.0

---

## 概要

Parasol価値トレーサビリティシステムは、「想像の設計を技術的に不可能にする」フレームワークです。すべての設計判断を観測可能な価値にトレースし、構造的必然性を自動判定することで、価値の純度を保証します。

### 核心原則

1. **ゼロトラスト原則**: すべての判断を疑い、根拠を要求
2. **観測可能性**: すべての価値が測定・検証可能
3. **継続的検証**: 設計判断の妥当性を継続監視
4. **自動防御**: 想像の設計を技術的に防止

## 4つの判定基準（構造的必然性）

### 1. 価値起源の明確性 (Value Origin Clarity)
- **検証点**: この判断は、明確に定義された価値（VL3）から直接導出されているか？
- **根拠**: 価値分解階層（VL1→VL2→VL3）での位置づけ
- **NG例**: 「使いやすそうだから」「業界標準だから」

### 2. 因果関係の証明 (Causal Relationship Proof)  
- **検証点**: この判断と価値実現の因果関係が証明されているか？
- **根拠**: 測定可能なメトリクスとKPIの設定
- **NG例**: 「きっと効果がある」「他社で成功している」

### 3. 代替案の検討 (Alternative Analysis)
- **検証点**: 複数の選択肢を比較検討し、最適解を選んだか？
- **根拠**: 代替案比較表と意思決定マトリクス
- **NG例**: 「これしか思いつかない」「いつもの方法」

### 4. 測定可能性 (Measurability)
- **検証点**: この判断の効果を客観的に測定できるか？
- **根拠**: 具体的な測定指標と測定方法
- **NG例**: 「感覚的にわかる」「雰囲気で判断」

## 自動防御メカニズム

### Phase実行時の自動チェック
1. **前提条件チェック**: 前Phaseの価値トレースが完了しているか
2. **根拠データ検証**: 提供されたデータが構造的必然性を満たすか
3. **価値継承検証**: 上位価値との整合性が保たれているか
4. **想像パターン検出**: 既知の「想像の設計」パターンに該当しないか

### 価値保証ゲート
- VMS1-VMS5各マイルストーンでの必須ゲート
- 価値劣化の自動検出と防止
- フェーズ間価値継承の完全性チェック
- 価値トレーサビリティチェーンの検証

## 使用方法

### 1. システム初期化
```bash
/parasol:value-trace-init [project-name]
```

### 2. Phase実行時の自動記録
各Phaseコマンド実行時に自動で：
- 設計判断を記録
- 価値根拠を検証
- 必然性判定を実行

### 3. 価値トレース確認
```bash
/parasol:value-trace-report [phase] [subdomain]
```

### 4. 品質チェック
```bash
/parasol:value-quality-check
```

- 想像の設計による開発工数の浪費防止

- チーム全体の価値意識向上
- 価値ベースの意思決定文化の定着
- プロジェクト成功率の向上

### 長期効果
- 組織の価値創造能力向上
- 持続可能な競争優位性の確立
- ビジネス価値と技術の完全アラインメント
