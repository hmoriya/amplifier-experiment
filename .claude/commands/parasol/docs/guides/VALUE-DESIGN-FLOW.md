## 2つのバックキャスティングの視点

Phase 2では2つのバックキャスティングを行います。どちらも「理想から逆算」しますが、**視点が異なります**。

### 全体構造

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      価値定義（VL1）                                         │
│                     「顧客の成功」                                           │
└───────────────────────────┬─────────────────────────────────────────────────┘
                            │
        ┌───────────────────┴───────────────────┐
        │                                       │
        ▼                                       ▼
┌───────────────────┐                 ┌───────────────────┐
│  VL→VMS の流れ    │                 │  VS→Stage の流れ  │
│                   │                 │                   │
│  【深さの軸】     │                 │  【流れの軸】     │
│  価値の本質は何か │                 │  価値をどう届けるか│
│                   │                 │                   │
│  WHAT の深掘り    │                 │  HOW の設計       │
└───────────────────┘                 └───────────────────┘
```

### 視点1: VL→VMS（価値の深さ・本質）

**問い**: 「顧客にとっての価値の本質は何か？」

```
VL1: 顧客の成功（抽象）
  │
  ├─VL2: 時間節約      ─→ VMS5: 「作業時間80%削減」
  │   │                      │
  │   └─VL3: 自動処理        ├─VMS4: 「主要作業の自動化」
  │       └─価値必然性:★★★  ├─VMS3: 「繰返し作業の自動化」
  │                          ├─VMS2: 「手動作業の可視化」
  │                          └─VMS1: 「現状把握」
  │
  └─VL2: 品質向上      ─→ VMS5: 「エラー率0.1%以下」
      │                      │
      └─VL3: エラー検出      └─...（同様にバックキャスト）
```

**特徴**:
- **垂直方向**の分解（深く掘り下げる）
- 「なぜそれが価値なのか」を問う
- **価値必然性**を評価（TVDC分類の基盤）
- アウトプット: **達成すべき状態**の定義

### 視点2: VS→Stage（価値の流れ・体験）

**問い**: 「顧客にどのような体験の流れで価値を届けるか？」

```
VS0: 認知・発見
  │
  ▼
VS1: 興味・関心      ←─「ここで何を体験させるか？」
  │
  ▼
VS2: 検討・評価      ←─「ここで何を体験させるか？」
  │
  ▼
VS3: 購入・契約
  │
  ▼
VS4: 導入・開始
  │
  ▼
VS5: 活用・習熟      ←─「ここでVMS3-4を実現」
  │
  ▼
VS6: 成功・成果      ←─「ここでVMS5を実現」
  │
  ▼
VS7: 推奨・拡大
```

**特徴**:
- **水平方向**の設計（流れを描く）
- 「いつ・どこで価値を届けるか」を問う
- **顧客体験**の旅路を設計
- アウトプット: **価値提供のタイミングと方法**

### 両者の関係性

```
                    【VL→VMS】
                    価値の深さ
                        │
                        │ VL1
                        │  │
                        │ VL2
                        │  │
                        │ VL3 ─────────────────────────┐
                        │  │                           │
                        │ VMS5 ←───────────────────────┤
                        ▼                              │
    ─────────────────────────────────────────────────  │
    VS0 → VS1 → VS2 → VS3 → VS4 → VS5 → VS6 → VS7     │
    ─────────────────────────────────────────────────  │
                    【VS→Stage】                       │
                    価値の流れ ──────→                 │
                                                       │
                                    VMSはVSの各段階で実現される
```

### 具体例：ECサイトの場合

| 視点 | バックキャスティングの問い | 結果 |
|------|--------------------------|------|
| **VL→VMS** | 「買い物の成功」とは本質的に何か？ | VL3:「欲しいものが確実に届く」→ VMS5:「注文後24時間以内配送」 |
| **VS→Stage** | その価値をどの段階で届けるか？ | VS3（購入時）:配送日表示、VS4（導入）:追跡通知、VS6（成果）:受取完了 |

### まとめ：2つの軸

| 軸 | 問い | バックキャストの方向 | アウトプット |
|----|------|---------------------|-------------|
| **VL→VMS（深さ）** | WHATの深掘り「価値の本質は何か」 | 理想状態(VMS5)→現在(VMS1) | **達成すべき状態**の定義 |
| **VS→Stage（流れ）** | HOWの設計「価値をどう届けるか」 | 最終段階(VS7)→初期段階(VS0) | **体験の旅路**の設計 |

**両者は補完関係**にあり、VL→VMSで「何を達成するか」を定義し、VS→Stageで「どう体験させるか」を設計します。

```
│  │ CL2: 決済処理ケイパビリティ                                          │   │
│  │                                                                     │   │
│  │ 差別化度評価（Phase 3で実施）:                                       │   │
│  │   Q: 決済方式で競合と差別化できるか？                                │   │
│  │   A: いいえ。業界標準の決済方式で十分。                              │   │
│  │   結果: ★（低）                                                     │   │

│  │                                                                     │   │

│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### TVDC 4分類の適用

```
                      差別化度
                   高 ←────→ 低
                ┌──────────┬──────────┐
   価値    高   │   Core   │   VCI    │
   必然性       │  ★★★/★★★ │  ★/★★★   │
   (VL3継承)    ├──────────┼──────────┤
           低   │Supporting│  Generic │
                │  ★★/★★   │  ★/★     │
                └──────────┴──────────┘
```

| 分類 | 差別化度 | 価値必然性 | VL3の特徴 | 投資方針 |
|------|----------|------------|-----------|---------|
| **Core** | ★★★ | ★★★ | 独自性が競争優位 | 差別化投資 |
| **VCI** | ★ | ★★★ | 必須だが標準的 | 信頼性投資 |
| **Supporting** | ★★ | ★★ | 品質向上に寄与 | 効率化 |
| **Generic** | ★ | ★ | 標準機能 | コスト最適化 |
