# Parasol V5フレームワーク包括的ガイド 執筆仕様書

## 1. ドキュメント基本設計

### 1.1 ターゲット読者プロファイル
- **主要読者**: Parasol初心者（DDD経験者含む）
- **副次読者**: 中級者（実践ノウハウ習得希望）
- **想定背景知識**: 
  - 基本的なソフトウェア開発経験
  - 要件定義・設計の基礎概念理解
  - DDD概要（必須ではないが有利）

### 1.2 ガイドの目的と成功指標
- **第一目標**: V5の理念と手法を体系的に理解できる
- **第二目標**: 実際のプロジェクトでV5を適用できる
- **第三目標**: 産業特化パターンを活用して効率的に価値設計できる
- **成功指標**: 読了後に中規模プロジェクトのPhase 0-3を独力実行可能

### 1.3 総構成とページ配分
- **総ページ数**: 350-400ページ（書籍化想定）
- **パート構成**: 4部構成（基礎・理解・実践・発展）
- **章数**: 15章
- **付録**: 実践リファレンス

## 2. 章立て詳細構成

### 第I部：基礎編「Parasol V5の世界観」（80-90ページ）

#### 第1章：なぜParasol V5なのか（15-20ページ）
**目的**: モチベーションと背景理解

**構成内容**:
現代のソフトウェア開発における課題として、要件の複雑化と変化の加速、ステークホルダーの多様化、技術負債の蓄積といった問題が深刻化しています。従来の設計手法では、これらの課題に対して部分的な解決しか提供できませんでした。

Parasol V5は、これらの課題を根本的に解決するための統合的なフレームワークとして開発されました。特にClaude Code統合により、AI支援による設計品質向上と作業効率化を実現しています。

**学習目標**:
- 従来手法の限界理解
- V5の革新性認識
- AI統合の意義理解

#### 第2章：Parasol V5の全体像（20-25ページ）
**目的**: フレームワーク全体の俯瞰理解

**構成内容**:
Parasol V5は、Value Stream駆動アプローチを核とする8フェーズ（Phase 0-7）の段階的設計プロセスです。各フェーズは明確な成果物と品質ゲートを持ち、価値の一貫性を保証します。

価値トレーサビリティシステムにより、ビジネス価値から技術実装まで、すべての設計判断を追跡可能にしています。6軸探索システム（capability-axis、business-unit-axis等）により、多角的な分析と設計最適化を支援します。

**学習目標**:
- 8フェーズプロセスの理解
- 価値駆動設計の概念把握
- トレーサビリティの重要性認識

#### 第3章：Parasol哲学とマインドセット（15-20ページ）
**目的**: 設計思想の深層理解

**構成内容**:
Parasol V5の根底にある哲学は、「価値中心設計」と「構造的必然性」です。すべての設計要素は、明確なビジネス価値に紐づき、その存在理由を説明できなければなりません。

この哲学は、技術的エレガンスと実用性のバランス、継続的価値創造、ステークホルダー価値最大化といった原則として具体化されています。

**学習目標**:
- 価値中心設計思想の習得
- 構造的必然性概念の理解
- 設計判断基準の内在化

#### 第4章：V5とDDDの関係性（20-25ページ）
**目的**: 既存知識との整合性理解

**構成内容**:
Domain-Driven Design（DDD）は優れた設計手法ですが、ビジネス価値との連携やステークホルダーコミュニケーションに課題があります。Parasol V5は、DDDの強みを活かしながら、これらの課題を解決します。

具体的には、Bounded ContextをValue Streamの文脈で定義し、Aggregate設計を価値創造の観点から最適化します。DDDの戦術的パターンは、V5のCapabilityフェーズで体系的に適用されます。

**学習目標**:
- DDD知識のV5文脈での活用法
- 既存手法との相違点理解
- 統合的アプローチの利点認識

### 第II部：理解編「概念と手法の習得」（120-130ページ）

#### 第5章：Phase 0-1：プロジェクト基盤構築（20-25ページ）
**目的**: 初期フェーズの実践方法習得

**構成内容**:
Phase 0では、企業研究と基本情報収集を通じて、プロジェクトの土台を築きます。Claude Codeの`0-init`コマンドを使用し、企業URLから自動的に基本情報を抽出し、構造化された分析を実施します。

Phase 1では、ステークホルダー分析と制約条件の整理を行います。構造的必然性チェックにより、プロジェクトの妥当性を客観的に評価し、成功要因を明確化します。

**実践要素**:
- 企業研究の効率的手法
- ステークホルダーマッピング
- 制約条件分析
- 必然性判定プロセス

#### 第6章：Phase 2：価値発見と価値設計（25-30ページ）
**目的**: 価値設計の核心プロセス習得

**構成内容**:
価値発見では、6軸探索システムを活用して、多角的な価値分析を実施します。各軸（capability、business-unit、value、platform、multi-tier、fusion）は、異なる視点からの価値創造機会を明確化します。

価値設計では、発見された価値を具体的なValue Streamとして構造化します。各Value Streamは、明確な価値命題、ステークホルダー、成功指標を持ちます。

**実践要素**:
- 6軸分析の実施方法
- Value Stream設計技法
- 価値命題の明確化
- 成功指標設定

#### 第7章：Phase 3：ケイパビリティ分解（25-30ページ）
**目的**: DDD統合型設計手法習得

**構成内容**:
ケイパビリティフェーズでは、Value Streamを技術実装可能な単位に分解します。Activity Area（CL1）では、活動領域を識別しCore/Supporting/Genericの**傾向**を把握します（参考情報）。

Capability Design（CL2）では、各活動領域内のケイパビリティを特定し、**正式な**Core/Supporting/Generic分類を行います。この分類が投資判断の根拠です。Operation Definition（CL3）では、業務オペレーションを詳細化し網羅性を確保します。

**実践要素**:
- 活動領域識別（傾向的分類）
- ケイパビリティ設計（正式分類）
- オペレーション定義
- Bounded Context設計（BC）

#### 第8章：Phase 4-7：アーキテクチャから実装まで（25-30ページ）
**目的**: 後期フェーズの効率的進行方法

**構成内容**:
Application Design（Phase 4）では、ユーザーインターフェースとアプリケーションフローを設計します。Software Design（Phase 5）では、API仕様と内部構造を詳細化します。

Implementation（Phase 6）では、実装計画とテスト戦略を策定します。Platform（Phase 7）では、インフラとデプロイメント戦略を設計します。

**実践要素**:
- アプリケーション設計
- API仕様策定
- テスト戦略
- プラットフォーム設計

#### 第9章：価値トレーサビリティシステム（20-25ページ）
**目的**: 品質保証システムの理解と活用

**構成内容**:
価値トレーサビリティシステムは、すべての設計判断を価値創造の観点から追跡し、品質を保証します。構造的必然性判定、価値エビデンス収集、想像パターン検出により、設計品質を客観的に評価します。

各フェーズのゲートバリデーションにより、品質劣化を防止し、一貫した価値創造を実現します。

**実践要素**:
- トレーサビリティ設定
- 品質ゲート運用
- 価値エビデンス管理
- 品質ダッシュボード活用

### 第III部：実践編「プロジェクト実行」（100-110ページ）

#### 第10章：産業特化パターンの活用（25-30ページ）
**目的**: 業界固有の効率的設計手法

**構成内容**:
V5には、地方銀行、人材紹介、飲料食品業界など、主要産業の特化パターンが組み込まれています。これらのパターンを活用することで、業界特有の価値創造構造を効率的に設計できます。

各業界パターンには、典型的なValue Stream、ステークホルダー構造、制約条件がテンプレート化されており、カスタマイズして使用できます。

**実践要素**:
- 業界パターン選択
- カスタマイズ手法
- パターン組み合わせ
- 新規パターン作成

#### 第11章：Claude Code統合実践（20-25ページ）
**目的**: AI支援機能の効果的活用

**構成内容**:
Claude Codeとの統合により、V5の各フェーズでAI支援を受けながら高品質な設計を効率的に実現できます。自動分析、品質チェック、文書生成、設計提案などの機能を実践的に活用する方法を説明します。

特に、zen-architectエージェントとの連携による設計品質向上、自動化されたトレーサビリティ管理、AIアシストによる文書作成などを詳しく解説します。

**実践要素**:
- AIエージェント活用
- 自動品質チェック
- 文書生成最適化
- 人間-AI協調設計

#### 第12章：チーム適用とスケーリング（25-30ページ）
**目的**: 組織レベルでの導入と展開

**構成内容**:
V5をチームや組織で効果的に活用するための戦略と実践方法を解説します。段階的導入アプローチ、チームトレーニング、品質標準策定、成果測定などを体系的に説明します。

複数プロジェクトでのパターン共有、知識蓄積、継続的改善プロセスの確立により、組織全体の設計能力を向上させる方法を提示します。

**実践要素**:
- チーム導入戦略
- トレーニングプログラム
- 品質標準化
- 成果測定システム

#### 第13章：トラブルシューティングと改善（25-30ページ）
**目的**: 問題解決と継続的改善

**構成内容**:
実践で遭遇する典型的な問題とその解決方法を整理します。価値設計の迷い、ステークホルダー合意形成、技術制約への対応、品質ゲート不合格への対処などを具体的に解説します。

また、プロジェクト完了後の振り返りと改善プロセス、パターンの更新と拡充、組織知識の蓄積方法についても説明します。

**実践要素**:
- 問題診断フレームワーク
- 解決策パターン
- 振り返りプロセス
- 継続的改善システム

### 第IV部：発展編「エキスパート知識」（80-90ページ）

#### 第14章：カスタムパターンの開発（25-30ページ）

また、開発したパターンのコミュニティ共有、フィードバック収集、継続的改良プロセスについても触れます。

**実践要素**:

- テンプレート開発
- バリデーション設計
- コミュニティ貢献

#### 第15章：V5の未来と進化（25-30ページ）

読者が継続的に学習し、フレームワークの発展に貢献できるよう、学習リソース、コミュニティ参加、貢献方法を整理します。

**実践要素**:

## 3. 執筆ガイドライン

### 3.1 文体とトーン
- **基本文体**: です・ます調（親しみやすさ重視）
- **専門用語**: 初出時に必ず定義、用語集併用
- **例文**: 具体的で実践的、架空企業での事例中心
- **説明方法**: Why→What→Howの順序で段階的説明

### 3.2 構成パターン
各章は以下の統一構成とします：

1. **導入部** (10%): 章の目的と学習目標明示
2. **概念説明** (40%): 理論と原則の体系的解説
3. **実践解説** (40%): 具体的手法とテクニック
4. **まとめと次章への橋渡し** (10%): 振り返りと発展方向

### 3.3 図表活用方針
- **プロセス図**: 各フェーズの流れと成果物
- **関係図**: ステークホルダー、Value Stream関連
- **画面例**: Claude Codeインターフェース
- **チェックリスト**: 実践で使える確認項目
- **テンプレート**: そのまま使える雛形

### 3.4 実例・ケーススタディ設計
- **メイン事例**: 地方銀行の新サービス開発（全章通して継続）
- **サブ事例**: 人材紹介、飲料メーカー、EC企業等
- **ミニ事例**: 具体的テクニック説明用の小規模例
- **アンチパターン**: 失敗例と回避方法

```
理解 → 実践 → 定着 → 応用 → マスタリー
  ↑     ↑      ↑      ↑       ↑
概念  手順   演習   カスタム  創造
```

### 4.2 理解度チェックポイント
各章末に以下を設置：
- **理解確認**: 重要概念の復習問題
- **実践課題**: 小規模な実作業
- **発展問題**: 応用的思考を促す課題

### 4.3 実践への橋渡し
- **段階的複雑化**: 簡単な例から実践的事例へ
- **ツール習熟**: Claude Codeの段階的マスタリー
- **自立支援**: 独力実践のためのチェックリスト

## 5. 書籍化考慮事項

### 5.1 商業出版適応性
- **独立性確保**: 他文書への依存最小化
- **完全性**: ガイド単体での実践可能性
- **普遍性**: 技術変化に耐えうる原則重視
- **実用性**: 即座に適用可能な具体性

### 5.2 更新・改版戦略
- **コアコンテンツ**: 原則と手法（変更頻度低）
- **実践事例**: 最新動向反映（定期更新）
- **ツール説明**: バージョン変化対応
- **付録情報**: 最新情報への参照

### 5.3 多媒体対応
- **電子版**: 検索性、リンク機能活用
- **印刷版**: 図表の視認性、携帯性
- **Web版**: 最新情報、コミュニティ連携
- **動画版**: 実演を中心とした補完コンテンツ
