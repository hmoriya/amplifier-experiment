# /parasol:value-inheritance - 組織価値継承分析

企業全体の価値構造を踏まえて、対象ブランドの価値継承を分析する新フェーズ（Phase 0.5）

## なぜこのフェーズが必要か

従来のParasolでは、分析対象（ブランドや事業）を単独で価値定義していました。しかし実際の企業では：

- **ブランドの価値は親会社の価値から派生している**
- **顧客セグメントは企業全体のターゲットの一部である**
- **戦略的位置づけはグループ内での役割に依存する**

この文脈を明示化することで、「なぜこの価値なのか」が構造的に説明できるようになります。

## 使用方法

```bash
/parasol:value-inheritance analyze          # 価値継承分析を実行
/parasol:value-inheritance validate         # 既存分析の継承関係を検証
```

## 実行フロー

### Step 1: 組織トポロジーの特定

まず対象企業の組織構造タイプを特定します。

**質問**:
1. 対象企業はホールディングス構造ですか？
2. 複数の事業会社がありますか？
3. 事業会社の下に複数ブランドがありますか？

**トポロジータイプ**:

| タイプ | 構造 | 例 |
|--------|------|-----|
| 単一事業型 | Company | スタートアップ |
| コングロマリット型 | HD → OpCo[] | 総合商社 |
| マルチブランド型 | HD → OpCo → Brand[] | 消費財メーカー |
| マトリクス型 | HD → (Division × Region) | グローバル企業 |

### Step 2: 各階層の価値定義収集

**HD（ホールディングス）レベル**:
- 企業理念・ビジョン・ミッション
- グループ全体の価値観
- 情報源: 統合報告書、コーポレートサイト

**OpCo（事業会社）レベル**:
- 事業ドメインにおける価値定義
- HDからの継承と独自の付加価値
- 情報源: 事業会社サイト、事業報告

**Brand（ブランド）レベル**:
- ブランド固有の価値提案
- ターゲット顧客セグメント
- 情報源: ブランドサイト、マーケティング資料

### Step 3: 継承タイプの特定

**HD → OpCo の継承タイプ**:

| タイプ | 説明 | 例 |
|--------|------|-----|
| 特殊化 | HD価値を特定事業に適用 | 「おいしさ」→「飲料でのおいしさ」 |
| 地域化 | HD価値を特定地域に適用 | グローバル→日本市場 |
| 拡張 | HD価値に要素を追加 | 「健康」→「健康×楽しさ」 |
| 精緻化 | HD価値を具体化 | 「顧客価値」→「BtoC価値」 |

**OpCo → Brand の継承タイプ**:

| タイプ | 説明 | 例 |
|--------|------|-----|
| セグメント化 | 特定顧客層に特化 | 「全世代」→「健康志向層」 |
| 補完 | グループ内で異なる役割 | 炭酸と乳酸菌で補完 |
| 分岐 | 異なる方向に発展 | 機能性→美容 |
| 精緻化 | より具体的に定義 | 「健康飲料」→「乳酸菌飲料」 |

### Step 4: 顧客セグメントツリーの構築

セグメントの包含関係を明示化します：

```
HD顧客セグメント: 「すべての消費者」
  └─ OpCo顧客セグメント: 「飲料を楽しむ消費者」
       └─ Brand顧客セグメント: 「健康を意識する消費者」
```

**検証ルール**: Brand顧客 ⊆ OpCo顧客 ⊆ HD顧客

## 既存分析との統合

Phase 0.5の結果は、既存のPhase 1-3分析と以下のように接続されます：

```
Phase 0.5: 価値継承分析（なぜこの価値か）
    ↓ 継承された価値をInputに
Phase 1: コンテキスト分析（現状把握）
    ↓
Phase 2: 価値定義（VL1-VL3）← 継承された価値を深掘り
    ↓
Phase 3: VStream設計
```

## ベストプラクティス

1. **公開情報を活用** - IR資料、統合報告書は信頼性の高い情報源
2. **継承理由を明記** - 「なぜ」を常に説明できる状態に
3. **セグメント検証** - 包含関係の論理的整合性を確認
4. **既存分析との整合** - Phase 1-3との矛盾がないか確認
