# サントリー - バリューストリーム探索ナレッジ

## 基本情報

| 項目 | 内容 |
|------|------|
| **企業** | サントリーホールディングス |
| **業界** | 飲料・食品（酒類、飲料、健康食品） |
| **探索期間** | 2024年12月 |
| **選択した軸** | 価値軸（統合型） |
| **探索手法** | 4軸並行探索（git worktree活用） |

## コンテキスト

### 企業特性
- 多角化企業（酒類、飲料、健康食品、外食など）
- 強い企業理念「水と生きる」
- グローバル展開（ビーム社買収など）
- サステナビリティへの強いコミットメント

### 探索の目的
- Phase 2（価値定義）として、最適なバリューストリーム構造を決定
- 複数の軸からのアプローチを並行探索し、比較評価

## 軸選択の判断

### 検討した軸（4軸並行探索）

| 軸 | アプローチ | 評価 |
|----|----------|------|
| 価値軸 | 「水と生きる」を核に統合型VS定義 | **採用** |
| 事業部軸 | 酒類/飲料/健康食品で独立VS | 不採用 |
| フュージョン型 | 共通層+事業固有層 | 次善 |
| ケーパビリティ型 | 機能別に再編 | 不採用 |

### 採用理由（価値軸）

1. **企業理念との整合**
   - 「水と生きる」が全事業を貫く強い軸となる
   - サステナビリティ戦略との自然な接続

2. **事業横断シナジーの可視化**
   - 研究開発（発酵技術→酒類・健康食品）
   - 原材料調達（水・穀物の共同調達）
   - ブランド力の相互強化

3. **DXの統一基盤**
   - 消費者データの事業横断活用
   - 共通データプラットフォーム構築の正当化

### 除外理由

**事業部軸**:
- 既に事業部間の連携が進んでおり、縦割り強化のリスク
- サントリーの強みである「総合力」が活かせない

**ケーパビリティ型**:
- 組織変革の規模が大きすぎる
- 事業固有のノウハウが希薄化するリスク

## 学んだこと

### うまくいった点

1. **4軸並行探索の有効性**
   - git worktreeで4つの視点を同時に探索
   - バイアスなく比較評価できた
   - 「なぜこの軸を選んだか」の説明力が向上

2. **企業理念の活用**
   - 「水と生きる」がVS0の核として機能
   - 抽象的な理念が具体的なVS設計指針に

3. **評価レポートの構造**
   - ASCII図表による視覚化が効果的
   - 多視点ディスカッション形式で論点整理

### 苦労した点

1. **事業間の粒度調整**
   - 酒類（成熟市場）と飲料（成長市場）で戦略が異なる
   - 統一VSで両方をカバーする抽象度の設定が難しい

2. **ステークホルダーの想定**
   - 実際のステークホルダー不在での探索
   - 事業部側の反発シナリオを想定しきれない

### 予想外だったこと

- フュージョン型が「次善」として有力だった
- 段階的導入を考えると、フュージョン型→価値軸への移行パスが現実的

## 発見したパターン

### 飲料・食品業界パターン

1. **理念駆動型VS設計**
   - 食品企業は「何のために存在するか」が強い
   - VS0に理念を据えると全体が整合しやすい

2. **研究開発の横展開構造**
   ```
   基礎研究（発酵、健康科学）
       ↓
   事業別応用（酒類、飲料、健康食品）
   ```

3. **サステナビリティの統合**
   - VS7だけでなくVS0-VS4全体に組み込む必要
   - 特に水資源はVS全体を貫く

### 多角化企業パターン

1. **共通層の重要性**
   - VS0（戦略）、VS6（データ）、VS7（オペレーション）は共通化メリット大
   - VS2-VS4は事業固有性が高い

2. **段階的統合アプローチ**
   - 一気に価値軸は抵抗が大きい
   - 共通層から始めて徐々に拡大

## 次回への提言

### 同業界（飲料・食品）

1. **企業理念を最初に確認**
   - 強い理念があれば価値軸が有力
   - 理念が弱ければフュージョン型から

2. **事業間シナジーの棚卸し**
   - 既存のシナジー（研究、調達、物流）を洗い出す
   - シナジーが多ければ価値軸、少なければ事業部軸

3. **サステナビリティを軸に**
   - 食品業界はサステナビリティが共通課題
   - これを統合の起点にできる

### 多角化企業全般

1. **4軸探索は時間がかかるが価値がある**
   - 工数：各軸1-2時間 × 4 + 比較評価1時間
   - ROI：軸選択の説得力が格段に向上

2. **git worktreeは有効な探索手法**
   - 並行作業でバイアス排除
   - 成果物を残しつつ比較可能

3. **フュージョン型を「保険」として**
   - 価値軸が理想でも、フュージョン型で段階的に進める選択肢を残す
