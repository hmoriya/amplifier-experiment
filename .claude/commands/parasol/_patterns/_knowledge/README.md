# バリューストリーム ナレッジベース

## 概要

このディレクトリには、バリューストリーム設計を通じて得られた知見を蓄積します。各プロジェクトでの探索結果から学びを抽出し、将来のプロジェクトに活用します。

## ナレッジの種類

| 種類 | 説明 | ファイル命名 |
|------|------|-------------|
| **プロジェクト知見** | 個別プロジェクトからの学び | `{company}-{year}-{month}.md` |
| **パターン発見** | 業界横断で発見したパターン | `pattern-{name}.md` |
| **アンチパターン** | 避けるべきアプローチ | `antipattern-{name}.md` |
| **ベストプラクティス** | 推奨アプローチ | `bestpractice-{name}.md` |

## ナレッジ追加のタイミング

バリューストリーム設計を完了した際に、以下を記録してください：

1. **軸選択の判断根拠** - なぜその軸を選んだか
2. **うまくいった点** - 再利用可能なアプローチ
3. **苦労した点** - 次回への改善ポイント
4. **発見したパターン** - 業界特有の構造
5. **ステークホルダーの反応** - 何が響いたか

## テンプレート

新しいナレッジを追加する際は、以下の構造を使用してください：

```markdown
# {プロジェクト名} - バリューストリーム探索ナレッジ

## 基本情報
- **企業**:
- **業界**:
- **探索期間**:
- **選択した軸**:

## コンテキスト
（企業の状況、課題、目標を簡潔に）

## 軸選択の判断
### 検討した軸
### 選択理由
### 除外理由

## 学んだこと
### うまくいった点
### 苦労した点
### 予想外だったこと

## 発見したパターン
（業界・企業特有の構造、再利用可能な知見）

## 次回への提言
（同様のプロジェクトへのアドバイス）

## 関連ドキュメント
- 成果物へのリンク
- 参照したテンプレートへのリンク
```

## 活用方法

1. **新規プロジェクト開始時**: 同業界のナレッジを確認
2. **軸選択時**: 過去の判断根拠を参照
3. **設計時**: パターン・アンチパターンを確認
4. **完了時**: 新しいナレッジを追加
