# 価値軸パターン (Value Axis)

## 概要

**価値軸**は、企業全体を「顧客への価値創造フロー」という単一の視点で捉えるアプローチです。事業部の境界を超えて、価値がどのように生まれ、届けられるかを可視化します。

## 適用条件

| 条件 | 説明 |
|------|------|
| **推奨** | 事業横断シナジーを発見したい場合 |
| **推奨** | 顧客中心の変革を目指す場合 |
| **推奨** | 企業理念・パーパスを軸に統合したい場合 |
| **注意** | 事業間の独立性が高い場合は抵抗が生じやすい |

## 基本構造

```
VS0: ビジョン・戦略立案
 ↓
VS1: 市場機会発見・インサイト獲得
 ↓
VS2: 製品開発・イノベーション
 ↓
VS3: ブランディング・マーケティング
 ↓
VS4: 販売・流通・サプライチェーン
 ↓
VS5: カスタマーサポート・品質保証
 ↓
VS6: データ活用・DX
 ↓
VS7: 業務改善・オペレーショナルエクセレンス
```

## 強み

- **シナジー発見**: 事業間の共通価値創造ポイントが見える
- **顧客視点統一**: 全社で一貫した顧客体験を設計可能
- **理念との接続**: 企業パーパスとの紐付けが明確

## 弱み

- **抽象度が高い**: 現場との距離が生じやすい
- **合意形成**: 事業部間の調整が必要
- **段階的適用困難**: 全社一括での適用が前提になりがち

## CL2 = BC = チーム境界

価値軸パターンにおいても、CL2レベルがBC（Bounded Context）となります：

```
┌─────────────────────────────────────────────────────────────────────┐
│  CL2 = BC = チーム境界（Conway's Law準拠）                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  VS3: ブランディング                                                │
│  │                                                                  │
│  └─ CL1: マーケティング活動                                         │
│     │                                                               │
│     └─ CL2: キャンペーンオーケストレーションチーム                   │
│        │   = BC: campaign-orchestration-bc                         │
│        │                                                            │
│        ├─ CL3: キャンペーン企画オペレーション                       │
│        │   └─ Actor UC → View                                      │
│        │                                                            │
│        └─ CL3: 効果測定オペレーション                               │
│            └─ Actor UC → View                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### Phase 4連携

| Phase | 活動 | 成果物 |
|-------|------|--------|
| Phase 3 | CL2（チーム境界）定義 | 価値フロー別チーム構成 |
| Phase 4 | BC→サービスマッピング | Context Map、サービス境界 |
| Phase 5 | API/DB設計 | ドメイン言語、スキーマ |

## 設計のポイント

1. **企業理念をVS0の核に据える**
   - パーパスや経営理念が全VSを貫く軸となる

2. **事業横断の共通言語を定義**
   - 各VSで使う用語を統一し、認識のズレを防ぐ

3. **シナジーポイントを明示**
   - どのVSで事業間連携が価値を生むか可視化

## 業種別の適用例

### 飲料・食品（例：サントリー）
- VS0に「水と生きる」を据え、全事業を貫く価値軸とする
- 研究開発（VS2）での知見が酒類・飲料・健康食品に横展開

### 総合商社
- VS0に「価値創造の仕組みづくり」を据える
- 各事業を「バリューチェーン上の役割」として再定義

## 関連パターン

### 多段VStrパターンとの関係

価値軸パターンは「顧客への価値創造フロー」を単一視点で捉えるが、組織成熟度が高い場合は**多段VStr**への拡張を検討できる：

| パターン | 価値の受け手 | 適用条件 |
|---------|-------------|---------|
| **価値軸（標準）** | 顧客のみ | 顧客VStrを確立したい初期段階 |
| **多段VStr** | 顧客+従業員+パートナー | EX/PXが経営課題、組織成熟度が高い |

**拡張の判断基準**:
- 従業員体験（EX）が経営課題になっているか？
- パートナーエコシステムが重要な競争要因か？
- 顧客VStrが既に確立されているか？

→ 詳細は `multi-tier-vstr-axis.md` 参照
