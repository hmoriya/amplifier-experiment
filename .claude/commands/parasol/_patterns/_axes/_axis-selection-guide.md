# 軸パターン選択ガイド（価値駆動ベース）

**バージョン**: V5.15
**作成日**: 2025-12-12

---

## 概要

本ガイドは、パラソルの価値駆動アプローチに基づき、適切な軸パターンを選択するための基準を提供します。

**価値駆動の核心**:
```
VL1（最上位価値）→ VL2（価値領域）→ VL3（価値要素）
    ↓ バックキャスティング
VS（バリューステージ）→ Capability → Operation
```

軸パターンの選択も「**価値体系の設計可能性**」を焦点に判断します。

---

## 1. 価値駆動ベースの選択基準（15点満点）

### 1.1 主要評価軸（価値駆動観点）

#### 【軸1】VL1統一可能性（0-3点）

**評価の焦点**: 全事業で共通する「最上位価値」を定義できるか？

| スコア | 基準 | 例 |
|--------|------|-----|
| 0点 | 全く異なる価値訴求 | 金融+製造+小売の非関連多角化 |
| 1点 | 抽象化すれば共通可能 | 「便利さ」など汎用的な表現のみ可能 |
| 2点 | 自然に共通価値を設定可能 | 「おいしさと健康」「快適な暮らし」 |
| 3点 | 既に共通価値が企業理念として浸透 | 「水と生きる」（サントリー） |

**なぜ重要か**:
- VL1が統一できれば、全社の活動を同一方向に整合させられる
- VL1が統一困難な場合、事業別VStrで個別管理が必要

---

#### 【軸2】顧客価値ジャーニーの共通性（0-3点）

**評価の焦点**: 顧客の「価値実感プロセス」（VS0→VS7）が事業間で共通か？

| スコア | 基準 | 例 |
|--------|------|-----|
| 0点 | 全く異なるジャーニー | B2B長期契約 vs B2C即時消費 |
| 1点 | 部分的に共通 | VS1-3は共通、VS4-7は異なる |
| 2点 | 大半が類似構造 | 購買プロセスは類似、利用体験が異なる |
| 3点 | ほぼ同一構造 | 同一顧客層への類似サービス |

**なぜ重要か**:
- ジャーニーが共通なら、VS単位での共通設計が可能
- ジャーニーが異なれば、事業別のVS定義が必要

**注意**: 顧客セグメント（属性）ではなく、価値体験の構造で判断

```
❌ 誤った判断: 「顧客が違うから分離」
   → 主婦と高齢者は属性が違うが、価値体験構造は類似かもしれない

✅ 正しい判断: 「価値体験の構造が違うから分離」
   → B2B長期契約とB2C即時消費は、VS構造が根本的に異なる
```

---

#### 【軸3】価値創出ケイパビリティの共通性（0-3点）

**評価の焦点**: VL1実現に必要な「能力」がどの程度共通か？

| スコア | 基準 | 例 |
|--------|------|-----|
| 0点 | 完全に異なる能力セット | 金融専門性 vs 製造技術 |
| 1点 | 一部共通 | R&Dのみ共通、販売は別系統 |
| 2点 | 大半が共通 | 製造・R&D・品質保証が共通 |
| 3点 | ほぼ同一能力セット | 全事業で同一ケイパビリティを活用 |

**なぜ重要か**:
- ケイパビリティが共通なら、統合型やフュージョン型が効率的
- ケイパビリティが異なれば、事業部軸で個別最適化が必要

**注意**: 「技術」ではなく「価値創出能力」の観点で判断

```
技術の共通性: 発酵技術、FD技術
    ↓ 価値創出能力に変換
価値創出能力: 「おいしさを保つ能力」「健康価値を創出する能力」
```

---

#### 【軸4】価値トレーサビリティの複雑度（0-3点）

**評価の焦点**: Operation→VS→VL1への追跡がどの程度複雑になるか？

| スコア | 基準 | 例 |
|--------|------|-----|
| 0点 | シンプルに追跡可能 | 単一事業、明確な因果関係 |
| 1点 | 一部複雑だが管理可能 | 2-3事業、因果関係は把握可能 |
| 2点 | 複雑で分割管理が必要 | 4-5事業、因果関係の追跡に工夫要 |
| 3点 | 極めて複雑で完全分離が必要 | 多数事業、因果関係の追跡困難 |

**なぜ重要か**:
- トレーサビリティが複雑だと、価値貢献度の測定・管理が困難
- 複雑な場合は事業単位での分離管理が現実的

**トレーサビリティの検証方法**:
```
この活動はどのVSに貢献するか？
    ↓ 説明可能か？
このVSはどのVL3に貢献するか？
    ↓ 説明可能か？
このVL3はVL1にどう接続するか？
    ↓ 説明可能か？

全て説明可能 → トレーサビリティ確保
説明困難 → 分割管理を検討
```

---

#### 【軸5】価値貢献評価の統一可能性（0-3点）

**評価の焦点**: 全事業を同一基準で価値貢献度評価できるか？

| スコア | 基準 | 例 |
|--------|------|-----|
| 0点 | 完全に異なる評価軸が必要 | 事業特性が根本的に異なる |
| 1点 | 部分的に共通評価可能 | 一部指標は共通、大半は個別 |
| 2点 | 大半を共通評価可能 | VS遷移率で統一評価可能 |
| 3点 | 同一評価体系で運用可能 | 全事業で同一KPI体系 |

**なぜ重要か**:
- 統一評価が可能なら、全社でのインセンティブ設計が容易
- 評価体系が異なると、事業間の公平性・比較可能性に課題

**評価統一の検証**:
```
共通評価可能な指標:
  ・VS遷移率（顧客状態遷移）
  ・NPS（顧客推奨度）
  ・LTV（顧客生涯価値）

事業固有の指標:
  ・B2B: 契約継続率、案件単価
  ・B2C: リピート率、購買頻度
  ・サブスク: MRR、チャーン率
```

---

### 1.2 スコア解釈と推奨パターン

| スコア範囲 | 特徴 | 推奨パターン |
|-----------|------|-------------|
| **12-15点** | 高い価値統一性 | **価値軸パターン** |
| **8-11点** | 中程度の価値統一性 | **フュージョン型パターン** |
| **5-7点** | 低い価値統一性だがCap共通 | **ケイパビリティ統合型パターン** |
| **0-4点** | 価値統一困難 | **事業部軸パターン** |

**特殊ケース**:
- **プラットフォーム軸**: 両面市場構造がある場合（スコアとは別判断）
- **多段VStr**: 従業員/パートナーVStrが経営課題の場合（追加オプション）

## 3. 補助評価軸（組織・市場・技術観点）

価値駆動の主要評価に加え、以下の補助情報も判断材料とします。

### 3.1 組織構造観点

| 観点 | 確認事項 |
|------|---------|
| 事業部独立性 | 事業部ごとのP/L責任、意思決定権限 |
| 機能統合度 | R&D/製造/物流などの統合・分散状況 |
| 親会社連携 | グループ共通基盤の有無 |

### 3.2 市場構造観点

| 観点 | 確認事項 |
|------|---------|
| 競合構造 | 事業間で競合が共通か異なるか |
| チャネル構造 | 販売チャネルの共通性 |
| 顧客セグメント | 顧客属性の分散度 |

### 3.3 技術・能力観点

| 観点 | 確認事項 |
|------|---------|
| 技術基盤 | コア技術の共通性 |
| ブランド構造 | マスターブランド vs 独立ブランド |
| 人材・スキル | 専門性の共通性 |

## 4. 軸パターン別の価値駆動適合性

### 4.1 価値軸パターン

**価値駆動適合条件**:
- VL1が全社で自然に統一できる
- 顧客価値ジャーニーが共通
- VS遷移率で全社統一評価が可能

**リスク**:
- 抽象度が高くなりすぎる
- 事業固有の価値創出が見えにくくなる

### 4.2 事業部軸パターン

**価値駆動適合条件**:
- VL1の統一が困難または不自然
- 顧客価値ジャーニーが根本的に異なる
- 事業ごとの独自評価体系が必要

**リスク**:
- 縦割りでVL1への整合が失われる
- シナジー機会を逃す

**対策**: VL1貢献度を事業部評価に組み込む

### 4.3 フュージョン型パターン

**価値駆動適合条件**:
- VL1は統一可能だが、ジャーニーは事業で異なる
- 共通層（VS0, VS6, VS7）と事業層（VS1-VS5）の分離が自然
- 技術シナジーを共通層で活用できる

**リスク**:
- 共通層が形骸化する
- 共通層の価値貢献が測定しにくい

**対策**: 共通層をイネーブラーとして評価

### 4.4 ケイパビリティ統合型パターン

**価値駆動適合条件**:
- VL1統一は困難だが、ケイパビリティは共通
- 機能別組織での効率化が優先課題
- 各ケイパビリティの担当VSが明確

**リスク**:
- 事業成果への責任が分散
- 価値創出よりも機能効率が優先される

**対策**: ケイパビリティをVS遷移貢献度で評価

### 4.5 プラットフォーム軸パターン

**価値駆動適合条件**:
- 両面市場構造（需要者/供給者）がある
- 両面それぞれに独自のVStrが必要
- マッチング価値がVL1の核心

**リスク**:
- 片側偏重で両面バランスが崩れる
- ネットワーク効果の因果が複雑

**対策**: 両VStrの遷移率を同時モニタリング

### 4.6 多段VStrパターン

**価値駆動適合条件**:
- 従業員体験（EX）が経営課題レベル
- パートナーエコシステムが競争優位
- 顧客VStrが確立されている

**リスク**:
- VStr間の優先順位が曖昧になる
- リソース分散で全体最適を逃す

**対策**: サービス・プロフィット・チェーンで因果分析
