# 業種別バリューストリームパターン

## 概要

Parasol V5では、業種の特性に応じて3つの基本パターンと、それぞれの業種特有の展開パターンを定義しています。

## 基本パターン分類

| パターン | 説明 | 適用条件 | 業種例 |
|----------|------|----------|---------|
| **標準型** | 単一顧客視点 | 顧客セグメントが単一または同質的 | 製造業、SaaS、小売 |
| **両面市場型** | 2セグメントの相互依存 | 2つの異なる顧客が相互に価値を提供 | 人材派遣、不動産仲介 |
| **多面ステークホルダー型** | 3セグメント以上の相互作用 | 複数のステークホルダーが複雑に絡み合う | 地方銀行、病院 |

## 1. 標準型パターン

### 基本構成（8つのVS）

| VS | 名称 | 共通的な目的 |
|----|------|-------------|
| VS0 | ビジョン・戦略立案 | 組織の方向性と戦略的意思決定 |
| VS1 | 市場機会発見・インサイト獲得 | 市場理解と成長機会の特定 |
| VS2 | 製品開発・イノベーション / 顧客体験 | 核となる価値の創造 |
| VS3 | ブランディング・マーケティング / オペレーション | 価値の伝達と実現 |
| VS4 | 販売・流通・サプライチェーン | 価値の届け方 |
| VS5 | カスタマーサポート・品質保証 | 価値の維持と保証 |
| VS6 | データ活用・DX | デジタル化による価値増幅 |
| VS7 | 業務改善・オペレーショナルエクセレンス | 継続的な最適化 |

### 業種別重点VS

#### 製造業・消費財
- **重点VS**: VS2（製品開発）、VS4（サプライチェーン）
- **特徴**: 物理的製品の開発と効率的な流通
- **KPI例**: 新製品開発サイクル、在庫回転率、製造原価率

#### SaaS・ソフトウェア
- **重点VS**: VS2（カスタマーサクセス）、VS3（プロダクト開発）
- **特徴**: 継続的な価値提供とリテンション重視
- **KPI例**: MRR/ARR、チャーンレート、NPS

#### 金融サービス
- **重点VS**: VS5（規制対応・リスク管理）、VS2（顧客体験）
- **特徴**: 厳格な規制下での顧客価値創造
- **KPI例**: 規制遵守率、リスク調整後収益、顧客満足度

#### ヘルスケア・医療
- **重点VS**: VS5（医療安全・品質）、VS2（患者体験）
- **特徴**: 生命に関わる品質管理と患者中心アプローチ
- **KPI例**: 医療事故率、患者満足度、治療成果

#### 小売・Eコマース
- **重点VS**: VS2（顧客体験）、VS4（物流・フルフィルメント）
- **特徴**: オムニチャネル体験と迅速な配送
- **KPI例**: コンバージョン率、配送時間、在庫精度

#### プロフェッショナルサービス
- **重点VS**: VS2（サービスデリバリー）、VS6（知識管理）
- **特徴**: 専門知識の価値化とナレッジマネジメント
- **KPI例**: 稼働率、プロジェクト利益率、顧客リピート率

#### メディア・エンターテインメント
- **重点VS**: VS4（コンテンツ制作・配信）、VS2（視聴者体験）
- **特徴**: コンテンツの創造と多チャネル配信
- **KPI例**: 視聴率/再生数、エンゲージメント率、コンテンツROI

#### 物流・運輸
- **重点VS**: VS3（オペレーション最適化）、VS4（ネットワーク管理）
- **特徴**: 物理的な移動の効率化とネットワーク最適化
- **KPI例**: 配送精度、積載率、燃費効率

## 2. 両面市場型パターン

### 基本構造

```
セグメントA ←─【プラットフォーム】─→ セグメントB
     ↓                               ↓
  [価値提供A]                    [価値提供B]
     ↓                               ↓
     └────────【相互依存】──────────┘
```

### 統合型VSの設計原則

各VSにおいて、両セグメントの価値を同時に設計：

| VS | セグメントAの価値 | セグメントBの価値 | 相乗効果 |
|----|------------------|------------------|----------|
| VS0 | 戦略的価値A | 戦略的価値B | プラットフォーム成長戦略 |
| VS2 | 体験価値A | 体験価値B | ネットワーク効果の最大化 |
| VS3 | ブランド価値A | ブランド価値B | 統一ブランドでの信頼構築 |

### 業種別展開

#### 人材派遣（成熟期対応モデル）
- **セグメント**: スタッフ / クライアント企業
- **相互依存**: スタッフ増→案件増→スタッフ需要増
- **重点**: グッドマッチ最大化、継続的成長支援
- **成熟期戦略**: 
  - ベストマッチ（100%）追求 → グッドマッチ（70%）最大化
  - 獲得重視 → 継続フォロー重視（VS2.5が最重要）
  - マッチング率10% → 40%（市場4倍拡大）
- **詳細**: `_patterns/staffing-agency-value-streams.md`参照

#### 不動産仲介
- **セグメント**: 売主・貸主 / 買主・借主
- **相互依存**: 物件増→買い手増→物件需要増
- **重点**: 情報の透明性、取引の信頼性

## 3. 多面ステークホルダー型パターン

### 基本構造（2層アプローチ）

```
Layer 1: 分離型（セグメント別深掘り）
  → 各セグメント固有のニーズに特化したVS

Layer 2: 統合型（クロスセグメント価値創造）
  → セグメント間の相互作用を最大化するVS
```

### 地方銀行パターン（地域共創型モデル）

- **セグメント**: 個人顧客 / 法人顧客 / 行員
- **特徴**: 「地域社会への貢献」を上位目標とした3者協創
- **詳細**: `_patterns/regional-bank-value-streams.md`参照

#### 構造図

```
                    【地域社会への貢献】
    ┌────────────────────────────────────────┐
    │              地 域 社 会              │
    │  ○ 地域経済活性化  ○ 雇用創出     │
    │  ○ 地場産業育成    ○ 課題解決     │
    └─────────────┬──────────────────────┘
                  │
         【価値の循環】
                  ↓
    ┌────────────────────────────────────────┐
    │         地方銀行プラットフォーム      │
    │  個人 ←→ 地銀 ←→ 法人             │
    │    ↑       ↑       ↑              │
    │    └───── 行員 ─────┘              │
    └────────────────────────────────────────┘
```

#### 地域共創型VS構成

| VS | 名称 | 3者統合価値 | 地域貢献 |
|----|------|------------|----------|
| VS0 | 地域ビジョン・戦略立案 | 地域の10年後を共に描く | SDGs・地方創生 |
| VS1 | 地域インサイト・機会発見 | 地域課題と可能性の発見 | 人口動態・産業構造分析 |
| VS2 | 地域体験価値創造 | ライフ/成長/キャリアステージ別サポート | 世代を超えた価値創造 |
| VS3 | 地域ブランド・発信 | 地域の魅力を内外に発信 | 関係人口・移住促進 |
| VS4 | 地域経済循環 | 資金の地産地消 | 地域通貨・ESG金融 |
| VS5 | 地域セーフティネット | 金融包摂・社会的弱者支援 | 見守り・金融教育 |
| VS6 | 地域DX推進 | デジタルで地域課題解決 | キャッシュレス・行政DX |
| VS7 | 地域共創プラットフォーム | 多様なステークホルダー連携 | 産官学金連携 |

#### 価値の循環メカニズム

```
個人顧客（預金）→ 地銀（融資）→ 法人顧客（事業拡大）
    ↑                                    ↓
    └──────（雇用・給与）←──────────────┘
              ↑
         行員が価値創造を媒介
         （地域のコーディネーター）
```

#### 成功要因

1. **地域エコシステムの中核**として機能
2. **長期的視点**での地域発展（10年スパン）
3. **多様なステークホルダー**との協創
4. **社会的価値と経済的価値**の両立

### 病院パターン

- **セグメント**: 患者 / 医療従事者 / 保険者
- **Layer 1**: 患者ケア、医療従事者支援、保険最適化
- **Layer 2**: 統合医療提供、地域医療連携
- **特徴**: 医療の質と経営の持続可能性のバランス

## パターン選択ガイドライン

### 1. セグメント分析

1. **顧客セグメントの数**を特定
2. **セグメント間の関係性**を分析
   - 独立型：相互作用なし → 標準型
   - 相互依存型：片方が増えると他方も増える → 両面市場型
   - 複雑相互作用型：3者以上が絡み合う → 多面型

### 2. 内部ステークホルダーの位置づけ

- サービス品質に**間接的**に関与 → 標準型で対応可能
- サービス品質に**直接的**に関与 → 多面型を検討
  - 例：銀行員の知識・対応力が顧客価値に直結

### 3. ビジネスモデルの本質

- **製品中心** → 標準型
- **マッチング中心** → 両面市場型
- **エコシステム中心** → 多面ステークホルダー型

### 4. 地域性・社会性の考慮

- **地域限定ビジネス** → 地域貢献要素を追加検討
- **社会インフラ的役割** → より広いステークホルダーを考慮

## カスタマイズの指針

1. **基本パターンから開始**し、業種特性に応じて調整
2. **重点VSを2-3個選定**し、リソースを集中
3. **KPIは業種標準**を参考にしつつ、独自指標も設定
4. **定期的な見直し**（年1-2回）で環境変化に対応

## ⚠️ 重要：バリューストリーム設計の原則

### 最重要原則：1価値定義 = 1バリューストリーム

**バリューストリームは価値定義からバックキャスティングして作成する「1本の統合されたストリーム」です。**

```
【正しい設計】
価値定義（VL3）
    ↓ バックキャスティング
┌─────────────────────────────────────────────────────┐
│   1本のバリューストリーム                            │
│   VS0 → VS1 → VS2 → VS3 → VS4 → VS5 → VS6 → VS7    │
│                                                     │
│   ※VS0-VS7はストリーム内の「ステージ」（直列）      │
│   ※複数の並列ストリームではない                    │
└─────────────────────────────────────────────────────┘
```

### パターン別バリューストリーム数

| パターン | 価値定義 | VStr数 | 例 |
|---------|---------|--------|-----|
| **統合型（価値軸）** | 1つの統合価値 | 1本 | 全社共通VStr |
| **セグメント型（事業部軸）** | 事業部別価値 | 事業部数 | 飲料VStr、酒類VStr、その他VStr |
| **ハイブリッド型** | 統合 + セグメント | 1 + N | 統合VStr + 事業部別VStr |

### アンチパターン（避けるべき設計）

| ❌ 誤った設計 | ✅ 正しい設計 |
|-------------|-------------|
| 機能別に複数VStr作成（価値創造VStr、価値提供VStr等） | 1本のVStrにVS0-VS7として統合 |
| 製品別にVStr作成（天然水VStr、茶VStr、ウイスキーVStr等） | 事業部レベル（飲料VStr、酒類VStr）で各1本 |
| バリューステージ（VS0-VS7）を並列ストリーム扱い | VSはストリーム内のステージ（直列） |
| 11以上のVStrを並列作成 | 価値定義の数と同じ本数のVStr |
