# 人材派遣業 - 事業部軸バリューストリーム

## 概要

人材派遣業における事業部軸アプローチは、派遣形態や業種別に組織を構成するモデルです。大規模派遣会社で事業セグメントが明確に分離されている場合に適用可能です。

## 構造

```
┌─────────────────────────────────────────────────────────┐
│                    共通層（本部）                        │
│  VS0: 経営戦略・コンプライアンス・ブランド管理           │
│  共通機能: IT基盤、給与計算、法務、人事                 │
└─────────────────────────────────────────────────────────┘
        │               │               │
        ▼               ▼               ▼
   ┌─────────┐     ┌─────────┐     ┌─────────┐
   │一般派遣  │     │紹介予定  │     │技術者   │
   │ 事業部  │     │ 派遣    │     │  派遣   │
   │ VS1-VS7 │     │ VS1-VS7 │     │ VS1-VS7 │
   └─────────┘     └─────────┘     └─────────┘
```

## 共通層（本部）

### VS0: 経営戦略・コンプライアンス

**責任範囲**:
- グループ全体の成長戦略
- 労働者派遣法対応
- コンプライアンス体制
- ブランド統一管理

**共通機能**:
| 機能 | 内容 |
|------|------|
| IT基盤 | 基幹システム、マッチングエンジン |
| 給与・経理 | 給与計算、請求管理、債権管理 |
| 法務 | 契約管理、派遣法対応、紛争対応 |
| 人事 | 社内人材、研修体系 |

## 事業部軸の注意点

**メリット**:
- セグメント別収益管理が明確
- 専門性の深化（特に技術者派遣）
- 営業の専門化

**デメリット**:
- スタッフの事業部間異動が困難
- クライアントへのワンストップ提案ができない
- 両面市場の統合価値が活かせない
- 重複投資のリスク

**対策**:
- クロスセル専任チームの設置
- 定期的な事業部間情報共有会議
- スタッフ・クライアントDBの統合運用
- 共通KPIの設定（グループLTV等）
