## V5.6 新機能の検証結果

### 1. Operational Stage（OS）定義 ✅ 検証完了

**V5.5までの課題**
- Value StreamとCapabilityの関係が不明確
- 「いつ」「どの段階で」Capabilityが活性化するかが定義されていなかった

**V5.6での改善**
- Value Stream内にOperational Stage（OS）を定義
- OS と Capability の1:1対応を明確化
- 各OSにvalue_accumulationを定義し、価値蓄積を可視化

**カルピス事例での適用**
```
VStr-1: ブランド認知・選好ストリーム
  ├── OS-1-1: ブランドメッセージ発信 → CL2-1-1 ブランドコミュニケーション
  ├── OS-1-2: 健康価値訴求 → CL2-1-2 健康エビデンス管理
  └── OS-1-3: 購買接点最適化 → CL2-1-3 購買接点管理

VStr-2: 初回体験ストリーム
  ├── OS-2-1: 製品品質確保 → CL2-2-1 発酵プロセス管理
  └── OS-2-2: 流通・配荷 → CL2-2-2 サプライチェーン管理
```

**検証結果**: OSとCapabilityの1:1対応により、「どの価値創出段階でどのCapabilityが動くか」が明確化された。

---

### 2. Stage依存TVDC ✅ 検証完了

**V5.5までの課題**
- 1つのCapabilityに1つのTVDC分類のみ
- 顧客ステージによる重要度の変化を表現できなかった

**V5.6での改善**
- 同一CapabilityがValue Stageによって異なるTVDCを持てる
- 投資判断の動的最適化が可能に

**カルピス事例での適用**
```
CL2-1-2: 健康エビデンス管理
  VS0-1: Supporting（認知段階では補助的）
  VS1-2: VCI（選好形成で差別化要因に）
  VS3-6: Core（継続利用の直接的動機）

CL2-3-2: 体験価値設計
  VS2-3: Supporting（初回は製品品質が主役）
  VS3-4: VCI（継続促進の差別化要因）
  VS4-6: Core（ロイヤルティ形成の中核）
```

**検証結果**: 顧客ジャーニーに沿った投資優先度の変化を論理的に説明可能になった。

---

### 3. Value Component（VC）分析 ✅ 検証完了

**V5.5までの課題**
- VL3の価値構成要素が不明確
- どのタイプの価値（製品/体験/関係等）が中心かの分析がなかった

**V5.6での改善**
- 5つのVC（VC1製品〜VC5情報）でVL3を分析
- VCパターン（製品主導型、体験主導型等）を特定
- VS-VC優先度マトリクスを導入

**カルピス事例での適用**
```
VL3-1-1-1「カルピス菌による発酵」
  → 製品主導型（VC1が主、VC3/VC5が副）

VL3-2-2-1「一緒に作る・割る体験」
  → 関係主導型（VC4が主、VC3/VC1が副）
```

**VS-VC優先度マトリクス**
```
VS0-1（認知）: VC5情報 + VC3体験 優先
VS1-2（選好）: VC1製品 + VC5情報 + VC3体験 優先
VS2-3（初回）: VC3体験 + VC1製品 優先
VS4-7（定着）: VC4関係 + VC3体験 優先
```

**検証結果**: VL3の多面的分析により、各ステージでどの価値次元を強化すべきか明確化。

---

### 4. 3軸価値分解 ✅ 検証完了

**V5.5までの課題**
- 価値を単一視点でしか捉えていなかった
- ステークホルダー別、時間軸別の分析がなかった

**V5.6での改善**
- ステークホルダー軸（顧客/株主/従業員/パートナー/社会）
- 時間軸（現在/将来/持続）
- 性質軸（機能的/情緒的/社会的）

**カルピス事例での適用**
```
ステークホルダー軸:
  顧客(50%): VL3-1-1-1, VL3-2-1-1, VL3-2-2-1
  株主(20%): VL3-1-2-2（ブランド価値）
  社会(10%): VL3-1-1-2（健康寿命貢献）

時間軸:
  現在(45%): VL3-2-1-1（伝統の味わい）
  将来(35%): VL3-1-1-3（機能性成分）
  持続(20%): VL3-1-1-1（発酵技術）

性質軸:
  機能的(35%): 発酵、腸内環境、機能性
  情緒的(45%): 味わい、作る体験、季節
  社会的(20%): ブランド信頼、家族絆
```

**検証結果**: 価値の多面的構造が可視化され、バランスの取れた価値設計が可能に。

## V5.5 vs V5.6 比較サマリー

| 観点 | V5.5 | V5.6 |
|------|------|------|
| VStr-Capability関係 | 間接的・不明確 | OS経由で1:1明確化 |
| TVDC分類 | Capability固定 | Stage依存で動的 |
| VL3分析 | 単純な階層構造 | VC5分類で多面分析 |
| 価値分解 | 単一視点 | 3軸（SH/時間/性質） |
| 投資判断 | 一律的 | ステージ別最適化 |

## 検証で得られた知見

### 1. OS定義の有効性
バリューストリーム内の「運用段階」を明示することで、抽象的だった価値創出プロセスが具体的なオペレーションとして理解可能になった。

### 2. Stage依存TVDCの実用性
「健康エビデンス管理」が認知段階ではSupportingだが継続段階ではCoreになるという分析は、マーケティング投資の優先度判断に直接活用できる。

### 3. VC分析の洞察力
カルピスの価値が「製品品質」だけでなく「家族関係」「情報（健康知識）」にも依存していることが定量的に把握できた。

### 4. 3軸分解の戦略的価値
情緒的価値(45%)が機能的価値(35%)を上回ることから、カルピスの競争優位は機能訴求よりも情緒訴求にあることが明確化。

## 推奨事項

### 短期的（〜3ヶ月）
1. VS1-2（選好形成）の顧客に対し、健康エビデンス（VCI）を強化
2. VS4-6（定着）の顧客に対し、体験価値設計（Core）を強化

### 中期的（3-12ヶ月）
1. Stage依存TVDCに基づく投資配分の見直し
2. VC分析結果を反映したコミュニケーション戦略の再設計

### 長期的（1年以上）
1. VL3-1-1-1（発酵技術）の持続的価値としての強化
2. 社会的価値（健康寿命貢献）の可視化と訴求

---

## 結論

Parasol V5.6の新機能は、カルピスブランドのような複雑な価値構造を持つ事例において、以下の点で有効性が確認された：

1. **構造的明確化**: OS-Capability 1:1対応による価値創出プロセスの可視化
2. **動的最適化**: Stage依存TVDCによる投資判断の精緻化
3. **多面的分析**: VC分析と3軸分解による価値構造の深い理解
4. **戦略的示唆**: VS-VC優先度マトリクスによる施策立案支援

V5.6は、価値駆動型のビジネス設計において、V5.5から大きな進化を遂げたバージョンであることが本検証で確認された。
