**プロジェクト**: KDDI / 法人向け事業
**Parasol Version**: V5
**作成日**: 2025-01-15

---

## 1. コンテキスト選択

### 選択結果

| 項目 | 内容 |
|------|------|
| **選択レベル** | 事業セグメント（ビジネス事業） |
| **対象** | ビジネス事業本部 / KDDI BUSINESS |
| **親会社** | KDDI株式会社 |

### コンテキストパス

```
KDDI株式会社（HD機能）
  └── ビジネス事業本部 ← カレントコンテキスト
        ├── KDDI BUSINESS（ブランド）
        ├── KDDI まとめてオフィス（中小企業向け）
        └── ソラコム（IoT）
```

---

## 2. 選択理由

### 2.1 なぜ事業セグメントレベルか

| 観点 | 説明 |
|------|------|
| **包括性** | 法人向けサービス全体を俯瞰できる粒度 |
| **価値設計** | 事業レベルでの価値定義が可能 |
| **実用性** | VStreamが実際のビジネスフローと対応 |

### 2.2 なぜビジネス事業本部か

| 観点 | 説明 |
|------|------|
| **市場規模** | 法人ICT市場は成長市場（DX需要拡大） |
| **戦略的重要性** | KDDIの成長ドライバー（NEXTコア事業） |
| **明確な価値提案** | 「つなぐチカラ」のB2B展開 |
| **ブランド確立** | KDDI BUSINESS（2024年〜）として独立ブランド化 |

## 4. 顧客セグメント

### 4.1 主要セグメント

| セグメント | 特徴 | 主要ニーズ |
|-----------|------|-----------|
| **大企業** | 従業員1,000名以上、グローバル展開 | 統合ICT基盤、セキュリティ、DX推進 |
| **中堅企業** | 従業員100-999名、成長フェーズ | クラウド移行、業務効率化、コスト最適化 |
| **中小企業** | 従業員100名未満 | ワンストップ、導入容易性、サポート |
| **官公庁** | 中央省庁、自治体 | 行政DX、セキュリティ、BCP |

### 4.2 業種別セグメント

製造、運輸・エネルギー、金融・保険、建設・不動産、流通・小売、サービス、教育、官公庁、農業、医療・福祉、情報通信（11業種）

## 5. 継承関係の確認ポイント

### 5.1 上位からの継承（確認必須）

ビジネス事業の価値を設計する前に、以下の継承関係を明確にします：

| 継承元 | 確認事項 |
|--------|---------|
| **企業理念 → ビジネス事業** | 「豊かなコミュニケーション社会」がB2Bでどう展開されるか |
| **VISION 2030 → ビジネス事業** | 「つなぐチカラ」がビジネス顧客にどう価値提供されるか |

### 5.2 下位への影響（参考）

| 影響先 | 考慮事項 |
|--------|---------|
| **ビジネス事業 → グループ会社** | 事業価値がKDDIまとめてオフィス、ソラコム等にどう展開されるか |
| **ビジネス事業 → サービス** | VStreamが具体的なサービス提供にどう落ちるか |
