# 制約条件

**プロジェクト**: KDDI / 法人向け事業
**Parasol Version**: V5
**作成日**: 2025-01-15

---

## 1. 分析の制約条件

### 1.1 情報アクセス制約

| 制約 | 説明 | 対応 |
|------|------|------|
| **非公開情報** | 内部KPI、詳細財務、組織図等 | 公開情報から推測 |
| **競合詳細** | 競合他社の詳細戦略 | 公開情報ベースの分析 |
| **顧客情報** | 個別顧客の契約内容 | セグメント単位での分析 |

### 1.2 利用する情報源

| 情報源 | 用途 | 信頼性 |
|--------|------|--------|
| KDDI IR資料・統合報告書 | 経営戦略、財務情報 | 高 |
| KDDI法人サイト（biz.kddi.com） | サービス情報、価値提案 | 高 |
| ニュースリリース | 最新動向、組織変更 | 高 |
| 業界レポート | 市場環境、競合分析 | 中 |

## 2. ビジネス制約

### 2.1 規制・法令制約

| 制約 | 説明 | 影響範囲 |
|------|------|---------|
| **電気通信事業法** | 通信サービスの規制 | 通信サービス全般 |
| **個人情報保護法** | データ取り扱い規制 | 顧客データ管理 |
| **サイバーセキュリティ基本法** | 重要インフラ保護 | セキュリティサービス |
| **公共調達規制** | 入札・契約ルール | 官公庁向けサービス |

### 2.2 業界標準・認証制約

| 標準・認証 | 必要性 | 対象領域 |
|-----------|--------|---------|
| ISO 27001 | 必須 | 全サービス |
| SOC 2 | 推奨 | クラウドサービス |
| PCI DSS | 対象業種で必須 | 金融・決済 |
| ISMAP | 必須 | 政府向けクラウド |

## 3. 組織・リソース制約

### 3.1 人的リソース

| 制約 | 説明 | 影響 |
|------|------|------|
| **専門人材不足** | DX/AI/セキュリティ人材の確保困難 | サービス拡大速度 |
| **営業リソース** | 全顧客への直接対応は困難 | パートナー活用必須 |
| **技術サポート** | 24時間対応の人員確保 | サービスレベル |

### 3.2 システム・インフラ

| 制約 | 説明 | 影響 |
|------|------|------|
| **既存システム** | レガシーシステムとの互換性 | 新サービス展開速度 |
| **ネットワーク容量** | 5G/IoT増加への対応 | サービス品質 |
| **データセンター** | 物理的容量・電力 | クラウドサービス拡大 |

---

## 4. 市場・競争制約

### 4.1 競争環境

| 制約 | 説明 | 対応方向 |
|------|------|---------|
| **NTT優位性** | 法人市場でのブランド・規模優位 | 差別化領域での勝負 |
| **価格競争** | 通信サービスの価格低下圧力 | 付加価値サービス強化 |
| **クラウドベンダー** | AWS/Azure/GCPとの競合・協調 | パートナー戦略 |

### 4.2 顧客側制約

| 制約 | 説明 | 対応方向 |
|------|------|---------|
| **予算制約** | 特に中小企業の投資余力 | 段階的導入プラン |
| **変更抵抗** | 既存システムからの移行抵抗 | 移行支援サービス |
| **意思決定遅延** | 大企業の稟議プロセス | 長期的関係構築 |

## 5. プロジェクト固有の制約

### 5.1 スコープ制約

| 含むもの（In Scope） | 含まないもの（Out of Scope） |
|---------------------|---------------------------|
| ビジネス事業本部全体 | パーソナル事業詳細 |
| 法人向けサービス11領域 | 金融事業詳細 |
| 顧客セグメント（大企業〜官公庁） | 海外事業詳細 |
| VL1-VL3、VStream設計 | 個別サービス施策 |

### 5.2 分析深度の制約

| 領域 | 分析深度 | 理由 |
|------|---------|------|
| 価値定義（VL） | 深い | コア分析対象 |
| VStream | 深い | コア分析対象 |
| Capability | 中程度 | 必要に応じて詳細化 |
| システム設計 | 概要 | 今回スコープ外 |

## 7. 制約への対応方針

### 7.1 情報制約への対応

- 公開情報の最大活用
- 業界一般的なベンチマークの適用
- 仮説を明示した上での分析

### 7.2 リソース制約への対応

- パートナー活用の最大化
- グループ会社との連携強化
- 段階的なサービス展開

### 7.3 競争制約への対応

- IoT/コネクテッドカーでの差別化
- 中小企業セグメントでの優位性活用
- 通信×DXの一気通貫価値の訴求
