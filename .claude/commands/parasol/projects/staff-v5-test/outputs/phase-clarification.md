# Parasol V5 フェーズと視点の整理

## 📍 視点の明確化

### Phase 3: ビジネス設計（Capability Decomposition）
**視点**: ビジネス能力の分解
**成果物**:
- CL1: ドメイン分類（Core/Supporting/Generic）
- CL2: サブドメイン（ビジネスオペレーション群）
- CL3: ビジネスオペレーション（業務活動）

### Phase 4: アプリケーションアーキテクチャ設計
**視点**: 技術的境界の定義
**成果物**:
- BC（Bounded Context）: 技術的凝集性による境界
- Context Map: BC間の関係
- マイクロサービス境界: チーム・デプロイ単位

## 🔄 同じ粒度レベル、異なる視点

```
粒度レベル
    │
    ├─ CL2（サブドメイン）     : ビジネス視点での分解
    ├─ BC（Bounded Context）   : 技術視点での境界
    └─ マイクロサービス        : 実装視点での単位
```

これらは**同じ抽象度レベル**だが、**異なる視点**から見た境界：

- **CL2**: 「この業務機能群はまとまっている」（ビジネス観点）
- **BC**: 「このデータモデルは一貫性を保つべき」（技術観点）
- **MS**: 「このチームが独立してデプロイできる」（運用観点）

## ⚠️ 避けるべき表記

- ❌ `CL2 = BC = チーム境界`
- ✅ `CL2 ≈ BC ≈ チーム境界`（同じ粒度、異なる視点）
