**プロジェクト**: サントリーホールディングス / SCM
**Parasol Version**: V5
**作成日**: 2025-01-15

## 1. VStr設計方針

### パターン選択

| 項目 | 内容 |
|------|------|
| **パターン** | 標準型（単一VStr） |
| **理由** | B2B流通顧客（卸売・小売・飲食店）への共通価値提供 |
| **顧客セグメント** | 流通顧客（統一ステージ設計） |
| **軸選択** | fusion-axis（グループ統合価値の提供） |

### 設計原則

- **主語は常に「顧客」** - 組織活動ではなく顧客の状態変化を記述
- **バックキャスティング** - VS7（理想状態）からVS0に向かって逆算設計
- **価値継承** - VL階層からの価値トレーサビリティを確保

## 3. VS詳細定義

### VS0: 供給課題潜在状態

**顧客状態**: 「安定供給や効率化の課題があるが、まだ具体的な解決策を探していない」

| 項目 | 内容 |
|------|------|
| **ステージ名** | 供給課題潜在 |
| **対象顧客** | 潜在的な流通パートナー（新規卸売・小売・飲食店） |
| **顧客のペイン** | 現取引先の欠品、納期遅延、2024年問題への不安 |
| **顧客の期待** | 安定した商品供給、効率的な物流 |

**対応するEnterprise Activity（EA）**:
- 市場動向分析
- 潜在顧客の課題把握
- 業界トレンド（2024年問題等）の発信

**関連VL要素**:
- VL2-1: 安定供給価値
- VL2-2: 効率化価値

### VS1: 認知到達状態

**顧客状態**: 「サントリーのSCM能力と供給体制を知る」

| 項目 | 内容 |
|------|------|
| **ステージ名** | 認知到達 |
| **対象顧客** | サントリー製品に興味を持つ流通業者 |

## 5. VMS（価値マイルストーン）定義

### バックキャスティングによるVMS設計

**VMS5（18ヶ月後の理想状態）**:
「流通顧客が、サントリーとの取引を通じて安定供給・効率化・品質保証・持続可能性の全ての価値を実感し、戦略的パートナーとして共に成長している」

**逆算設計**:

| VMS | 顧客価値状態 | 目標時期 | 成功指標 |
|-----|-------------|---------|---------|
| **VMS5** | 顧客が全価値を享受し共創関係にある | 18ヶ月後 | 戦略パートナー数、共同プロジェクト件数 |
| **VMS4** | 顧客が持続可能性価値を実感している | 12ヶ月後 | ESG評価向上、CO2削減実績 |
| **VMS3** | 顧客が安定供給と効率化を体験できる | 9ヶ月後 | 欠品率、リードタイム、配送効率 |
| **VMS2** | 顧客がサントリーSCMの価値を認識している | 6ヶ月後 | 認知度、選好度 |
| **VMS1** | 顧客が最初の取引価値を体験できる | 3ヶ月後 | 新規取引開始数、初回満足度 |

## 9. 設計ストーリー

### なぜこのVS設計なのか

**VS設計の理由**:

1. **B2B SCMの特性反映**
   - 一般消費者向けではなく流通パートナー向けの状態遷移
   - 取引関係の開始→深化→共創という長期視点

2. **2024年問題への対応**
   - VS4（価値獲得）とVS7（価値共創）で物流制約への対応を明示
   - 単独解決ではなくパートナーとの共同対策

3. **価値階層との整合**
   - VL2の4つの価値グループをVS全体でカバー
   - 差別化ポイント（水源管理、トレーサビリティ）をVS2で訴求

**バックキャスティングの適用**:

VS7（価値共創）の理想状態を起点に設計:
- 「大王グループとの共同輸送のような戦略的パートナーシップを全顧客に拡大」
- この状態から逆算して各VSの顧客状態を定義
