**プロジェクト**: suntory-scm-v5
**作成日**: 2026-01-20
**バージョン**: Parasol V5.7
**ステータス**: Phase 4（アーキテクチャ設計）完了

---

## 1. プロジェクトの背景

### 1.1 なぜこのプロジェクトが必要か

サントリーグループは、飲料・酒類業界のリーディングカンパニーとして、**年間約31,000億円**の売上を支えるサプライチェーンを運営しています。しかし、以下の課題が顕在化しています：

| 課題領域 | 現状の問題 | ビジネスインパクト |
|---------|-----------|------------------|
| **需要予測** | 季節変動・販促影響の予測精度不足 | 欠品・過剰在庫による機会損失 |
| **物流効率** | 配送ルートの非最適化 | 物流コスト高、CO2排出増 |
| **品質管理** | 紙ベースのHACCP記録 | トレーサビリティ遅延（リコール対応） |
| **サステナビリティ** | 環境データの分散管理 | ESG報告の工数増大 |

### 1.2 プロジェクトのゴール

**18ヶ月後の理想状態（VMS5）**:
> 「流通パートナーが、サントリーのSCMを通じて、安定供給・効率化・品質保証・サステナビリティの全ての価値を享受している」

**具体的な成功指標**:
- 需要予測精度: 85% → 95%
- 欠品率: 2.5% → 0.5%以下
- 配送効率: 30%向上（共同配送による）
- CO2排出: 20%削減
- トレーサビリティ応答時間: 24時間 → 2時間以内

## 2. Parasol V5.7 適用の意義

### 2.1 従来のアプローチの限界

従来のシステム開発では、以下の問題が発生していました：

```
従来のアプローチ:
┌─────────────────────────────────────────────────────────────────────────────────┐
│  要件定義 → 機能設計 → 実装 → テスト → 運用                                     │
│      │                                                                          │
│      └─→ 問題: 「なぜその機能が必要か」が失われる                               │
│          ・ビジネス価値との関連が不明確                                          │
│          ・優先度判断の根拠がない                                                │
│          ・変更時の影響範囲が不明                                                │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 Parasol V5.7 のアプローチ

Parasol V5.7は「価値駆動アーキテクチャ」を採用し、**すべての設計決定を価値にトレース**します：

```
Parasol V5.7 アプローチ:
┌─────────────────────────────────────────────────────────────────────────────────┐

│  Phase 0.5: 価値継承                                                             │

│  │ 「人と自然と響きあい...」→「安定供給と効率的サプライチェーン」           │    │
│  └────────────────────────────────────────────────────────────────────────┘    │
│         │                                                                        │
│         ▼                                                                        │
│  Phase 1-2: 価値定義                                                             │
│  ┌────────────────────────────────────────────────────────────────────────┐    │
│  │ VL1 → VL2 → VL3 の価値階層を定義                                         │    │
│  │ VS0-VS7 の顧客状態遷移を定義                                             │    │
│  └────────────────────────────────────────────────────────────────────────┘    │
│         │                                                                        │
│         ▼                                                                        │
│  Phase 3: ケイパビリティ                                                         │
│  ┌────────────────────────────────────────────────────────────────────────┐    │
│  │ CL1 → CL2 → CL3 のケイパビリティ階層を定義                               │    │

│  └────────────────────────────────────────────────────────────────────────┘    │
│         │                                                                        │
│         ▼                                                                        │
│  Phase 4: アーキテクチャ ← 【現在ここ】                                          │
│  ┌────────────────────────────────────────────────────────────────────────┐    │
│  │ CL3 → BC → Service のマッピング                                          │    │
│  │ マイクロサービス設計、ADR策定                                            │    │
│  └────────────────────────────────────────────────────────────────────────┘    │
│         │                                                                        │
│         ▼                                                                        │
│  Phase 5-7: 実装・運用（今後）                                                   │
│                                                                                  │
│  ═══════════════════════════════════════════════════════════════════════════    │
│  全ての設計決定が「なぜ」から「どのように」まで完全にトレース可能                │
│  ═══════════════════════════════════════════════════════════════════════════    │
│                                                                                  │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 2.3 価値継承の具体例

グループバリューがどのようにシステム機能に継承されるか：

| グループバリュー | SCM価値（VL2） | 機能（CL3） | サービス（BC） |
|-----------------|---------------|------------|---------------|
| **やってみなはれ** | 効率化価値 | AI需要予測 | DemandPlanning BC |
| **水と生きる** | 持続可能性価値 | 水使用量測定 | WaterSustainability BC |
| **Growing for Good** | 品質保証価値 | トレーサビリティ | Traceability BC |
| **利益三分主義** | 効率化価値 | 共同配送計画 | TransportPlanning BC |
