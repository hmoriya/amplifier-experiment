# CL1 ドメイン分類（TVDC傾向分類）

**プロジェクト**: サントリーホールディングス / SCM
**Parasol Version**: V5
**作成日**: 2025-01-15
**Phase**: 3-1 CL1 Domain Classification

---

## 概要

Phase 2で定義されたVL3価値要素（20項目）を、TVDC傾向分類に基づき5つのCL1ドメインに分類します。

### TVDC傾向分類基準

| 分類 | 投資配分 | 定義 | 判定基準 |
|------|---------|------|---------|
| **Core** | 40-50% | 競争優位の源泉 | 差別化★★★ かつ 価値必然★★★ |
| **VCI** | 20-25% | 価値必須だが差別化困難 | 価値必然★★★ かつ 差別化★☆☆ |
| **Supporting** | 15-25% | 業務支援・効率化 | 価値必然★★☆ かつ 差別化★★☆ |
| **Generic** | 10-15% | 汎用基盤 | 価値必然★☆☆〜★★☆ かつ 差別化★☆☆ |

## 2. CL1ドメイン詳細

### CL1-01: demand-management（需要管理）

**TVDC分類**: Core（競争優位の源泉）

#### 分類根拠

| 評価軸 | スコア | 根拠 |
|--------|--------|------|
| **価値必然性** | ★★★ | 需要予測精度がSCM全体効率の起点 |
| **差別化可能性** | ★★★ | AI活用、異業種データ連携で差別化可能 |
| **競争優位貢献** | 高 | 欠品ゼロと過剰在庫削減の両立 |

#### 対応VL3価値要素

| VL3 ID | 価値要素 | 価値必然性 | VCI候補 |
|--------|---------|-----------|---------|
| VL3-1-1 | 需要予測精度向上 | ★★★ | No（差別化可能） |
| VL3-1-2 | 安全在庫最適化 | ★★★ | Yes |
| VL3-1-3 | 季節・イベント対応 | ★★☆ | No |

#### 差別化戦略

1. **AI需要予測**: POS・天気・イベントデータ統合
2. **異業種連携**: 大王グループとの需要情報共有
3. **リアルタイム調整**: traevoによる配送状況反映

#### 投資配分: 25%

---

### CL1-02: logistics-optimization（物流最適化）

**TVDC分類**: Core（競争優位の源泉）

#### 分類根拠

| 評価軸 | スコア | 根拠 |
|--------|--------|------|
| **価値必然性** | ★★★ | 2024年問題対応の核心領域 |
| **差別化可能性** | ★★★ | スマートロジスティクスで先行 |
| **競争優位貢献** | 高 | 物流コスト競争力の源泉 |

#### 対応VL3価値要素

| VL3 ID | 価値要素 | 価値必然性 | VCI候補 |
|--------|---------|-----------|---------|
| VL3-2-1 | 配送ルート最適化 | ★★★ | No（差別化可能） |
| VL3-2-2 | 共同輸送 | ★★★ | No（差別化可能） |
| VL3-2-3 | 荷待ち時間削減 | ★★★ | Yes |
| VL3-2-4 | パレット効率化 | ★★★ | Yes |

#### 差別化戦略

1. **スマートロジスティクス**: 全体最適・計画性重視のSCM
2. **異業種共同輸送**: 大王グループとの連携拡大
3. **traevo活用**: 車両位置可視化による待機時間削減
4. **パレット革新**: 35→45ケース/パレット（29%効率向上）

#### 投資配分: 25%

---

### CL1-03: quality-assurance（品質保証）

**TVDC分類**: VCI（Value Critical Infrastructure）

#### 分類根拠

| 評価軸 | スコア | 根拠 |
|--------|--------|------|
| **価値必然性** | ★★★ | 食品安全は絶対要件 |
| **差別化可能性** | ★☆☆ | 規制要件として差別化困難 |
| **競争優位貢献** | 中 | 失敗回避（必須だが優位にはならない） |

#### 対応VL3価値要素

| VL3 ID | 価値要素 | 価値必然性 | VCI候補 |
|--------|---------|-----------|---------|
| VL3-3-1 | 温度管理 | ★★★ | Yes |
| VL3-3-2 | ロットトレース | ★★★ | Yes |
| VL3-3-3 | 賞味期限管理 | ★★★ | Yes |
| VL3-3-4 | 品質検査連携 | ★★★ | Yes |
| VL3-3-5 | リコール対応 | ★★★ | Yes |

#### VCI設計原則

1. **信頼性最優先**: 99.99%以上の稼働率要件
2. **標準準拠**: HACCP、ISO 22000対応
3. **日立連携**: トレーサビリティシステム統合
4. **コスト効率**: 差別化より安定・効率を優先

#### 投資配分: 20%

---

### CL1-04: sustainability（持続可能性）

**TVDC分類**: Core/Supporting混在

#### 分類根拠

| サブ領域 | TVDC分類 | 根拠 |
|---------|---------|------|
| **水源管理** | Core | サントリー固有の差別化要因 |
| **ESG対応** | Supporting | 規制・投資家要件（差別化困難） |

| 評価軸 | スコア | 根拠 |
|--------|--------|------|
| **価値必然性** | ★★★ | ESG要件として必須 |
| **差別化可能性** | ★★★/★★☆ | 水源管理は差別化、ESGは業界標準 |
| **競争優位貢献** | 高/中 | 領域により異なる |

#### 対応VL3価値要素

| VL3 ID | 価値要素 | 価値必然性 | VCI候補 | サブ分類 |
|--------|---------|-----------|---------|---------|
| VL3-4-1 | CO2削減物流 | ★★★ | No | Supporting |
| VL3-4-2 | リサイクル物流 | ★★★ | Yes | Supporting |
| VL3-4-3 | 水源管理 | ★★★ | No（差別化） | **Core** |
| VL3-4-4 | 容器リサイクル | ★★★ | Yes | Supporting |
| VL3-4-5 | ESGレポート | ★★☆ | Yes | Supporting |

#### 差別化戦略（Core領域）

1. **水源管理**: 「天然水の森」21,000haの管理能力
2. **サステナブルSCM**: 業界リーダーとしてのポジショニング

#### 標準対応（Supporting領域）

1. **環境規制対応**: 容器リサイクル法、省エネ法
2. **ESG開示**: 投資家向けレポーティング

#### 投資配分: 20%（Core 10% + Supporting 10%）

### CL1-05: order-management（受発注管理）

**TVDC分類**: Generic（汎用基盤）

#### 分類根拠

| 評価軸 | スコア | 根拠 |
|--------|--------|------|
| **価値必然性** | ★★☆ | 業務効率化に寄与 |
| **差別化可能性** | ★☆☆ | 業界標準EDI、差別化困難 |
| **競争優位貢献** | 低 | 基盤として必要だが優位にはならない |

#### 対応VL3価値要素

| VL3 ID | 価値要素 | 価値必然性 | VCI候補 |
|--------|---------|-----------|---------|
| VL3-2-5 | 発注業務効率化 | ★★☆ | Yes |

#### Generic設計原則

1. **標準化優先**: 業界標準EDI準拠
2. **SaaS活用**: 自社開発より外部サービス活用
3. **コスト最小化**: 投資対効果を厳格に管理
4. **API連携**: 他ドメインとの疎結合インターフェース

### TVDC分布の妥当性検証

| 分類 | 推奨範囲 | 実際配分 | 判定 |
|------|---------|---------|------|
| Core | 40-50% | 50% | ✅ 適正（上限） |
| VCI | 20-25% | 20% | ✅ 適正（下限） |
| Supporting | 15-25% | 20% | ✅ 適正 |
| Generic | 10-15% | 10% | ✅ 適正（下限） |

**総評**: SCMという業務特性上、Coreドメインへの投資比率が上限の50%となっているが、2024年問題への対応と差別化の両立を考慮すると妥当な配分。

## 6. Phase 3-2（CL2）への入力サマリー

### CL1→CL2分解方針

| CL1ドメイン | TVDC分類 | CL2分解方針 |
|------------|---------|-------------|
| demand-management | Core | 機能別に3-4サブドメインに分解 |
| logistics-optimization | Core | 機能別に4-5サブドメインに分解 |
| quality-assurance | VCI | 規制要件別に分解、統合可能性検討 |
| sustainability | Core/Supporting | Core（水源）とSupporting（ESG）を分離 |
| order-management | Generic | 最小構成で1-2サブドメイン |

### CL2設計優先度

1. **最優先**: Core領域（demand-management, logistics-optimization）
2. **優先**: VCI領域（quality-assurance）
3. **標準**: Supporting領域（sustainability-supporting）
4. **最後**: Generic領域（order-management）
