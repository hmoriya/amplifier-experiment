**プロジェクト**: アサヒグループ / カルピス価値設計
**手法**: Parasol V5.7
**作成日**: 2025-01-14

---

## 本プロジェクトの目的

### 背景

企業グループにおけるブランド価値設計には、根本的な課題が存在します。個別ブランドの価値を定義する際、「なぜその価値なのか」という問いに対して、論理的かつ構造的な説明ができないケースが多く見られます。

カルピスは1919年の創業以来100年以上の歴史を持ち、「乳酸菌」「健康」「家族の絆」といった価値を提供してきました。しかし、これらの価値がアサヒグループ全体の企業理念とどのように接続しているのか、なぜカルピスがこれらの価値を担うべきなのかは、従来の手法では明確に説明することが困難でした。

### 目的

本プロジェクトでは、**Parasol V5.7**の手法を用いて、カルピスブランドの価値設計を**アサヒグループ全体の企業価値から構造的に導出**します。これにより、以下を実現します：

1. **価値の正当性の証明**: カルピスの価値がグループ理念から論理的に導かれることを示す
2. **戦略整合性の担保**: ブランド戦略とグループ戦略の一貫性を構造的に確認する
3. **設計根拠の可視化**: VStream、Capability、システムアーキテクチャの設計判断が価値から一貫して説明可能となる

## 分析の工程と意義

### なぜこの分析工程が必要か

従来のブランド価値設計では、**「なぜこの価値なのか」を説明することが困難**でした。

| 課題 | 従来アプローチの問題 | V5.7での解決 |
|------|---------------------|-------------|
| **価値の根拠不明** | 「健康と家族の絆」と定義しても根拠が曖昧 | 上位組織の価値から**継承チェーン**で導出 |
| **戦略との不整合** | ブランド戦略とグループ戦略の整合性が個別確認 | **構造的に整合性が担保**される |
| **VStream構成の恣意性** | 顧客セグメント別に複数VStreamを任意に設計 | **価値継承構造から本数が導出**される |
| **Capability設計の断絶** | 価値とCapabilityが別々に設計される | **価値継承 → VStream → Capability**の一貫性 |

### V5.7の核心原則

この分析工程を貫く核心原則：

```
価値継承構造（基盤）
    ↓ 導出
VStreamアーキテクチャ
    ↓ 導出
Capabilityアーキテクチャ
    ↓ 導出
アプリケーションアーキテクチャ（マイクロサービス境界）
```

**意味**:
- **価値継承構造が全てのアーキテクチャの基盤**となる
- 価値構造から**VStreamアーキテクチャ**を導出（本数、構成、顧客範囲）
- VStreamから**Capabilityアーキテクチャ**を導出（共有/固有の判断）
- Capabilityから**アプリケーションアーキテクチャ**を導出（統合/分離判断）

**効果**:
- 全てのアーキテクチャ判断が**価値から一貫して導出**される
- 各アーキテクチャ間の**構造的整合性が担保**される
- 「なぜこの設計か」が**価値まで遡って説明可能**

## 価値継承チェーン（本プロジェクトの核心）

### 分析結果の概要

本分析により、カルピスの価値がアサヒグループHDの企業理念から**3段階の継承**を経て導出されることが明らかになりました。

アサヒグループHDは「期待を超えるおいしさ、楽しい生活文化の創造、健康への貢献」を掲げています。この抽象度の高い企業理念が、事業会社レベルで「飲料事業」という文脈に**特殊化**され、さらにカルピスブランドでは「乳酸菌」「健康志向層」という文脈に**セグメント化**されています。

この継承プロセスにより、カルピスが「心身の健やかさ」「世代を超えた記憶」という価値を担う**必然性**が説明可能になります。

### 継承構造の詳細

```
アサヒグループHD
│
│  「期待を超えるおいしさ、楽しい生活文化の創造」
│
│  HV-01: 期待を超えるおいしさ
│  HV-02: 楽しい生活文化の創造
│  HV-03: 健康への貢献
│
└───┬─────────────────────────────────────────
    │ 継承タイプ: 特殊化（飲料事業へ）
    ▼
アサヒ飲料
│
│  OV-01: 飲料でのおいしさ ← HV-01
│  OV-02: 生活シーンの豊かさ ← HV-02
│  OV-03: 飲料による健康 ← HV-03
│
└───┬─────────────────────────────────────────
    │ 継承タイプ: セグメント化（乳酸菌・健康志向層へ）
    ▼
カルピス
│
│  「カラダにピース。」
│
│  BV-01: 乳酸菌のおいしさ ← OV-01
│  BV-02: 世代を超えた幸福な記憶 ← OV-02
│  BV-03: 乳酸菌による心身の健やかさ ← OV-03
│
└───┬─────────────────────────────────────────
    ▼
価値定義（VL1-VL3）へ展開
```

### 継承タイプの意味

| 継承タイプ | 意味 | 本プロジェクトでの適用 |
|-----------|------|----------------------|
| **特殊化** | 上位価値を特定の事業領域に適用 | HD「おいしさ」→ 事業会社「飲料でのおいしさ」 |
| **セグメント化** | 上位価値を特定の顧客層・技術領域に絞り込み | 事業会社「飲料健康」→ Brand「乳酸菌による健やかさ」 |

### この分析の意義

従来「カルピスは健康と家族の絆を提供する」と定義しても、「なぜカルピスがそれを担うのか」は説明できませんでした。本分析により、以下が明確になります：

- **HV-03（健康への貢献）**がOV-03（飲料による健康）を経て、BV-03（乳酸菌による健やかさ）に継承される構造により、カルピスが「健康価値」を担う**組織的正当性**が証明される
- **HV-02（生活文化の創造）**がOV-02（生活シーンの豊かさ）を経て、BV-02（世代を超えた記憶）に継承される構造により、カルピスが「家族の絆」を担う**戦略的必然性**が説明される

## カルピスの価値体系

### 導出された本質的価値

継承分析の結果、カルピスブランドの本質的価値（VL1）として以下の2つが導出されました。これらは「カルピスが存在する理由」を定義するものであり、上位組織からの継承によってその正当性が担保されています。

### VL1: 本質的価値（なぜ存在するか）

| ID | 価値 | 継承元 | 継承の論理 |
|----|------|--------|-----------|
| VL1-1 | 心身の健やかさを支える | BV-03 | HD「健康への貢献」→事業会社「飲料健康」→Brand「乳酸菌健康」の継承により、100年以上の乳酸菌研究に基づく健康価値を担う |
| VL1-2 | 世代を超えた幸福な記憶を紡ぐ | BV-02 | HD「生活文化創造」→事業会社「生活シーン」→Brand「世代記憶」の継承により、家族の絆と文化継承を担う |

### VL2-VL3: 中核価値→詳細価値

本質的価値（VL1）は、中核価値（VL2）、詳細価値（VL3）へと具体化されます。この階層構造により、抽象的な価値が実行可能なレベルまで分解されます。

```
VL1-1: 心身の健やかさ
├── VL2-1-1: 乳酸菌による健康価値
│   ├── VL3: カルピス菌発酵
│   ├── VL3: 腸内環境への好影響
│   └── VL3: 機能性成分
└── VL2-1-2: 安心・信頼の品質
    ├── VL3: 厳格な品質管理
    └── VL3: 100年ブランドの信頼

VL1-2: 世代を超えた記憶
├── VL2-2-1: 懐かしさと新しさの融合
│   ├── VL3: 伝統の甘酸っぱい味わい
│   └── VL3: 時代に合わせた商品展開
└── VL2-2-2: 家族のコミュニケーション触媒
    ├── VL3: 一緒に作る・割る体験
    └── VL3: 季節・イベントとの結びつき
```

## VStream（価値実現の顧客体験）

### VStreamアーキテクチャの導出

価値定義が完了した後、次の課題は「その価値をどのように顧客に届けるか」という顧客体験の設計です。従来のアプローチでは、VStream（顧客体験の流れ）を顧客セグメント別に複数設計することが一般的でした。しかし、この方法では「なぜその本数なのか」「なぜその構成なのか」を説明することが困難でした。

V5.7では、**価値継承構造を基盤としてVStreamアーキテクチャを導出**するという原則を確立します。これにより、VStreamの本数と構成に明確な根拠が生まれます。

### V5.7 核心原則

**価値継承構造 → VStream構成**

V5.7では、**価値定義の継承構造を基盤として、VStreamも構造的整合性を持って構成される**という原則を確立します。

**重要**: VStreamの本数は固定ではなく、**組織の価値継承構造から導出**されます。本プロジェクトでは、HD→事業会社→Brandの3階層構造から、3本のVStreamが導出されました。

| 継承構造の例 | VStream構成 | 詳細度 |
|-------------|------------|--------|
| HD のみ | HD VStream（1本） | 詳細 |
| HD → 事業会社 | HD + 事業会社 VStream（2本） | 上位概要、ターゲット詳細 |
| **HD → 事業会社 → Brand** | **HD + 事業会社 + Brand VStream（3本）** | **上位概要、ターゲット詳細** |
| HD → 事業会社 → Brand → Product | 4本 | 上位概要、ターゲット詳細 |

## 価値統合アーキテクチャ（Value Integration Architecture）

### アーキテクチャ導出の考え方

ここまでの分析で、価値継承構造からVStreamアーキテクチャを導出しました。V5.7の最も重要な特徴は、この導出プロセスがVStreamで終わらず、**Capabilityアーキテクチャ**、さらには**アプリケーションアーキテクチャ（マイクロサービス境界）**まで一貫して適用されることです。

これにより、「なぜこのシステム構成なのか」「なぜこの機能は統合し、この機能は分離するのか」という問いに対して、**価値まで遡って説明**することが可能になります。

### 全体構造

## L1ケイパビリティアーキテクチャ

### Capabilityアーキテクチャ導出の意義

VStreamアーキテクチャが「顧客にどのように価値を届けるか」を定義するのに対し、Capabilityアーキテクチャは「価値を実現するために組織が持つべき能力」を定義します。

V5.7では、価値継承構造を基盤としてCapabilityアーキテクチャを導出します。これにより、「どのCapabilityをグループで共有すべきか」「どのCapabilityをBrand固有とすべきか」という判断に明確な根拠が生まれます。

### V5.7原則の適用

**価値継承構造 → Capability構成**

価値の継承構造を基盤として、ケイパビリティも3層で構造的整合性を持って導出されます。共有すべきCapabilityは価値が継承されている領域、固有とすべきCapabilityはBrand独自価値を担う領域として判断されます。

## 結論

### 本プロジェクトの成果

本プロジェクトでは、Parasol V5.7の手法を用いて、カルピスブランドの価値設計をアサヒグループ全体の企業価値から構造的に導出することに成功しました。

**主要な発見**:

1. **価値の構造的導出**: カルピスの価値「心身の健やかさ」「世代を超えた記憶」は、アサヒグループHDの企業理念「期待を超えるおいしさ、楽しい生活文化の創造、健康への貢献」から、特殊化・セグメント化という継承プロセスを経て論理的に導出されることが証明されました。

2. **アーキテクチャの一貫性**: 価値継承構造を基盤として、VStreamアーキテクチャ（3本）、Capabilityアーキテクチャ（共有/固有の判断）、アプリケーションアーキテクチャ（統合/分離の判断）が一貫して導出されました。

3. **説明可能性の確保**: 「なぜカルピスがこの価値を担うのか」「なぜこのシステム構成なのか」という問いに対して、価値まで遡って構造的に説明することが可能になりました。

### V5.7アプローチの価値

| 観点 | 従来の課題 | V5.7による解決 |
|------|-----------|---------------|
| **戦略整合性** | ブランド戦略とグループ戦略の整合性を個別に確認する必要があった | 価値継承構造により**自動的に整合性が担保**される |
| **設計根拠** | VStream本数、Capability共有/固有、システム統合/分離の判断根拠が曖昧だった | 全ての設計判断が**価値から一貫して導出**される |
| **オンボーディング** | 新規メンバーが「なぜこうなっているのか」を理解するのが困難だった | 価値継承チェーンにより**構造的に理解**可能 |

### 推奨事項

本プロジェクトの成果を踏まえ、以下を推奨します：

1. **他ブランドへの展開**: 同様の手法を三ツ矢、ワンダ等の他ブランドに適用し、グループ全体の価値アーキテクチャを構築する
2. **システム設計への適用**: 導出されたCapabilityアーキテクチャを基に、具体的なシステム設計（マイクロサービス境界、API設計）を進める
3. **継続的な整合性確認**: 新規施策・新規システム開発時に、価値継承構造との整合性を確認するプロセスを確立する
