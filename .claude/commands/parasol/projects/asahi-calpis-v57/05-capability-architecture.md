## V5.7原則の適用

### 核心原則: 価値継承構造 = VStream構成 = Capability構成

V5.7では、**価値定義の継承構造を基盤として、Capabilityも構造的整合性を持って構成される**という原則を適用します。

```
【V5.7 Capability構成原則】

  価値継承構造              Capability構成
  ─────────────────────     ─────────────────────
  HD価値（HV-01〜03）   →   HD共有Capability（グループ全体）
        ↓ 継承
  事業会社価値（OV-01〜03） →   事業会社共有Capability（飲料事業）
        ↓ 継承
  Brand価値（BV-01〜03）→   Brand固有Capability（カルピス）

→ 結果として3層のCapability構造（継承・共有・固有）
```

### 5.2 共有可能性分析

```
【高共有】グループ全体で共有すべき
─────────────────────────────────
HC-04 グループSCM         → 効率化のため統合
HC-05 顧客データ基盤      → 活用は個別、基盤は共有

【中共有】事業会社レベルで共有すべき
─────────────────────────────────

OC-04 飲料SCM             → 飲料特有の配送・季節対応

【低共有】Brand固有を維持すべき
─────────────────────────────────

BC-02 ブランド体験設計    → 100年ブランドの独自性

【完全固有】他と共有不可
─────────────────────────────────
```

## 7. 統合・分離判断への示唆

### 7.1 システム統合の観点

| 判断項目 | Capability | 判断 | 理由 |
|---------|------------|------|------|
| 基幹系統合 | HC-04, OC-04 | **統合** | 効率化、価値継承あり |
| 品質管理系 | HC-01→OC-01 | **統合**（基盤共有+拡張） | 品質は継承価値 |
| 顧客データ | HC-05 | **統合**（基盤）| 活用は個別 |
| R&D系 | HC-03→BC-03 | **部分統合** | 基盤共有、研究は分離 |
| ブランド系 | BC-02, BC-04, BC-05 | **分離** | 固有価値、差別化源泉 |

### 7.2 M&A/グループ再編への示唆

カルピスブランドを独立・売却する場合に失われる/移転すべきCapability：

| Capability | 移転難易度 | 理由 |
|------------|-----------|------|
| BC-01 乳酸菌発酵技術 | **高** | 100年の技術蓄積、暗黙知多い |
| BC-02 ブランド体験設計 | **中** | 文書化可能だが歴史が重要 |
| BC-03 乳酸菌研究 | **高** | HC-03との連携が価値を生む |
| BC-04 世代間コミュニケーション | **中** | 独自だが再構築可能 |
| BC-05 希釈体験設計 | **低** | 比較的移転容易 |

**結論**: カルピスはグループのR&D基盤（HC-03）との連携により乳酸菌研究（BC-03）の価値が高まっているため、分離すると価値毀損リスクが高い。
