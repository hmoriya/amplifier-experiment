# explore.md 更新計画

## 現状の問題

`explore.md`は古い3軸定義（価値/事業部/ハイブリッド）を使用しており、新しい軸定義体系と不整合。

## 新しい軸定義体系

### 利用可能な軸（6種類）

1. **capability-axis** - ケイパビリティ軸
   - ビジネス能力中心の分解
   - 機能横断的な価値創出

2. **business-unit-axis** - 事業部軸
   - 組織構造に基づく分解
   - 事業部別の価値定義

3. **value-axis** - 価値軸
   - 純粋な価値種別での分解
   - 組織横断的な価値体系

4. **platform-axis** - プラットフォーム軸
   - 共通基盤 + 事業別サービス
   - プラットフォーム思考

5. **fusion-axis** - 融合軸
   - ハイブリッドアプローチ
   - 複数視点の統合

6. **multi-tier-vstr-axis** - 多層VST-R軸
   - 階層的なValue Stream
   - 段階的な価値実現

### 業界別推奨

- **飲料・食品業界**: プラットフォーム軸 or 融合軸
- **地域銀行**: 価値軸 or プラットフォーム軸
- **人材派遣業**: ケイパビリティ軸 or プラットフォーム軸

## 更新方針

### 1. 軸選択の動的化

```bash
# 現在（固定3軸）
/parasol:explore run value
/parasol:explore run business
/parasol:explore run hybrid

# 新方式（動的選択）
/parasol:explore init          # 業界に基づく推奨軸を提示
/parasol:explore select-axes   # 探索する軸を3つ選択
/parasol:explore run [axis]    # 選択した軸で探索
```

### 2. 業界別ガイダンス統合

```
1. プロジェクト初期化時に業界を特定
2. axis-selection-guide.mdから推奨軸を取得
3. 推奨に基づいて3軸を選択（カスタマイズ可能）
```

### 3. 軸定義ファイル参照

各軸の詳細は`_patterns/_axes/*.md`から動的に読み込み：
- 分解観点
- テンプレート
- 出力構造

## 実装アプローチ

### Phase 1: 既存機能の保持
- 現在の3軸（value/business/hybrid）を維持
- 新しい軸選択機能を追加

### Phase 2: 動的軸選択
- `select-axes`サブコマンドの追加
- 業界別推奨の自動提示

### Phase 3: 完全移行
- 旧3軸の廃止
- 新6軸体系への完全移行

## 影響範囲

1. **explore.md** - コマンド定義の更新
2. **各軸テンプレート** - 新軸に対応したテンプレート作成
3. **比較ロジック** - 異なる軸の組み合わせに対応

2. 動的な軸選択インターフェース
