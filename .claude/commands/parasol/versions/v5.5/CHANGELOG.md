# Parasol V5.5 リリースノート

**リリース日**: 2025-01-07
**バージョン**: V5.5
**タイプ**: Major Feature Release

---

## 🎯 概要

Parasol V5.5では、**TVDC（Three-Dimensional Value-Driven Classification）4分類フレームワーク**を導入し、価値駆動型ドメイン分類の精度を大幅に向上させました。従来の3分類（Core/Supporting/Generic）に**VCI（Value-Critical Infrastructure）**を追加し、「差別化には直接貢献しないが、価値提供に不可欠なケイパビリティ」を明確に識別できるようになりました。

---

## ✨ 主な新機能

### 1. TVDC 4分類フレームワーク

#### 1.1 新分類体系

| 分類 | 差別化度 | 価値必然性 | 特徴 |
|------|----------|------------|------|
| **Core** | ★★★ | ★★★ | 戦略的投資対象、競争優位の源泉 |
| **VCI** | ★ | ★★★ | 価値提供に不可欠、高品質が必須 |
| **Supporting** | ★★ | ★★ | 品質向上に貢献、内部効率化 |
| **Generic** | ★ | ★ | 標準化・外注可能、コスト最適化 |

#### 1.2 VCI（Value-Critical Infrastructure）の導入

**定義**: 差別化度は低いが、価値提供に不可欠なケイパビリティ

**典型例**:
- 決済処理（決済自体は差別化要因ではないが、失敗は価値喪失に直結）
- 認証基盤（認証方式で差別化しないが、セキュリティ障害は致命的）
- データ配信基盤（配信方法で競争しないが、遅延は顧客体験を破壊）

**判断基準**:
- 差別化度: ★（業界標準、外部調達可能）
- 価値必然性: ★★★（失敗時の価値喪失が甚大）

### 2. Phase 2-3 価値継承フロー

#### 2.1 VL3価値必然性評価（Phase 2）

```yaml
# Phase 2: VL3（価値要素）定義時に評価
vl3_with_necessity:
  id: VL3-P-1
  name: "安心・信頼の決済"
  value_necessity: "★★★"  # 失敗時の価値喪失度
  reasoning: "決済失敗は購買体験を完全に破壊する"
```

#### 2.2 TVDC分類継承（Phase 3）

```yaml
# Phase 3: CL2定義時にVL3価値必然性を継承
cl2_classification:
  id: VS4-CL2-3
  name: "決済処理"
  differentiation: "★"      # 差別化度（Phase 3で評価）
  value_necessity: "★★★"   # VL3から継承
  tvdc_class: "VCI"         # 2軸から自動導出
```

## 💡 移行ガイド

### V5.4からV5.5への移行

#### 1. 既存プロジェクトの再評価

既存の3分類（Core/Supporting/Generic）を見直し、VCI候補を特定:

```
チェックリスト:
□ Supportingに分類されているが、障害時の影響が甚大なもの
□ Genericに分類されているが、品質要件が高いもの
□ 「差別化しないが重要」と感じるケイパビリティ
```

#### 2. VCI候補の判定

```yaml
判定基準:
  差別化度: ★（業界標準技術で実現可能）
  価値必然性: ★★★（障害時に顧客価値が大幅に毀損）
  → 両方満たせばVCI
```

#### 3. 設計方針の調整

| 分類 | V5.4の方針 | V5.5の方針 |
|------|------------|------------|
| VCI（旧Supporting/Generic） | コスト最適化 | **信頼性優先** |

## 🎯 設計への影響

### VCIの設計指針

```yaml
VCI設計原則:
  技術選定: 標準技術・実績ある外部サービス優先
  品質要件: 高可用性（99.9%+）、低レイテンシ
  監視: 詳細なSLI/SLO設定、アラート即時対応
  障害対応: 迅速なフォールバック、グレースフルデグラデーション
  投資判断: 差別化投資より信頼性投資を優先
```

### 分類別投資配分ガイド

| 分類 | 投資の方向性 |
|------|--------------|
| Core | 差別化機能開発、競争優位強化 |
| VCI | 信頼性向上、SLA達成、障害対策 |
| Supporting | 効率化、自動化、品質維持 |
| Generic | コスト最適化、標準化、外注検討 |
