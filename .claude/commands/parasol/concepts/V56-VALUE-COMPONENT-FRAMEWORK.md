## 1. 背景：WHATの二面性

### 1.1 5W1H文書での発見

5W1Hフレームワークにおいて、WHATには2つの側面があることが明確化された：

```
WHAT = 問題領域
│
├── 価値構成要素（外向きWHAT）
│     「顧客に何を提供するか」
│     → 顧客視点・価値提案
│
└── ケイパビリティ（内向きWHAT）
      「自社に何ができるか」
      → 組織視点・能力定義
```

### 1.2 V5現行との比較

| 観点 | V5現行 | V5.6提案 |
|------|--------|---------|
| WHATの定義 | ケイパビリティ中心 | **二面性を明示** |
| 顧客視点 | VL（Value Level）で表現 | VL + **Value Component** |
| 対応関係 | 暗黙的 | **明示的マッピング** |

## 2. Value Component（価値構成要素）の定義

### 2.1 定義

**Value Component（価値構成要素）**: 顧客に提供する価値を構成する要素。顧客視点で「何を受け取るか」を記述する。

### 2.2 5つの基本要素

| 要素 | 英語 | 定義 | 顧客の問い |
|------|------|------|-----------|
| **製品価値** | Product Value | 製品そのものが提供する価値 | 「何を買うのか」 |
| **サービス価値** | Service Value | 利便性・アクセス性の価値 | 「どう手に入るか」 |
| **体験価値** | Experience Value | 購入・使用時の体験価値 | 「どう感じるか」 |
| **関係価値** | Relationship Value | ブランドとの関係性価値 | 「なぜ選び続けるか」 |
| **情報価値** | Information Value | 提供される情報の価値 | 「何を知れるか」 |

### 2.3 Value Componentの構造

```
Value Component（価値構成要素）
│
├── VC1: 製品価値 (Product Value)
│   ├── VC1-1: 機能的価値（性能、品質、機能）
│   ├── VC1-2: 品質価値（安全、信頼性、耐久性）
│   └── VC1-3: 多様性価値（選択肢、カスタマイズ）
│
├── VC2: サービス価値 (Service Value)
│   ├── VC2-1: アクセス価値（入手しやすさ、チャネル）
│   ├── VC2-2: 価格価値（価格妥当性、コスパ）
│   └── VC2-3: 安定価値（安定供給、可用性）
│
├── VC3: 体験価値 (Experience Value)
│   ├── VC3-1: 購入体験（購買プロセス、接客）
│   ├── VC3-2: 使用体験（使い心地、感動）
│   └── VC3-3: 所有体験（デザイン、満足感）
│
├── VC4: 関係価値 (Relationship Value)
│   ├── VC4-1: 信頼価値（ブランド信頼、安心感）
│   ├── VC4-2: 帰属価値（コミュニティ、所属感）
│   └── VC4-3: 継続価値（ロイヤリティ、習慣）
│
└── VC5: 情報価値 (Information Value)
    ├── VC5-1: 知識価値（製品知識、使い方）
    ├── VC5-2: 発見価値（新商品、トレンド）
    └── VC5-3: 共有価値（口コミ、推奨）
```

## 3. Value ComponentとValue Level (VL)の関係

### 3.1 概念の整理

| 概念 | 定義 | 抽象度 | 用途 |
|------|------|--------|------|
| **VL (Value Level)** | 提供価値の階層的定義 | 高〜低 | 投資判断、優先順位 |
| **VC (Value Component)** | 価値の構成要素分類 | 中 | 価値提案設計、ギャップ分析 |

### 3.2 VLとVCの関係図

```
┌─────────────────────────────────────────────────────────────────────┐
│ VL (Value Level) - 価値の階層分解                                  │
│                                                                     │
│ VL1: 最上位価値                                                    │
│   └─ VL2: 価値グループ                                            │
│        └─ VL3: 詳細価値                                           │
│              │                                                      │
│              │ 「この価値はどの要素で構成されるか？」               │
│              ▼                                                      │
└─────────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────────┐
│ VC (Value Component) - 価値の構成要素                              │
│                                                                     │
│ VL3の各詳細価値は、以下の要素で構成される:                         │
│                                                                     │
│ ┌─────────┬─────────┬─────────┬─────────┬─────────┐                │
│ │ 製品    │ サービス│  体験   │  関係   │  情報   │                │
│ │ (VC1)   │ (VC2)   │ (VC3)   │ (VC4)   │ (VC5)   │                │
│ └─────────┴─────────┴─────────┴─────────┴─────────┘                │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.3 VL-VC マッピング例（飲料メーカー）

```
VL1: 期待を超えるおいしさで、心身の潤いを届ける
│
├── VL2-1: 製品イノベーション価値
│   │
│   └── VL3-1-1: 発酵技術による味の革新
│           │
│           └── VC構成:
│               ├── VC1: 製品価値 ★★★（本格的な味、独自技術）
│               ├── VC2: サービス価値 ★（標準的な流通）
│               ├── VC3: 体験価値 ★★（飲む瞬間の感動）
│               ├── VC4: 関係価値 ★（ブランド認知向上）
│               └── VC5: 情報価値 ★（技術の訴求）
│
├── VL2-2: 入手容易性価値
│   │
│   └── VL3-2-1: いつでもどこでも手に入る
│           │
│           └── VC構成:
│               ├── VC1: 製品価値 ★（製品は同じ）
│               ├── VC2: サービス価値 ★★★（チャネル、在庫）
│               ├── VC3: 体験価値 ★★（購入の手軽さ）
│               ├── VC4: 関係価値 ★（習慣化）
│               └── VC5: 情報価値 ★（店頭情報）
│
└── VL2-3: ブランド体験価値
    │
    └── VL3-3-1: 愛着のあるブランドとの関係
            │
            └── VC構成:
                ├── VC1: 製品価値 ★（製品は手段）
                ├── VC2: サービス価値 ★（標準的）
                ├── VC3: 体験価値 ★★★（ブランド体験）
                ├── VC4: 関係価値 ★★★（愛着、信頼）
                └── VC5: 情報価値 ★★（ブランドストーリー）
```

## 4. Value Componentとケイパビリティの対応

### 4.1 対応モデル

```
┌─────────────────────────────────────────────────────────────────────┐
│ 顧客視点                     │ 組織視点                            │
│ (External WHAT)             │ (Internal WHAT)                     │
├─────────────────────────────┼────────────────────────────────────┤
│                              │                                     │
│ Value Component             │ Capability                          │
│ 「顧客に何を提供するか」     │ 「自社に何ができるか」              │
│                              │                                     │
│ ┌─────────────────────┐     │ ┌─────────────────────────────┐    │
│ │ VC1: 製品価値       │←───→│ │ 製品系Cap: レシピ開発、製造  │    │
│ │ VC2: サービス価値   │←───→│ │ 供給系Cap: チャネル、在庫    │    │
│ │ VC3: 体験価値       │←───→│ │ 体験系Cap: 店頭設計、UX設計  │    │
│ │ VC4: 関係価値       │←───→│ │ 関係系Cap: ブランド、CRM     │    │
│ │ VC5: 情報価値       │←───→│ │ 情報系Cap: マーケ、コンテンツ│    │
│ └─────────────────────┘     │ └─────────────────────────────┘    │
└─────────────────────────────┴────────────────────────────────────┘
```

### 4.2 対応分析の活用

**ギャップ分析**: VC要求とCapability充足度のギャップを識別

```
例: VL3-1-1「発酵技術による味の革新」

VC要求                Cap充足度        ギャップ
────────────────────────────────────────────────────
VC1: 製品価値 ★★★    → Cap充足 ★★★   なし
VC2: サービス価値 ★   → Cap充足 ★★    過剰（効率化余地）
VC3: 体験価値 ★★      → Cap充足 ★     【ギャップ】体験設計強化必要
VC4: 関係価値 ★       → Cap充足 ★     なし
VC5: 情報価値 ★       → Cap充足 ★     なし

→ 体験価値（VC3）に対応するCapabilityの強化が必要
```

## 5. Value Stage (VS) との関係

### 5.1 VS × VC マトリクス

各VSで重要なVCは異なる：

```
           │  VC1    │  VC2    │  VC3    │  VC4    │  VC5   │
           │ 製品    │サービス │  体験   │  関係   │  情報  │
───────────┼─────────┼─────────┼─────────┼─────────┼────────┤
VS0→VS1   │    ★    │    ★    │   ★★   │    ★    │  ★★★  │
(認知)     │         │         │         │         │        │
───────────┼─────────┼─────────┼─────────┼─────────┼────────┤
VS1→VS2   │  ★★★   │   ★★   │  ★★★   │   ★★   │  ★★★  │
(選好)     │         │         │         │         │        │
───────────┼─────────┼─────────┼─────────┼─────────┼────────┤
VS2→VS3   │  ★★★   │  ★★★   │  ★★★   │    ★    │   ★★  │
(初回)     │         │         │         │         │        │
───────────┼─────────┼─────────┼─────────┼─────────┼────────┤
VS3→VS4   │  ★★★   │  ★★★   │   ★★   │   ★★   │    ★   │
(継続)     │         │         │         │         │        │
───────────┼─────────┼─────────┼─────────┼─────────┼────────┤
VS4→VS5   │   ★★   │   ★★   │   ★★   │  ★★★   │    ★   │
(定着)     │         │         │         │         │        │
───────────┼─────────┼─────────┼─────────┼─────────┼────────┤
VS5→VS6   │   ★★   │    ★    │   ★★   │  ★★★   │   ★★  │
(発展)     │         │         │         │         │        │
───────────┼─────────┼─────────┼─────────┼─────────┼────────┤
VS6→VS7   │    ★    │    ★    │  ★★★   │  ★★★   │  ★★★  │
(推奨)     │         │         │         │         │        │
───────────┴─────────┴─────────┴─────────┴─────────┴────────┘
```

### 5.2 VS-VC分析の活用

**VS別VC投資判断**:
- 初期段階（VS0-VS2）: 情報価値・体験価値重視
- 獲得段階（VS2-VS4）: 製品価値・サービス価値重視
- 定着段階（VS4-VS7）: 関係価値・体験価値重視

## 6. V5.6への統合提案

### 6.1 Phase構成への追加

```
Phase 2: 価値定義
├── Step 2.1: VL定義（既存）
│   └── VL1 → VL2 → VL3 の階層分解
│
├── Step 2.2: VC分析（V5.6追加）
│   └── 各VL3のVC構成を分析
│   └── VC別重要度を設定
│
├── Step 2.3: VS設計（既存）
│   └── VS0-VS7の顧客状態定義
│
└── Step 2.4: VS-VC対応（V5.6追加）
    └── 各VSで重要なVCを特定
    └── 投資優先度の導出
```

### 6.2 テンプレート

```yaml
# VL3 価値要素定義（V5.6版）

value_element:
  id: "VL3-X-Y"
  name: "{価値要素名}"
  parent: "VL2-X"

  # Value Component構成（V5.6追加）
  value_components:
    VC1_product:
      importance: "{★ | ★★ | ★★★}"
      description: "{製品価値の説明}"
    VC2_service:
      importance: "{★ | ★★ | ★★★}"
      description: "{サービス価値の説明}"
    VC3_experience:
      importance: "{★ | ★★ | ★★★}"
      description: "{体験価値の説明}"
    VC4_relationship:
      importance: "{★ | ★★ | ★★★}"
      description: "{関係価値の説明}"
    VC5_information:
      importance: "{★ | ★★ | ★★★}"
      description: "{情報価値の説明}"

  # VS別VC重要度（オプション）
  vs_vc_priority:
    VS0_VS1: ["VC5", "VC3"]  # 認知段階で重要なVC
    VS2_VS3: ["VC1", "VC2"]  # 初回段階で重要なVC
    VS4_VS7: ["VC4", "VC3"]  # 定着段階で重要なVC

  # Phase 3への引継ぎ
  capability_mapping:
    VC1: ["{対応Cap1}", "{対応Cap2}"]
    VC2: ["{対応Cap3}"]
    VC3: ["{対応Cap4}"]
    VC4: ["{対応Cap5}"]
    VC5: ["{対応Cap6}"]
```

## 7. 価値分解の方向性まとめ

### 7.1 V5.6で整理した価値分解の軸

```
┌─────────────────────────────────────────────────────────────────────┐
│                    価値分解の多軸構造                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  WHY（なぜ）                                                        │
│  └── VL1 → VL2 → VL3（階層分解）                                   │
│                                                                     │
│  分解軸1: ステークホルダー軸                                        │
│  └── 顧客/株主/従業員/パートナー/社会                              │
│                                                                     │
│  分解軸2: 時間軸                                                    │
│  └── 現在価値/将来価値/持続価値                                    │
│                                                                     │
│  分解軸3: 価値性質軸                                                │
│  └── 機能的/情緒的/社会的                                          │
│                                                                     │
│  WHAT（何を）= 二面性                                               │
│  ├── 外向きWHAT: Value Component（VC1-VC5）                        │
│  │   └── 製品/サービス/体験/関係/情報                              │
│  │                                                                  │
│  └── 内向きWHAT: Capability                                        │
│      └── TVDC分類: Core/VCI/Supporting/Generic                     │
│                                                                     │
│  WHEN（いつ）                                                       │
│  ├── VS (Value Stage): VS0-VS7（顧客状態）                         │
│  └── OS (Operational Stage): 運用段階（組織活動）                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 7.2 統合フレームワーク

```
【V5.6 価値駆動設計フレームワーク】

Phase 1: コンテキスト
└── 事業環境、制約条件

Phase 2: 価値定義
├── VL分解（WHY）: VL1→VL2→VL3
├── VC分析（外向きWHAT）: 5要素の構成
├── VS設計（WHEN）: VS0-VS7
└── 価値分解軸の適用
    ├── ステークホルダー軸
    ├── 時間軸
    └── 性質軸

Phase 3: ケイパビリティ
├── Cap分解（内向きWHAT）: CL1→CL2→CL3
├── TVDC分類
├── VC-Cap対応分析
└── OS（運用段階）定義

Phase 4-7: 設計・実装
└── （既存フェーズ）
```
