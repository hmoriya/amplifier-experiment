## 1. TVDC分類の復習

### 4分類定義

| 分類 | 定義 | 価値必然性 | 差別化 | 投資優先度 |
|------|------|-----------|--------|-----------|
| **Core** | 価値必然 + 差別化 | ★★★ | ★★★ | 最高 |
| **VCI** | 価値必然だが差別化にならない | ★★★ | ★ | 高（品質重視） |
| **Supporting** | 重要な支援機能 | ★★ | ★～★★ | 中 |
| **Generic** | 汎用基盤機能 | ★ | ★ | 低（SaaS活用） |

## 5. 価値構成要素とTVDCの対応

5W1H文書の「価値構成要素（外向きWHAT）」と「ケイパビリティ（内向きWHAT）」の対応にTVDCを追加。

### 5.1 対応マトリクス

```
価値構成要素（外向き）    ケイパビリティ（内向き）   TVDC
───────────────────────────────────────────────────────────────

製品価値                  製品系ケイパビリティ
├── 本格的な味            ├── レシピ開発             Core
├── 品質・安全            ├── 品質管理               VCI
└── バリエーション        └── 製造                   VCI

サービス価値              供給系ケイパビリティ
├── 入手しやすさ          ├── チャネル管理           Core/Supporting
├── 価格の手頃さ          ├── 在庫管理               VCI
└── 安定供給              └── 配送管理               VCI

体験価値                  体験設計系ケイパビリティ
├── 購入時の楽しさ        ├── 店頭体験設計           Core
└── 飲む瞬間              └── パッケージ設計         Core

関係価値                  関係構築系ケイパビリティ
├── ブランド愛着          ├── ブランド管理           Core
└── 習慣・安心感          └── 顧客管理               Supporting

情報価値                  情報提供系ケイパビリティ
├── 新商品情報            ├── マーケティング         Supporting
└── キャンペーン          └── 販促管理               Supporting
```

### 5.2 TVDC分類の根拠パターン

| 価値構成要素 | 対応Cap傾向 | 根拠 |
|------------|------------|------|
| **製品価値** | Core/VCI | 製品そのものの価値は差別化の核心（Core）、品質保証は必然（VCI） |
| **サービス価値** | VCI/Supporting | 入手可能性は必然（VCI）、効率化は支援（Supporting） |
| **体験価値** | Core | 体験設計は差別化の大きな要因 |
| **関係価値** | Core/Supporting | ブランド関係は差別化、運用は支援 |
| **情報価値** | Supporting | 価値提供の補助的役割 |

## 6. ステージ依存TVDC

### 6.1 ステージによって分類が変わる例

```
能力: 商品情報提供

VS0→VS1（認知形成）: Core
  └─ ブランド認知に直結、訴求力が差別化

VS1→VS2（選好形成）: Core
  └─ 購買決定に直接影響

VS3→VS4（リピート促進）: Supporting
  └─ 使用方法の補助、差別化要因小

VS5以降: Generic
  └─ 基本的な参照機能
```

### 6.2 ステージ依存TVDC記述例

```yaml
capability:
  name: product-information
  display_name: 商品情報提供

  tvdc:
    default: Supporting

    stage_overrides:
      VS0-VS1: Core    # 認知形成段階では差別化要因
      VS1-VS2: Core    # 選好形成段階では購買決定に直結
      VS5+: Generic    # 定着後は基本機能

  rationale: |
    顧客ジャーニーの初期段階（認知・選好形成）では
    商品情報の質が購買決定に直結するためCore。
    定着後は参照頻度が下がり、標準機能で十分。
```

## 8. 結論
