# V5.6検討：価値分解の方向性（Value Decomposition Directions）

> **ステータス**: V5.6検討中
> **作成日**: 2026-01-13
> **関連文書**: V56-BIZBOK-5W1H-FRAMEWORK.md, value-methodology.md

---

## 1. 問題提起

### 1.1 現行V5の価値分解

現行V5では、価値分解は以下の構造で行われる：

```
WHY軸: Value Level（VL）
│
VL1: 最上位価値（経営レベル）
├── VL2: 価値グループ（部門レベル）
│     └── VL3: 詳細価値（チームレベル）
│
└── HOW軸: Value Stream（VStr）
    └── WHEN軸: Value Stage（VS0-VS7）
```

**課題**: この構造は「顧客への価値」を中心に設計されているが、以下の観点が不足：

1. **ステークホルダーの多様性**: 顧客以外（株主、従業員、社会）への価値が後付け
2. **時間軸の明示**: 現在価値・将来価値・持続価値の区別がない
3. **価値の性質**: 機能的・情緒的・社会的価値の区分がない

### 1.2 新提案のアプローチ

WHY（価値）からWHAT（何を）への分解に**3つの軸**を導入：

```
┌─────────────────────────────────────────────────────────────┐
│                    WHY → WHAT 分解の方向性                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                         WHY                                 │
│                      （価値）                               │
│                          │                                  │
│          ┌───────────────┼───────────────┐                 │
│          │               │               │                 │
│          ▼               ▼               ▼                 │
│    ┌──────────┐   ┌──────────┐   ┌──────────┐            │
│    │ステーク  │   │  時間軸  │   │ 価値の   │            │
│    │ホルダー軸│   │          │   │  性質軸  │            │
│    └──────────┘   └──────────┘   └──────────┘            │
│                                                             │
│    誰への価値か   いつの価値か   どんな価値か               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 2. 分解軸の詳細定義

### 2.1 軸1：ステークホルダー軸（誰への価値か）

```
WHY: 企業の存在意義・ミッション
│
▼
WHAT: ステークホルダー別価値
│
├── 顧客価値（Customer Value）
│     └── 製品、サービス、体験、関係、情報
│
├── 株主価値（Shareholder Value）
│     └── 収益、成長、配当、企業価値
│
├── 従業員価値（Employee Value）
│     └── 報酬、成長機会、働きがい、安全
│
├── パートナー価値（Partner Value）
│     └── 取引機会、成長支援、関係性
│
└── 社会価値（Social Value）
      └── 環境、地域、雇用、文化
```

**V5.6への統合提案**:
- VL1定義時にステークホルダー別の価値を明示的に定義
- 優先ステークホルダーを明確化（多くの場合は顧客）
- 他ステークホルダーへの価値は「制約条件」または「共益条件」として扱う

### 2.2 軸2：時間軸（いつの価値か）

```
WHY: 企業の存在意義
│
▼
WHAT: 時間軸別価値
│
├── 現在価値（Current Value）
│     └── 今、提供している価値
│     └── 現行製品・サービス
│     └── VS分類: VS0-VS7の全ステージ
│
├── 将来価値（Future Value）
│     └── これから提供する価値
│     └── 新規事業、イノベーション
│     └── VS分類: 主にVS0-VS2（価値創造フェーズ）
│
└── 持続価値（Sustainable Value）
      └── 長期的に維持すべき価値
      └── ブランド、信頼、関係性
      └── VS分類: 主にVS6-VS7（価値継続フェーズ）
```

**V5.6への統合提案**:
- VMS（Value Milestone System）に時間軸分類を追加
- 投資ポートフォリオとして現在/将来/持続のバランスを管理
- TVDC分類と連動（Core=将来価値創造、VCI=持続価値維持）

### 2.3 軸3：価値の性質軸（どんな価値か）

```
WHY: 企業の存在意義
│
▼
WHAT: 価値の性質別
│
├── 機能的価値（Functional Value）
│     └── 実用性、品質、性能、価格
│     └── 「何ができるか」「どう役立つか」
│
├── 情緒的価値（Emotional Value）
│     └── 体験、感動、安心、喜び
│     └── 「どう感じるか」「どんな気持ちになるか」
│
└── 社会的価値（Social Value）
      └── 所属、ステータス、共感、貢献
      └── 「社会にどう見られるか」「どう貢献するか」
```

**V5.6への統合提案**:
- VL3（詳細価値）定義時に性質分類を明示
- Core領域は情緒的・社会的価値の創造に注力
- Supporting/Generic領域は機能的価値の効率化に注力

## 3. 3軸の組み合わせ

### 3.1 価値の3次元マトリクス

```
┌─────────────────────────────────────────────────────────────┐
│                    価値の3次元マトリクス                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                     時間軸                                  │
│              現在 ─────────→ 将来                          │
│               │                                             │
│    ステーク   │   ┌─────────────────────────┐             │
│    ホルダー   │   │                         │             │
│    軸        │   │    機能的               │             │
│               │   │    情緒的    ← 性質軸   │             │
│    顧客       │   │    社会的               │             │
│      ↓       │   │                         │             │
│    社会       │   └─────────────────────────┘             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 飲料メーカーでの具体例

```
┌─────────────────────────────────────────────────────────────┐
│  サントリー「水と生きる」のWHAT分解                          │
├─────────────────────────────────────────────────────────────┤

WHY: 水と生きる（企業理念）

─────────────────────────────────────────────────────────────

【ステークホルダー × 時間軸】

              現在価値          将来価値          持続価値
─────────────────────────────────────────────────────────────
顧客        美味しい飲料      健康価値飲料      信頼・安心
            手軽に買える      パーソナライズ    ブランド愛着

            税金              地域活性化        文化貢献

【顧客価値 × 性質軸 × 時間軸】

              現在              将来
─────────────────────────────────────────────────────────────

            品質              パーソナライズ

            癒し              自己実現
```

## 4. ケイパビリティへの展開

### 4.1 価値方向性 × ケイパビリティ

価値の方向性ごとに必要なケイパビリティが異なる：

```
┌─────────────────────────────────────────────────────────────┐
│  価値方向 × ケイパビリティ                                  │
├─────────────────────────────────────────────────────────────┤

【顧客 × 現在価値】→ 既存事業の遂行
├── 製品開発      [Core]
├── 製造          [VCI]
├── 供給          [Supporting]
└── ブランド管理  [Core]

【顧客 × 将来価値】→ イノベーション
├── R&D・イノベーション      [Core] ← 現在と異なる
├── 市場探索                 [Core]
├── 新規事業開発             [Core]
└── デジタルケイパビリティ   [Supporting]

【社会 × 持続価値】→ サステナビリティ
├── 環境管理                 [VCI] ← 現在と異なる
├── 水源保全                 [VCI]
├── サステナビリティ推進     [Supporting]
└── ステークホルダー対話     [Supporting]

【従業員 × 将来価値】→ 人的資本
├── 人材開発                 [Supporting] ← 現在と異なる
├── キャリア管理             [Supporting]
├── 組織開発                 [Generic]
└── 文化醸成                 [Generic]

└─────────────────────────────────────────────────────────────┘
```

### 4.2 TVDC分類との連動

```
価値方向性           TVDC分類傾向        投資優先度
─────────────────────────────────────────────────────────────
顧客×現在×情緒的    → Core             最高（競争優位）
顧客×将来×機能的    → Core             高（将来投資）
顧客×現在×機能的    → VCI              中高（品質維持）
社会×持続×社会的    → VCI              中（必須維持）
従業員×将来         → Supporting       中（効率化）
全体×基盤          → Generic          低（SaaS活用）
```

## 5. 価値ポートフォリオとしての整理

### 5.1 投資配分の視点

```
┌─────────────────────────────────────────────────────────────┐
│                    価値ポートフォリオ                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│     投資・注力                                              │
│         ↑                                                   │
│         │     ┌───────────┐      ┌───────────┐           │
│     将来│     │ 顧客×将来 │      │ 社会×将来 │           │
│     価値│     │ 健康飲料  │      │ 環境対応  │           │
│         │     │ デジタル  │      │ 水源保全  │           │
│         │     └───────────┘      └───────────┘           │
│         │                                                   │
│         │     ┌───────────┐      ┌───────────┐           │
│     現在│     │ 顧客×現在 │      │ 株主×現在 │           │
│     価値│     │ 既存製品  │      │ 収益確保  │           │
│         │     │ 供給      │      │           │           │
│         │     └───────────┘      └───────────┘           │
│         │                                                   │
│         └───────────────────────────────────────→         │
│               維持                  成長                    │
│                       リソース配分                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 バランス判断基準

| 価値区分 | 投資配分目安 | 判断基準 |
|---------|-------------|---------|
| 顧客×現在 | 50-60% | 現在の収益基盤 |
| 顧客×将来 | 20-30% | 将来の成長機会 |
| 社会×持続 | 10-15% | 持続可能性確保 |
| 従業員×将来 | 5-10% | 人的資本投資 |

```
       ┌────────────┼────────────┐

       ▼            ▼            ▼
```

### 6.3 TVDC分類への連動

```
価値分解結果 → TVDC分類への入力

1. ステークホルダー軸
   - 顧客価値直結 → Core候補
   - 社会価値必須 → VCI候補
   - 内部価値     → Supporting/Generic候補

2. 時間軸
   - 将来価値創造 → Core（差別化投資）
   - 現在価値維持 → VCI（品質投資）
   - 持続価値維持 → Supporting（効率化）

3. 性質軸
   - 情緒的・社会的価値 → Core（差別化源泉）
   - 機能的価値         → VCI/Supporting（効率化対象）
```

## 7. 実践ガイドライン

### 7.1 価値分解ワークショップの進め方

```
Step 1: WHY（企業理念）の確認
        ├── Vision/Mission/Core Valuesの確認
        └── 変えないもの vs 深化するもの

Step 2: ステークホルダー軸の展開
        ├── 各ステークホルダーへの価値を列挙
        ├── 優先順位の決定
        └── トレードオフの明確化

Step 3: 時間軸の展開
        ├── 現在価値の棚卸し
        ├── 将来価値の構想
        └── 持続価値の特定

Step 4: 性質軸の展開
        ├── 機能的価値の明確化
        ├── 情緒的価値の探索
        └── 社会的価値の定義

Step 5: 3軸の統合
        ├── 価値ポートフォリオの作成
        ├── 投資配分の決定
        └── VL1-VL3への落とし込み
```

### 7.2 よくある落とし穴

| 落とし穴 | 症状 | 対策 |
|---------|------|------|
| 顧客偏重 | 他ステークホルダーが後付け | 最初から全ステークホルダーを列挙 |
| 現在偏重 | 将来価値が曖昧 | 時間軸を明示的に分離 |
| 機能偏重 | 情緒的価値が抜け落ち | 性質軸で3分類を強制 |
| 抽象的 | 「良い製品」など | 具体的な顧客状態で記述 |

## 8. 既存V5からの移行

### 8.1 既存VL定義の拡張

既存のVL定義がある場合の拡張手順：

```
1. 既存VL1の分析
   └── 「誰への」「いつの」「どんな」価値かを分解

2. 不足軸の補完
   └── 明示されていない軸を追加定義

3. ポートフォリオの可視化
   └── 3軸マトリクスにマッピング

4. ギャップの特定
   └── 不足している価値領域を発見

5. VL2/VL3の再構成
   └── 3軸を反映した階層に再編
```

### 8.2 互換性の維持

- 既存のVS0-VS7構造は変更なし
- 既存のVMS構造は変更なし
- VL定義のメタデータとして3軸分類を追加
- 後方互換性を維持しつつ拡張

## 9. まとめ

### 9.1 V5.6で追加される概念

| 概念 | 定義 | V5との関係 |
|------|------|-----------|
| **価値分解3軸** | ステークホルダー/時間/性質 | VL定義の前工程として追加 |
| **価値ポートフォリオ** | 3軸による投資配分管理 | 新規概念 |
| **ステークホルダー別VL** | 各ステークホルダーへの価値定義 | VL1の拡張 |
| **時間軸別価値** | 現在/将来/持続の区分 | VMS/TVDCとの連動 |
| **価値性質分類** | 機能的/情緒的/社会的 | Core判定の入力 |

### 9.2 期待効果

1. **投資判断の明確化**: どの価値領域に投資すべきかが明確に
2. **ステークホルダー整合**: 顧客以外への価値も体系的に管理
3. **将来への布石**: イノベーション投資の根拠が明確に
4. **ESG対応**: 社会価値・持続価値が体系的に組み込まれる
