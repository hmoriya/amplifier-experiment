# VS2: 製品開発・イノベーション - Subdomain Design (CL2)

**プロジェクト:** asashi (アサヒグループホールディングス)
**Value Stream:** VS2 - 製品開発・イノベーション
**ドメインタイプ:** Core Domain
**作成日:** 2025-11-26
**分析モード:** zen-architect (ARCHITECT mode)

---

## 1. Value Stream 概要

### 1.1 目的

独自技術を活かした製品開発と継続的イノベーションを通じて、「期待を超えるおいしさ」を実現し、競争優位性を確立する。

### 1.2 Core Domainとしての位置づけ

VS2は以下の理由からCore Domainに分類される：

| 評価軸 | 根拠 |
|--------|------|
| 競争優位性 | 100年以上の醸造技術・独自酵母株は模倣困難 |
| 差別化要因 | Asahi Super Dry辛口製法、ノンアル技術 |
| 投資優先度 | 最高（フルカスタム開発必須） |

## 3. サブドメイン詳細

### 3.1 vs2-fermentation-research（発酵技術研究）

**ドメインタイプ:** Core

**目的:**
競争優位の源泉である酵母・発酵技術の継続的研究と革新を行う。100年以上の蓄積を活かし、模倣困難な技術資産を維持・発展させる。

**主要機能:**

| 機能 | 説明 |
|------|------|
| 酵母研究・育種 | 独自酵母株の開発・改良・保存 |
| 発酵プロセス開発 | 発酵条件の最適化・新製法研究 |
| 乳酸菌・枯草菌研究 | 機能性微生物の探索・活用 |
| 素材探索 | 新規原料・機能性素材の研究 |

**対応活動:** A1（基盤技術研究）

**データ所有範囲:**

| データ | 説明 | 機密レベル |
|--------|------|----------|
| 酵母株データベース | 独自酵母株の遺伝情報・特性 | 極秘 |
| 発酵パラメータ | 温度・時間・pH等の最適条件 | 極秘 |
| 研究成果・論文 | 学術成果・特許情報 | 秘 |
| 実験データ | 研究過程の測定データ | 秘 |

**依存関係:**

| 関係 | 対象 | 内容 |
|------|------|------|
| 提供先 | vs2-alcoholic-product-dev | 酵母・発酵技術の提供 |
| 提供先 | vs2-beverage-product-dev | 発酵・微生物技術の提供 |
| 提供先 | vs2-recipe-management | 発酵条件データの提供 |
| 入力元 | 外部（大学・研究機関） | 学術情報・共同研究 |

**マイクロサービス評価:**

| 評価項目 | スコア | 根拠 |
|----------|--------|------|
| 責務の凝集性 | ★★★★★ | 発酵・酵母技術に完全に集中 |
| 変更の独立性 | ★★★★☆ | 研究進捗は他と独立して進行 |
| チーム配置 | ★★★★★ | 専門研究者チームで運用 |
| データ境界 | ★★★★★ | 極秘データを独自に管理必須 |
| スケーリング要件 | ★★★☆☆ | 研究規模に依存、急拡大は稀 |

**推奨:** 独立マイクロサービス候補（最優先）

## 7. 付録: サブドメイン分解の評価基準

### 使用した評価軸

| 評価軸 | 定義 | 基準 |
|--------|------|------|
| 責務の凝集性 | 単一の明確な責務を持つか | ★5=完全に単一責務 |
| 変更の独立性 | 他と独立して変更できるか | ★5=完全に独立 |
| チーム配置 | 独立したチームで運用可能か | ★5=専門チーム配置可能 |
| データ境界 | 独自のデータストアを持つべきか | ★5=独立管理必須 |
| スケーリング要件 | 独立したスケーリングが必要か | ★5=独立スケーリング必須 |

### ドメインタイプ分類基準

| 分類 | 基準 | VS2での例 |
|------|------|----------|
| Core | 競争優位の源泉、独自技術・ノウハウ | 発酵技術、レシピ、製品開発 |
| Supporting | ビジネス運営を支援する重要機能 | 品質保証、パッケージ、生産技術 |
| Generic | 汎用的な管理機能、外部活用可能 | プロジェクト管理 |
