# VS1: 研究開発 - CL2サブドメイン設計

**プロジェクト:** asashi (アサヒグループホールディングス)
**作成日:** 2025-11-27
**ステータス:** CL2完了（ビジネスオペレーション群定義）
**分析モード:** zen-architect (ARCHITECT mode)

---

## エグゼクティブサマリー

VS1（研究開発）のCore活動領域を**10個のサブドメイン（ビジネスオペレーション群）**に分解しました。

**分類結果:**
- **Core Subdomains（6個）**: 競争優位性を生む研究・技術基盤領域
- **Supporting Subdomains（4個）**: Core業務を支援する管理・連携領域

**主要設計判断:**
1. **発酵プラットフォーム思考の体現**: 酵母ライブラリ＝API群として「fermentation-platform」を中核に設計
2. **「予期せぬ価値の計画的創出」**: 失敗実験からの価値発見を「serendipity-harvesting」として独立サブドメイン化
3. 基盤研究（VS1-C1〜C4）と知識管理（VS1-C5）の明確な分離

**革新的インサイトの反映:**
- 酵母ライブラリをAPIとして捉え、外部・内部からの呼び出し可能な「発酵プラットフォーム」設計
- 失敗・想定外結果の体系的記録と価値変換プロセスを専門サブドメインとして確立

## 命名理由（Naming Rationale）

各サブドメインの命名は、Parasol V5の命名ガイドラインに基づき、
「XXX管理」パターンを避け、DXにふさわしい**進化・発展・改善**を表現しています。

| サブドメイン | 日本語名 | 命名理由 |
|-------------|---------|----------|
| fermentation-platform | 発酵プラットフォーム | 「発酵管理」ではなく「プラットフォーム」を採用。酵母ライブラリ＝API群という革新的インサイトを反映。100年以上の醸造技術を**呼び出し可能な基盤**として表現。`-platform`パターン（基盤型）を適用 |
| functional-material-discovery | 機能性素材探索 | 「素材管理」ではなく「探索」を採用。新規機能性成分の**発見・開拓**を表現。`-discovery`パターン（探索型）を適用 |
| ingredient-innovation | 原料イノベーション | 「原料研究」ではなく「イノベーション」を採用。原料調達・品質を超えた**価値創造**を表現。`-innovation`パターン（革新型）を適用 |
| foundational-science | 基盤科学研究 | 「基礎研究」ではなく「基盤科学」を採用。長期的競争力の**科学的土台構築**を表現。`-science`パターン（学術型）を適用 |
| serendipity-harvesting | セレンディピティ収穫 | 「予期せぬ価値の計画的創出」インサイトを具現化。失敗・想定外結果からの**価値収穫**を表現。革新的命名で差別化 |
| ip-portfolio-strategy | 知財ポートフォリオ戦略 | 「知財管理」ではなく「戦略」を採用。特許を防御的資産ではなく**事業成長の戦略的武器**として表現。`-strategy`パターン（戦略型）を適用 |
| academia-alliance | 産学連携 | 「産学管理」ではなく「連携」を採用。大学・研究機関との**協働的価値創造**を表現。`-alliance`パターン（連携型）を適用 |
| research-knowledge-synthesis | 研究知識統合 | 「ナレッジ管理」ではなく「統合」を採用。分散した研究知見の**横断的統合・創発**を表現。`-synthesis`パターン（統合型）を適用 |
| technology-transfer | 技術移転 | 業界標準用語。研究成果の製品化への**橋渡し・実装**を表現 |
| research-operations | 研究オペレーション | 研究活動を支える設備・リソース・プロジェクトの**運営最適化**を表現。`-operations`パターン（運営型）を適用 |

**VS横断一意性**: 上記10個のサブドメイン名は、全VS（VS0〜VS7）において一意であることを確認済み。
VS2の fermentation-research との差別化: VS1は「platform」として基盤提供、VS2は製品開発における「research」として利用側を表現。

## 詳細設計

### 1. fermentation-platform（発酵プラットフォーム）

**所属CL1:** VS1-C1 発酵プラットフォーム研究
**ドメインタイプ:** Core ★★★★★

#### 目的・責務

100年以上蓄積された醸造技術を「呼び出し可能なAPI群」として体系化し、全製品開発の技術基盤を提供する。酵母ライブラリを中心に、発酵プロセスを変換エンジンとして運用する。アサヒグループの競争優位性の核心。

#### 主要ビジネスオペレーション

| オペレーション | 説明 | トリガー | 成果物 |
|---------------|------|---------|--------|
| 酵母株ライブラリ管理 | 酵母株の分類・特性定義・API化 | 継続業務、新株発見 | 酵母API仕様、株マスター |
| 酵母株探索・収集 | グローバルでの新規酵母株発見 | 研究計画、外部情報 | 新規酵母サンプル、評価報告 |
| 酵母株育種・改良 | 目的特性を持つ酵母の育種 | 製品要件、技術課題 | 改良酵母株、育種記録 |
| 発酵条件最適化 | 発酵プロセスのパラメータ最適化 | 品質要件、効率目標 | 最適条件API、レシピ |
| 発酵実験・検証 | 新技術・新条件の実験検証 | 研究仮説、開発要件 | 実験データ、検証報告 |
| 発酵プロセス設計 | 新製品向け発酵プロセス設計 | VS2製品要件 | プロセス仕様書 |
| プラットフォームAPI公開 | 内部・外部向けAPI提供 | 新規利用申請 | APIドキュメント |

#### 管理するデータ（情報資産）

- **酵母ライブラリ（Yeast API Catalog）**: 全酵母株の特性・パラメータ・呼び出し仕様
- **発酵プロセスレシピDB**: 製品別発酵条件・変換ロジック
- **発酵実験データレイク**: 過去全実験の条件・結果・メタデータ
- **酵母系統図**: 育種履歴・遺伝的関係性マップ

#### ユビキタス言語

| 用語 | 定義 |
|-----|------|
| 酵母API | 特定の酵母株を利用するためのインターフェース定義（入力条件、出力特性、制約） |
| 発酵変換 | 原料と条件を入力として、目的特性を持つ中間体または最終製品を出力するプロセス |
| 株プロファイル | 酵母株の全特性を記述したメタデータセット |
| 発酵パイプライン | 複数の発酵プロセスを連結した処理フロー |
| 活性スコア | 酵母株の現在の生存性・発酵能力を示す指標 |

#### 他サブドメインとの関係

| 関係タイプ | サブドメイン | 内容 |
|-----------|-------------|------|
| 提供先 | VS2 premium-beer-development | 酵母API、発酵プロセス |
| 提供先 | VS2 craft-innovation-development | 特殊酵母API、新発酵技術 |
| 提供先 | VS2 spirits-development | 蒸留用酵母API |
| 連携 | functional-material-discovery | 発酵による機能性成分生成 |
| 連携 | serendipity-harvesting | 予期せぬ発酵結果の価値化 |
| 受領元 | academia-alliance | 新規酵母株、共同研究成果 |

#### 境界コンテキスト候補

**fermentation-platform-bc** として独立BC化推奨
- 理由: 独自の複雑なドメインモデル（酵母、発酵プロセス、条件パラメータ）
- 他BCとの明確な境界（API経由での提供）
- 最重要機密情報（レシピ）の保護要件

### 2. functional-material-discovery（機能性素材探索）

**所属CL1:** VS1-C2 機能性素材研究
**ドメインタイプ:** Core ★★★★☆

#### 目的・責務

乳酸菌・枯草菌等の微生物や天然素材から、健康効果を持つ新規機能性成分を発見・開発する。成長市場（健康・ウェルネス）への対応を推進。

#### 主要ビジネスオペレーション

| オペレーション | 説明 | トリガー | 成果物 |
|---------------|------|---------|--------|
| 機能性微生物探索 | 新規機能性微生物の発見 | 研究計画、健康トレンド | 新規微生物株、特性データ |
| 機能性成分スクリーニング | 候補成分の機能性評価 | 探索結果 | スクリーニング報告 |
| エビデンス構築 | 健康効果の科学的検証 | 候補成分確定 | 臨床試験データ、論文 |
| 機能性表示対応 | 規制要件への適合研究 | 製品化決定 | 届出資料、エビデンスパッケージ |
| 微生物バンク管理 | 乳酸菌等の保存・品質管理 | 継続業務 | 微生物バンク、品質記録 |

#### 管理するデータ（情報資産）

- **機能性微生物ライブラリ**: 乳酸菌・枯草菌等の株情報・特性
- **機能性エビデンスDB**: 健康効果の科学的根拠・臨床データ
- **スクリーニング履歴DB**: 評価済み候補成分・結果
- **規制対応資料DB**: 各国機能性表示届出資料

#### ユビキタス言語

| 用語 | 定義 |
|-----|------|
| 機能性プロファイル | 微生物または成分が持つ健康効果の総合特性記述 |
| エビデンスレベル | 健康効果の科学的根拠の強さを示す段階（in vitro→動物試験→ヒト試験→RCT） |
| 届出エビデンス | 機能性表示食品の届出に必要な科学的根拠セット |
| 有効性閾値 | 機能性効果が発現する最小投与量 |

#### 他サブドメインとの関係

| 関係タイプ | サブドメイン | 内容 |
|-----------|-------------|------|
| 提供先 | VS2 beverage-development | 機能性成分、乳酸菌株 |
| 連携 | fermentation-platform | 発酵による機能性成分生成 |
| 連携 | ip-portfolio-strategy | 機能性関連特許出願 |
| 連携 | serendipity-harvesting | 想定外の機能性発見 |

### 3. ingredient-innovation（原料イノベーション）

**所属CL1:** VS1-C3 原料研究
**ドメインタイプ:** Core ★★★★☆

#### 主要ビジネスオペレーション

| オペレーション | 説明 | トリガー | 成果物 |
|---------------|------|---------|--------|
| 原料品種開発 | 新品種の共同開発・育種 | 研究計画、品質課題 | 新品種、育種記録 |
| 原料特性分析 | 原料の化学・物理特性の詳細分析 | 品質管理、開発要件 | 分析データ、特性シート |
| 原料-製品相関研究 | 原料特性と製品品質の相関解明 | 品質向上要件 | 相関モデル、レシピ改善案 |
| サステナブル原料研究 | 環境負荷低減・安定調達の研究 | ESG方針、気候変動 | サステナビリティ評価報告 |
| 原料サプライヤー技術支援 | 契約農家・サプライヤーへの技術指導 | パートナーシップ | 技術指導記録、品質改善 |

#### 管理するデータ（情報資産）

- **原料マスターDB**: 原料品種・産地・特性・調達先情報
- **原料-品質相関モデル**: 原料特性と製品品質の予測モデル
- **品種開発履歴DB**: 育種プロジェクト・結果履歴
- **サステナビリティ評価DB**: 原料別環境負荷・持続可能性スコア

#### ユビキタス言語

| 用語 | 定義 |
|-----|------|
| 原料プロファイル | 原料の品種、産地、収穫年、特性値の総合記述 |
| 品質寄与度 | 特定の原料特性が最終製品品質に与える影響度 |
| テロワール特性 | 産地固有の風土が原料に与える特性 |
| 育種パイプライン | 品種開発の段階的プロセス（探索→評価→育種→試験→登録） |

#### 他サブドメインとの関係

| 関係タイプ | サブドメイン | 内容 |
|-----------|-------------|------|

| 提供先 | VS2 beverage-development | 飲料原料知見 |
| 連携 | fermentation-platform | 原料-発酵適合性 |
| 連携 | VS4 調達 | 原料調達戦略 |

### 6. ip-portfolio-strategy（知財ポートフォリオ戦略）

**所属CL1:** VS1-C5 知的財産管理

## CL3への準備

### 優先的にBounded Context定義すべきサブドメイン

| 優先度 | サブドメイン | 理由 |
|--------|-------------|------|
| **1** | fermentation-platform | 競争優位の核心。酵母API、発酵プロセスの複雑なドメインモデル |
| **2** | serendipity-harvesting | アサヒ独自の革新的概念。想定外結果の価値転換プロセス |
| **3** | functional-material-discovery | 成長市場対応。エビデンス管理が重要 |
| **4** | ip-portfolio-strategy | 事業戦略との連携。特許ポートフォリオ管理 |

### 境界コンテキスト統合/分離の判断基準

| 判断基準 | 統合推奨 | 分離推奨 |
|----------|----------|----------|
| 言語の一致度 | 同一用語が同一意味 | 同一用語が異なる意味 |
| データ所有権 | 共有データが多い | 独立データが明確 |
| 変更頻度 | 同時に変更される | 独立して変更される |
| チーム構造 | 同一チームが担当 | 異なるチームが担当 |
| セキュリティ要件 | 同等のセキュリティ | 異なる機密レベル |

### 次のコマンド

```
CL3: fermentation-platform のBounded Context定義
→ 集約設計（YeastStrain, FermentationProcess, FermentationExperiment）
→ ドメインイベント（YeastStrainRegistered, FermentationCompleted）
→ API設計（酵母API、発酵プロセスAPI）
```
