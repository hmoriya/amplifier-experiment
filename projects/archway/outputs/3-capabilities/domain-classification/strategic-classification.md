# Strategic Domain Classification (CL1)

プロジェクト: archway
作成日: 2025-01-26

## ドメイン分類サマリー

### Core Domains（競争優位の源泉）
1. **architecture-design**: アーキテクチャ設計とモダナイゼーション（VS2）
2. **development-process-transformation**: 開発プロセス変革（VS3）
3. **continuous-improvement**: 継続的改善とイノベーション（VS6）

### Supporting Domains（ビジネス支援）
1. **market-opportunity**: 市場機会発見と技術評価（VS1）
2. **organizational-transformation**: 組織・文化変革支援（VS4）
3. **tool-integration**: ツール統合とプラットフォーム構築（VS5）
4. **knowledge-transfer**: 人材育成とナレッジ移転（VS7）

### Generic Domains（汎用機能）
1. **strategic-planning**: ビジョン策定と戦略立案（VS0）

## 詳細分析

### Core Domain: architecture-design

**説明**: 最適なソフトウェアアーキテクチャの設計とレガシーシステムの近代化

**関連Value Stream**:
- VS2: アーキテクチャ設計とモダナイゼーション

**評価**:
- 戦略的重要性: ★★★★★ (5/5)
- 競争優位性: ★★★★★ (5/5)
- 複雑度: ★★★★★ (5/5)
- カスタマイズ必要性: ★★★★★ (5/5)

**分類理由**:
アークウェイの最も重要な競争優位の源泉。技術的専門性の中核であり、顧客が最も価値を見出す領域。マイクロサービス設計、クラウドネイティブアーキテクチャなど、高度な専門知識が必要。各顧客のコンテキストに応じた完全なカスタマイズが必須。

**推奨アプローチ**: 最高品質の人材投入、継続的な技術研鑽、独自メソドロジーの開発

---

### Core Domain: development-process-transformation

**説明**: DevOps導入による開発プロセスの最適化と自動化推進

**関連Value Stream**:
- VS3: 開発プロセス変革

**評価**:
- 戦略的重要性: ★★★★★ (5/5)
- 競争優位性: ★★★★☆ (4/5)
- 複雑度: ★★★★☆ (4/5)
- カスタマイズ必要性: ★★★★★ (5/5)

**分類理由**:
"Developers Happiness" の実現に直結する中核領域。DevOps導入は技術だけでなく、組織文化の変革も含む複雑な取り組み。アークウェイの実装支援力が最も発揮される領域で、顧客の開発現場に即したカスタマイズが必要。

**推奨アプローチ**: DevOpsエキスパートの育成、独自のCI/CDテンプレート開発、成功事例の蓄積

---

### Core Domain: continuous-improvement

**説明**: 継続的な改善活動の推進と新技術を活用したイノベーション創出

**関連Value Stream**:
- VS6: 継続的改善とイノベーション

**評価**:
- 戦略的重要性: ★★★★★ (5/5)
- 競争優位性: ★★★★☆ (4/5)
- 複雑度: ★★★★☆ (4/5)
- カスタマイズ必要性: ★★★★☆ (4/5)

**分類理由**:
顧客の持続的な競争力向上に貢献する領域。データ分析に基づく改善提案、新技術の実験的導入など、アークウェイの技術力と革新性が差別化要因となる。長期的な顧客関係構築の鍵。

**推奨アプローチ**: イノベーションラボの設立、継続的な技術実験、顧客との共同イノベーション

---

### Supporting Domain: market-opportunity

**説明**: 新技術の評価と活用機会の発見、実証を通じた価値検証

**関連Value Stream**:
- VS1: 市場機会発見と技術評価

**評価**:
- 戦略的重要性: ★★★☆☆ (3/5)
- 競争優位性: ★★☆☆☆ (2/5)
- 複雑度: ★★★☆☆ (3/5)
- カスタマイズ必要性: ★★★☆☆ (3/5)

**分類理由**:
Core Domainを支える重要な活動だが、技術評価自体は多くのコンサルティング会社が提供可能。アークウェイの強みは評価後の実装にあるため、Supporting として位置づけ。

**推奨アプローチ**: 効率的な技術評価フレームワーク構築、パートナーシップ活用

---

### Supporting Domain: organizational-transformation

**説明**: 技術組織の変革と開発文化の醸成、働き方改革の推進

**関連Value Stream**:
- VS4: 組織・文化変革支援

**評価**:
- 戦略的重要性: ★★★★☆ (4/5)
- 競争優位性: ★★★☆☆ (3/5)
- 複雑度: ★★★☆☆ (3/5)
- カスタマイズ必要性: ★★★☆☆ (3/5)

**分類理由**:
DevOps成功には組織変革が必須だが、純粋な組織コンサルティングは競争優位の直接的な源泉ではない。技術変革と一体で提供することで価値を生む支援領域。

**推奨アプローチ**: 変革方法論の標準化、外部専門家との協業

---

### Supporting Domain: tool-integration

**説明**: 開発ツールチェーンの最適化と統合プラットフォームの構築

**関連Value Stream**:
- VS5: ツール統合とプラットフォーム構築

**評価**:
- 戦略的重要性: ★★★☆☆ (3/5)
- 競争優位性: ★★☆☆☆ (2/5)
- 複雑度: ★★★☆☆ (3/5)
- カスタマイズ必要性: ★★★☆☆ (3/5)

**分類理由**:
ツール導入自体は差別化要因にならないが、アーキテクチャやプロセスと統合したツールチェーン設計は価値を生む。Atlassian、Microsoftなどのパートナーシップを活用。

**推奨アプローチ**: パートナー製品の活用、統合テンプレートの開発

---

### Supporting Domain: knowledge-transfer

**説明**: 技術人材の育成と組織へのナレッジ定着による自走化支援

**関連Value Stream**:
- VS7: 人材育成とナレッジ移転

**評価**:
- 戦略的重要性: ★★★☆☆ (3/5)
- 競争優位性: ★★☆☆☆ (2/5)
- 複雑度: ★★☆☆☆ (2/5)
- カスタマイズ必要性: ★★★☆☆ (3/5)

**分類理由**:
顧客の自走化を支援する重要な活動だが、教育・研修自体は汎用的。アークウェイの価値は実践的な技術知識の移転にある。

**推奨アプローチ**: 研修プログラムの体系化、e-learning活用

---

### Generic Domain: strategic-planning

**説明**: 顧客企業のデジタル戦略立案とIT活用による未来デザイン支援

**関連Value Stream**:
- VS0: ビジョン策定と戦略立案

**評価**:
- 戦略的重要性: ★★☆☆☆ (2/5)
- 競争優位性: ★☆☆☆☆ (1/5)
- 複雑度: ★★☆☆☆ (2/5)
- カスタマイズ必要性: ★★☆☆☆ (2/5)

**分類理由**:
戦略立案は多くのコンサルティング会社が提供する汎用サービス。アークウェイの強みは戦略の実装にあり、戦略立案自体は差別化要因にならない。標準的なフレームワークで対応可能。

**推奨アプローチ**: 既存フレームワークの活用、効率的なテンプレート利用

## 投資配分推奨

```
Core Domains (3個):      70% の投資
- architecture-design:         30%
- development-process-transformation: 25%
- continuous-improvement:      15%

Supporting Domains (4個): 25% の投資
- market-opportunity:          5%
- organizational-transformation: 8%
- tool-integration:           5%
- knowledge-transfer:         7%

Generic Domains (1個):    5% の投資
- strategic-planning:         5%
```

## 戦略的推奨事項

1. **Core Domain集中投資**: 
   - アーキテクチャ設計とDevOps実装に最高品質のエンジニアを配置
   - 独自メソドロジーとIPの開発に注力
   - 成功事例とベストプラクティスの蓄積

2. **Supporting Domain効率化**: 
   - 標準化可能な部分はテンプレート化
   - パートナー製品・サービスの積極活用
   - Core Domainとの相乗効果を重視

3. **Generic Domain最小化**: 
   - 戦略立案は既存フレームワークを活用
   - 必要最小限のリソース配分
   - Core Domain への導入として位置づけ

4. **差別化の強化**:
   - "Developers Happiness" を体現する独自サービス開発
   - 実装力・技術力での圧倒的優位性確立
   - 顧客の開発現場に寄り添う姿勢の維持

Core Domain から優先的にサブドメイン設計を実施：

```bash
# Core Domain のVSから開始（優先度高）
/parasol:3-capabilities cl2 VS2    # アーキテクチャ設計とモダナイゼーション
/parasol:3-capabilities cl2 VS3    # 開発プロセス変革
/parasol:3-capabilities cl2 VS6    # 継続的改善とイノベーション

# Supporting Domain のVS
/parasol:3-capabilities cl2 VS1    # 市場機会発見と技術評価
/parasol:3-capabilities cl2 VS4    # 組織・文化変革支援
/parasol:3-capabilities cl2 VS5    # ツール統合とプラットフォーム構築
/parasol:3-capabilities cl2 VS7    # 人材育成とナレッジ移転

# Generic Domain のVS
/parasol:3-capabilities cl2 VS0    # ビジョン策定と戦略立案
```
