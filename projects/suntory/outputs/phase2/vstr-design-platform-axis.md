# サントリー Value Stream設計 - プラットフォーム軸パターン分析

**作成日**: 2025-12-12
**分析対象**: サントリーホールディングス株式会社
**フレームワーク**: Parasol V5 - Platform Axis Pattern

---

## エグゼクティブサマリー

サントリーは現在、製造業を中心としたビジネスモデルを展開しており、両面市場型のプラットフォーム構造は形成されていない。ただし、自販機ネットワーク（66万台）やウエルネス事業のEC通販といった特定アセットには、将来的なプラットフォーム化の潜在的可能性が認められる。

### 結論: 現状不適合 / 将来オプションとして有望

| 評価項目 | 現状評価 | 将来可能性 |
|---------|---------|-----------|
| 両面市場構造 | 未形成 | 自販機ネットワークで構築可能 |
| ネットワーク効果 | 限定的 | IoT化で創出可能 |
| プラットフォーム価値 | マッチング機能なし | データプラットフォームへ進化可能 |
| **総合適合度** | **20/100** | **70/100（5年後ポテンシャル）** |

## 1. プラットフォーム軸パターンの概要

### 1.1 適用条件の確認

プラットフォーム軸パターンは以下の特性を持つ事業に適用する：

| 条件 | 説明 | サントリー適合性 |
|------|------|----------------|
| **推奨** | 需要者と供給者をマッチングするプラットフォーム事業 | **不適合** |
| **推奨** | ネットワーク効果（片側または双方向）が事業価値の源泉 | **限定的** |
| **推奨** | 取引手数料/サブスクリプションが主要収益モデル | **不適合** |
| **注意** | B2B2C構造との違いを明確に | 仲介者モデル |

## 2. プラットフォーム適合性の詳細評価

### 2.1 両面市場構造の有無

| チャネル/事業 | 需要側 | 供給側 | マッチング機能 | 評価 |
|-------------|-------|-------|--------------|------|
| **酒類（小売）** | 消費者 | サントリー（製造者） | なし | 単一供給者 |
| **飲料（自販機）** | 消費者 | サントリー（製造者） | なし | 単一供給者 |
| **飲料（コンビニ/SM）** | 消費者 | サントリー+他社 | 小売店が選択 | B2B2C |
| **業務用（飲食店）** | 飲食店 | サントリー（製造者） | なし | B2B取引 |
| **ウエルネス（EC）** | 消費者 | サントリー（製造者） | なし | D2C |

**結論**: 現状、サントリーは全チャネルにおいて「供給者」として機能しており、「需要者と供給者をマッチングするプラットフォーマー」の役割を果たしていない。

### 2.2 ネットワーク効果の源泉

#### 2.2.1 片側ネットワーク効果

| 要素 | 現状 | 可能性 |
|-----|------|-------|
| 消費者同士の相互作用 | なし | SNSコミュニティで限定的に創出可能 |
| ブランドファンの集積効果 | 限定的（ウイスキー愛好家） | ファンクラブ/会員制で強化可能 |

#### 2.2.2 双方向ネットワーク効果

```
プラットフォーム型:
  需要者↑ → 供給者価値↑ → 供給者↑ → 需要者価値↑

サントリー現状:
  消費者↑ → サントリー収益↑（直線的）
  ネットワーク効果は発生しない
```

**結論**: 現状のビジネスモデルではネットワーク効果が発生する構造になっていない。

### 2.3 プラットフォーム価値の検討

| 機能 | 現状 | 必要性 |
|------|------|-------|
| マッチング | なし（自社製品のみ販売） | プラットフォーム化には必須 |
| 決済 | 小売経由（コンビニ等） | D2C拡大で自社決済が成長 |
| 信頼構築 | ブランド信頼（製造者として） | 既に強い |
| サポート | 顧客対応（製造者として） | 標準的 |
| データ活用 | 内部利用中心 | 外部開放でプラットフォーム化可能 |

### 3.2 乖離要因

| 要因 | 説明 | 解消難易度 |
|------|------|----------|
| **単一供給者構造** | サントリーが唯一の供給者であり、複数供給者のマッチングが不在 | 高 |
| **チャネル依存** | 小売・飲食店が顧客接点を保有し、サントリーは製造者 | 高 |
| **収益モデル** | 製品販売収益が主、手数料・サブスクは健康食品のみ | 中 |
| **データ非開放** | 消費者データは内部利用が中心、外部供給者への開放なし | 低 |

## 8. 結論と提言

### 8.1 結論

**サントリーにプラットフォーム軸パターンは現状不適合である。**

理由:
1. **製造業モデル**: サントリーは「供給者」であり「プラットフォーマー」ではない
2. **両面市場の欠如**: 需要者と複数供給者をマッチングする構造がない
3. **ネットワーク効果なし**: 利用者増加が直接的な価値増大に繋がらない
4. **収益モデル**: 製品販売が主であり、手数料/データ収益は限定的

### 8.2 将来オプションとしての可能性

**自販機データプラットフォーム構想（Option B）** は将来的に検討の価値がある。

| 検討ポイント | 内容 |
|------------|------|
| **資産活用** | 66万台のネットワークは模倣困難な資産 |

### A. プラットフォーム軸 VStr定義（参考）

サントリーに適用した場合の仮想的な定義：

#### 需要者向け VStr（DS: Demand Stage）

| Stage | 状態 | サントリー仮想定義 |
|-------|------|-------------------|
| DS0 | ニーズ潜在 | 飲料・酒類のニーズを自覚していない |
| DS1 | 認知到達 | 自販機/ボスマートの存在を知る |
| DS2 | 探索開始 | アプリで近くの自販機を検索 |
| DS3 | 選択決定 | 商品を選択（複数メーカーから） |
| DS4 | 取引完了 | 購入完了 |
| DS5 | 利用開始 | 商品を消費 |
| DS6 | 継続利用 | リピート購入 |
| DS7 | 推奨共有 | アプリで評価、友人に推奨 |

#### 供給者向け VStr（SS: Supply Stage）

| Stage | 状態 | サントリー仮想定義 |
|-------|------|-------------------|
| SS0 | 参入検討 | 自販機ネットワークへの出品を検討 |
| SS1 | 認知到達 | サントリー自販機PFを知る |
| SS2 | 登録完了 | 供給者として登録 |
| SS3 | 出品開始 | 商品を自販機に出品 |
| SS4 | 取引成立 | 販売成立 |
| SS5 | 収益実感 | 売上・データインサイトを獲得 |
| SS6 | 拡大成長 | 出品拡大 |
| SS7 | パートナー化 | 戦略的パートナー |
