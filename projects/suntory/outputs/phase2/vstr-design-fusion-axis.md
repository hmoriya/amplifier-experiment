# サントリー フュージョン型バリューストリーム設計

**作成日**: 2025-12-12
**パターン**: フュージョン型（共通層 + 事業層）
**評価フレームワーク**: 価値駆動5軸評価

---

## 1. フュージョン型パターン概要

フュージョン型は、**共通バリューストリームで骨格を作り、事業部固有の拡張を許容**する柔軟な構造です。サントリーの「水と生きる」というVL1を全社で共有しながら、酒類・飲料・健康食品の事業特性を尊重します。

### 1.1 設計原則

| 原則 | 内容 |
|------|------|
| **共通層 = Generic/Supporting Layer** | VS0（戦略）、VS6（データ基盤）、VS7（品質・サステナビリティ）は全事業を支える基盤 |
| **事業層 = Core Domain** | VS1-VS5は各事業部が顧客価値創造を担う |
| **共通層 ≠ シナジーVStr** | 共通層は「シナジー創出のVStr」ではなく、設計原則・基盤機能として機能 |

### 4.2 判定理由

| 観点 | 評価 | 理由 |
|------|------|------|
| **VL1との整合性** | 非常に高い | 「水と生きる」が共通層VS0として自然に設計原則化できる |
| **組織構造との親和性** | 高い | 現行のHD-事業会社構造とフュージョン型が整合 |
| **技術シナジーの実現** | 非常に高い | 水科学・発酵技術が共通層VS7で全事業に横展開可能 |
| **顧客データ活用** | 高い | CDP統合（VS6）によりクロスセル機会を最大化 |
| **ガバナンス実現可能性** | 中〜高 | 共通層のイネーブラー評価は設計可能だが運用に注意要 |

```
│  5. 段階的導入が可能                                                       │
│  ├─────────────────────────────────────────────────────────────────────│   │
│  │ ・まず共通層（VS0,6,7）を整備し、事業層は順次適用                   │   │
│  │ ・既存組織構造を大きく変えずに導入可能                              │   │
│  │ ・新規事業追加時も事業層を追加するだけで対応                        │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.2 弱み

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    フュージョン型の弱み（サントリー適用時）                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 共通層の形骸化リスク                                                   │
│  ├─────────────────────────────────────────────────────────────────────│   │
│  │ ・「水と生きる」がお題目化し、事業層との実質的な接続が弱まる        │   │
│  │ ・共通層の成果が「使われること」でしか測定できない構造的課題        │   │
│  │ 【対策】価値ストーリーの可視化、共通層のイネーブラー評価の徹底      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  2. HD-事業間の調整コスト                                                  │
│  ├─────────────────────────────────────────────────────────────────────│   │
│  │ ・共通層と事業層の責任分界が曖昧になりやすい                        │   │
│  │ ・共通化範囲の決定に時間がかかる（合意形成コスト）                  │   │
│  │ 【対策】RACI定義、価値統合会議の月次開催、迅速な意思決定ルール      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  3. 二重ガバナンス構造                                                     │
│  ├─────────────────────────────────────────────────────────────────────│   │
│  │ ・共通層と事業層で異なるKPI体系が並存                               │   │
│  │ ・両層の整合性維持に継続的な努力が必要                              │   │
│  │ 【対策】価値起点の統合ダッシュボード、半期ごとの投資配分調整        │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  4. 顧客ジャーニーの分断リスク                                             │
│  ├─────────────────────────────────────────────────────────────────────│   │
│  │ ・事業層VS3以降の差異が大きく、事業横断の顧客体験設計が困難         │   │
│  │ ・酒類顧客が飲料に移行する際の体験連続性の確保が課題                │   │
│  │ 【対策】CDPでの顧客ジャーニー統合管理、事業横断CXチームの設置       │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 6. 推奨事項

### 6.1 導入ロードマップ

```
Phase 1（3ヶ月）: 共通層の設計・合意
├── VS0: 「水と生きる」設計原則の明文化
├── VS6: CDP統合構想の策定
├── VS7: 品質基準・サステナ目標の統一
└── 共通層評価フレームワークの設計

Phase 2（6ヶ月）: パイロット事業での適用
├── 飲料事業でのVS1-5詳細設計
├── 共通層⇔事業層の連携プロセス検証
├── 価値ストーリーの可視化ダッシュボード構築
└── ガバナンス体制の試行

Phase 3（12ヶ月）: 全事業展開
├── 酒類事業・健康食品事業への展開
├── 事業間クロスセル施策の実行
├── インセンティブ設計の本格導入
└── 継続的改善サイクルの確立
```

### 6.2 重点施策

| 施策 | 内容 | 優先度 |
|------|------|--------|
| **共通層イネーブラー評価の導入** | 共通層の成果を事業層の能力向上度で評価 | 高 |
| **価値ストーリー可視化** | Operation→Capability→VS→VL1の因果をダッシュボード化 | 高 |
| **RACI定義の策定** | HD-事業間の責任分界を明確化 | 高 |
| **CDP統合の推進** | 顧客データ基盤の事業横断活用 | 中 |
| **価値統合会議の設置** | 月次で共通層の貢献度と事業層からのフィードバックを確認 | 中 |
