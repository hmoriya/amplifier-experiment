**作成日**: 2025-12-12
**Phase**: 2 - Value（価値定義）
**軸パターン**: ケイパビリティ統合型（Capability Integration Axis）
**評価フレームワーク**: 価値駆動5軸評価

---

## 1. ケイパビリティ統合型パターン概要

### 1.1 パターン定義

ケイパビリティ統合型は、事業横断で共通する「能力（ケーパビリティ）」を中心にValue Streamを設計するアプローチです。機能別組織やシェアードサービス化を志向する企業に適しています。

### 1.2 サントリーへの適用理由

| 適用条件 | サントリーの状況 | 判定 |
|---------|----------------|------|
| 共通ケイパビリティの存在 | 発酵・醸造技術、水科学研究、品質保証、サプライチェーン | 該当 |
| 重複投資の解消ニーズ | 各事業部で類似機能が存在 | 該当 |
| 技術シナジー追求 | ビール酵母→健康食品、水科学→全事業 | 該当 |
| 機能別組織化検討 | 研究開発、マーケティングの集約可能性 | 検討余地あり |

### 3.2 軸別詳細評価

#### 【軸1】VL1統一可能性: 1/3点

**評価の焦点**: 全事業で共通する「最上位価値」を定義できるか？

| スコア | 基準 | ケイパビリティ統合型での状況 |
|--------|------|----------------------------|
| 0点 | 全く異なる価値訴求 | - |
| **1点** | **抽象化すれば共通可能** | **「技術力による顧客満足」という機能軸** |
| 2点 | 自然に共通価値を設定可能 | - |
| 3点 | 既に共通価値が企業理念として浸透 | - |

**根拠**:

```
【問題点】ケイパビリティ統合型ではVL1が「機能」起点になりやすい

    ❌ 機能軸VL1（自然に導出される）
    ┌─────────────────────────────────────────┐
    │ 「優れた技術力で高品質な製品を提供する」 │
    │                                         │
    │ → 技術力（HOW）が起点                   │
    │ → 顧客価値（WHY）が後付け               │
    └─────────────────────────────────────────┘

    ✅ 価値軸VL1（本来あるべき姿）
    ┌─────────────────────────────────────────┐
    │ 「水と生きる」                           │
    │ 人と自然と響きあい、豊かな生活文化を創造│
    │                                         │
    │ → 顧客・社会価値（WHY）が起点           │
    │ → 技術力はそれを実現する手段（HOW）     │
    └─────────────────────────────────────────┘

【サントリーの実態】
・「水と生きる」は強力なVL1候補
・しかしケイパビリティ統合型で設計すると
  「研究開発力」「品質力」が前面に出て
  顧客価値が背景に退く
```

#### 【軸2】顧客価値ジャーニー共通性: 1/3点

**評価の焦点**: 顧客の「価値実感プロセス」（VS0→VS7）が事業間で共通か？

| スコア | 基準 | ケイパビリティ統合型での状況 |
|--------|------|----------------------------|
| 0点 | 全く異なるジャーニー | - |
| **1点** | **部分的に共通** | **機能視点で統合するとジャーニーが断片化** |
| 2点 | 大半が類似構造 | - |
| 3点 | ほぼ同一構造 | - |

**根拠**:

```
【問題点】ケイパビリティ統合型は「機能」でVSを設計するため
         「顧客のジャーニー」が断片化しやすい

    顧客ジャーニー: VS0 → VS1 → VS2 → VS3 → VS4 → VS5 → VS6 → VS7
                    │      │      │      │      │      │      │      │
    ケイパビリティ: 経営   R&D   R&D   マーケ  SCM   品質   IT    運営
                   企画    ↓     ↓     ↓      ↓     ↓     ↓     ↓
                           └──────┴──────┴──────┴──────┴──────┴──────┘
                                顧客は「たらい回し」状態

【サントリー固有の課題】
・酒類: 20歳以上の成人が「味わい・嗜好」を求めるジャーニー
・飲料: 全世代が「リフレッシュ・健康」を求めるジャーニー
・健康食品: 健康課題を持つ層が「継続・効果実感」を求めるジャーニー

→ ケイパビリティ統合型では、これらを「研究開発」「マーケティング」
  という機能で横断的に扱おうとするため、各事業の顧客ジャーニー固有性が失われる
```

#### 【軸3】価値創出ケイパビリティの共通性: 3/3点 ★最大強み

**評価の焦点**: VL1実現に必要な「能力」がどの程度共通か？

| スコア | 基準 | ケイパビリティ統合型での状況 |
|--------|------|----------------------------|
| 0点 | 完全に異なる能力セット | - |

#### 【軸4】価値トレーサビリティの複雑度: 1/3点

**評価の焦点**: Operation→VS→VL1への追跡がどの程度複雑になるか？

| スコア | 基準 | ケイパビリティ統合型での状況 |
|--------|------|----------------------------|
| 0点 | シンプルに追跡可能 | - |
| **1点** | **複雑で変換レイヤーが必要** | **機能→価値の変換が常に必要** |
| 2点 | 複雑で分割管理が必要 | - |
| 3点 | 極めて複雑で完全分離が必要 | - |

**根拠**:

```
【問題点】機能起点の活動を価値に変換するレイヤーが必要

    ケイパビリティ統合型のトレーサビリティ:

    Operation（機能活動）
        │ 「酵母研究を行った」
        ▼
    機能指標
        │ 「特許出願10件、論文5本」
        ▼
    【変換レイヤー】必要 ← ここが複雑
        │ 「それが顧客価値にどう貢献したか？」
        ▼
    VS遷移
        │ 「VS2（選好形成）の遷移率に貢献」
        ▼
    VL1貢献
        │ 「豊かな生活文化の創造」

    vs. 価値軸のトレーサビリティ:

    Operation（価値活動）
        │ 「山崎蒸留所見学会を開催した」
        ▼
    VS遷移
        │ 「VS5（価値実感）→VS6（継続）の遷移率改善」
        ▼
    VL1貢献
        │ 「豊かな生活文化の創造」への直接貢献

→ ケイパビリティ統合型は「変換レイヤー」が常に必要で複雑
```

#### 【軸5】価値貢献評価の統一可能性: 1/3点

**評価の焦点**: 全事業を同一基準で価値貢献度評価できるか？

| スコア | 基準 | ケイパビリティ統合型での状況 |
|--------|------|----------------------------|
| 0点 | 完全に異なる評価軸が必要 | - |
| **1点** | **部分的に共通評価可能** | **機能効率と価値貢献の二重評価が必要** |
| 2点 | 大半を共通評価可能 | - |
| 3点 | 同一評価体系で運用可能 | - |

**根拠**:

```
【問題点】機能効率と価値貢献の二重評価が必要

    ケイパビリティの評価体系:

    ┌─────────────────────────────────────────────────────────────────┐
    │                  二重評価の複雑さ                                │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  【評価軸1: 機能効率】                                          │
    │  ・研究開発: 開発サイクル、特許出願数、研究効率                 │
    │  ・物流: 配送コスト、リードタイム                               │
    │  ・マーケティング: 広告効率、リーチ数                           │
    │                                                                 │
    │  【評価軸2: 価値貢献】                                          │

    │  ・VL1への最終貢献                                              │
    │                                                                 │
    │  【二重評価の調整問題】                                         │
    │  ・R&Dが「機能効率は高い（特許多数）」が                        │

    │  ・逆に「機能効率は低い」が「一つのヒット商品で価値貢献大」     │
    │    の場合の評価                                                 │
    │                                                                 │
    │  → ケイパビリティ統合型では常にこの調整が必要                   │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘
```

## 4. 強み・弱み分析

### 4.1 ケイパビリティ統合型の強み

| 強み | サントリーでの具体例 | 効果 |
|------|---------------------|------|
| **技術シナジー最大化** | ビール酵母→健康食品、水科学→全事業 | 開発効率向上、差別化強化 |
| **重複投資削減** | 研究開発センター統合、IT基盤統合 | コスト30%削減可能 |
| **専門性深化** | 発酵研究の世界トップレベル維持 | 競争優位の源泉 |
| **スケールメリット** | 調達統合、物流統合 | 調達コスト改善 |
| **ナレッジ共有** | 技術知見の横展開 | イノベーション加速 |

### 4.2 ケイパビリティ統合型の弱み

| 弱み | サントリーでの具体例 | リスク |
|------|---------------------|--------|
| **顧客視点の希薄化** | 機能組織が「顧客」より「技術」を見る | 市場感度低下 |

| **価値追跡の困難** | 機能活動→顧客価値の因果が見えにくい | 投資判断の歪み |
