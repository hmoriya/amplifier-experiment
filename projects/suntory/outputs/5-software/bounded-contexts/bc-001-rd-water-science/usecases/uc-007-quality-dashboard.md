# UseCase: uc-007 品質ダッシュボード [QualityDashboard]

## 概要

| 項目 | 値 |
|------|-----|
| ID | uc-007 |
| 日本語名 | 品質ダッシュボード |
| 英語名 | QualityDashboard |
| 定数名 | QUALITY_DASHBOARD |
| 関連Operation | op-001, op-002 |
| 関連Page | page-007 品質ダッシュボード |
| ステータス | Draft |

## アクター

- **主アクター**: 品質管理責任者、経営層

## 概要説明

品質管理責任者や経営層が水質検査の状況、品質トレンド、KPIを
ダッシュボードで確認する。異常検知時のアラートも表示する。

## 事前条件

- ユーザーがシステムにログイン済みであること
- 閲覧権限を持っていること

## 事後条件

- なし（参照のみ）

## 基本フロー

1. ユーザーは「品質ダッシュボード」を開く
2. システムはダッシュボードを表示する
3. ユーザーはサマリーKPIを確認する
   - 検査合格率
   - 月間検査件数
   - 未処理アラート数
4. ユーザーは期間・製品カテゴリでフィルタを設定する
5. システムはフィルタに応じたデータを表示する
6. ユーザーはトレンドグラフを確認する
7. ユーザーはアラート一覧を確認する
8. ユーザーは詳細を確認したい項目をクリックする
9. システムは詳細画面へ遷移する

## 代替フロー

### ALT-1: レポート出力

- 発生条件: データをエクスポートしたい場合
- フロー:
  1. ユーザーは「レポート出力」をクリックする
  2. システムは出力形式（PDF/Excel）を選択させる
  3. システムはレポートを生成しダウンロード可能にする

## 例外フロー

### EXC-1: データなし

- 発生条件: 選択した期間にデータが存在しない
- フロー:
  1. システムは「データがありません」と表示する
  2. ユーザーは期間を変更する

## ビジネスルール

| ルールID | ルール | 説明 |
|----------|--------|------|
| BR-001 | 権限制御 | ダッシュボードは権限に応じた範囲のみ表示 |
| BR-002 | リアルタイム | データは5分間隔で自動更新 |
| BR-003 | アラート表示 | 未対応アラートは常に表示 |
| BR-004 | 履歴保持 | 過去5年分のデータを参照可能 |

## UI要件

- KPIはカード形式で大きく表示
- トレンドは折れ線グラフ
- アラートは重要度で色分け
- フィルタはドロップダウン+日付ピッカー
- レスポンシブ対応（タブレット可）

## 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | 初期表示3秒以内、フィルタ反映1秒以内 |
| セキュリティ | 権限に応じたデータアクセス制御 |
| 可用性 | 24時間365日99.5%稼働 |
