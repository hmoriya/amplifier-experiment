## 1. サブドメイン設計方針

### 1.1 設計原則

```
┌─────────────────────────────────────────────────────────────────────┐
│                    サブドメイン設計原則                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【凝集度優先】                                                      │
│  ├─ 関連する機能は同一サブドメインに集約                            │
│  ├─ ビジネス概念の整合性を維持                                      │
│  └─ 変更理由が同じものをグループ化                                  │
│                                                                     │
│  【疎結合設計】                                                      │
│  ├─ サブドメイン間の依存を最小化                                    │
│  ├─ 明確なインターフェースで連携                                    │
│  └─ 独立した進化が可能                                              │
│                                                                     │
│  【ビジネス境界尊重】                                                │
│  ├─ 組織構造との整合性                                              │
│  ├─ 意思決定権限の境界と一致                                        │
│  └─ ドメインエキスパートの責任範囲に対応                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 命名規則

| レベル | 形式 | 例 |
|--------|------|-----|
| CL1ドメイン | `{ドメイン名}` | R&D基盤研究 |
| CL2サブドメイン | `{ドメイン}.{サブドメイン}` | R&D.水科学 |
| CL3コンテキスト | `{ドメイン}.{サブドメイン}.{コンテキスト}` | R&D.水科学.水質分析 |
