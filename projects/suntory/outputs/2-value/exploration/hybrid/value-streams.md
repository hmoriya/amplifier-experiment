# バリューストリーム（ハイブリッド軸）

## 概要

ハイブリッド分解では、**価値軸の統合VStrと事業部軸のVStrを融合**して構築します。

⚠️ **重要**: ハイブリッド軸でも、バリューストリームは価値定義ごとに1本です。「共通基盤層」と「事業差別化層」は別々のVStrではなく、1本のVStr内での役割分担です。

### アンチパターン（避けるべき設計）

| ❌ 誤った設計 | ✅ 正しい設計 |
|-------------|-------------|
| 機能別に複数の共通VStr作成 | 1本の統合VStr + 事業部別VStrの融合 |
| 製品別にVStr作成 | 事業部レベル（飲料、酒類、その他）で各1本 |
| 共通基盤と事業別を完全分離 | 各VSで連携・融合する統合設計 |

## 統合バリューストリーム（HD統括）

**統括**: サントリーホールディングス

**目的**: 全事業に価値を提供する共通基盤の構築・運営

## ハイブリッド軸の特徴

### メリット

1. **シナジーと独自性の両立**: 共通基盤でシナジー、事業別で差別化
2. **投資判断の明確化**: 共通基盤投資 vs 事業差別化投資を分離
3. **サントリーの実態適合**: HD＋事業会社構造に整合
4. **柔軟性**: 共通と事業別で異なる意思決定スピード

### デメリット

1. **構造の複雑性**: 融合構造の管理コスト
2. **境界の曖昧さ**: 統合VStrと事業VStrの切り分けが難しい場合
3. **調整コスト**: HD・事業会社間の調整が必要

## 3軸比較サマリー

```
┌──────────────┬─────────────┬─────────────┬─────────────────────────────────┐
│     軸       │ VStr数      │ 特徴        │ 適用シナリオ                     │
├──────────────┼─────────────┼─────────────┼─────────────────────────────────┤
│ 価値軸       │ 1本         │ 統合・一貫  │ 強力なブランド統一が必要な場合    │
├──────────────┼─────────────┼─────────────┼─────────────────────────────────┤
│ 事業部軸     │ 3本         │ 独立・最適化│ 事業間の独立性が高い場合         │
├──────────────┼─────────────┼─────────────┼─────────────────────────────────┤
│ ハイブリッド │ 4本         │ 融合・柔軟  │ 統合と独立のバランスが必要な場合  │
│              │ (統合1+事業3)│             │                                 │
└──────────────┴─────────────┴─────────────┴─────────────────────────────────┘

核心原則: 1価値定義 = 1バリューストリーム
         VS0-VS7は直列ステージ（並列ストリームではない）
```

| 観点 | 価値軸 | 事業部軸 | ハイブリッド軸 |
|------|-------|---------|--------------|
| VStr数 | 1本 | 3本 | 1本（統合）+ 3本（事業別）= 4本 |
| シナジー | 最大 | 最小 | 中程度 |
| 事業特性反映 | 最小 | 最大 | 中程度 |
| 管理複雑性 | 低 | 中 | 高 |
| サントリー適合 | 中 | 中 | 高 |
