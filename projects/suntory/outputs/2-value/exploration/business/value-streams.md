# バリューストリーム（事業部軸）

## 概要

事業部軸分解では、**各事業セグメントごとに1本のバリューストリーム**を構築します。

⚠️ **重要**: バリューストリームは事業セグメントごとに1本です。製品カテゴリー別ではありません。VS0-VS7は並列ストリームではなく、1本のストリーム内の「ステージ」（直列）です。

### アンチパターン（避けるべき設計）

| ❌ 誤った設計 | ✅ 正しい設計 |
|-------------|-------------|
| 製品別にVStr作成（天然水VStr、茶VStr、ウイスキーVStr等） | 事業部レベル（飲料VStr、酒類VStr）で各1本 |
| 11+の製品カテゴリー別VStr | 3本（飲料、酒類、その他） |
| VSを並列ストリーム扱い | VSはストリーム内のステージ（直列） |

## バリューストリーム体系

```
VL1: 豊かな生活文化の創造
│
├── VStr-B1: 飲料バリューストリーム（1本）
│   └── VS0 → VS1 → VS2 → VS3 → VS4 → VS5 → VS6 → VS7
│
├── VStr-B2: 酒類バリューストリーム（1本）
│   └── VS0 → VS1 → VS2 → VS3 → VS4 → VS5 → VS6 → VS7
│
└── VStr-B3: その他バリューストリーム（1本）
    └── VS0 → VS1 → VS2 → VS3 → VS4 → VS5 → VS6 → VS7
```

## VStr-B1: 飲料バリューストリーム

**責任**: サントリー食品インターナショナル

**目的**: 飲料事業における価値創造から持続的改善までの統合フロー

| VS | ステージ | 活動内容 | 対象製品カテゴリー |

| VS4 | 製造・流通 | 製造、物流、自販機、コンビニ配荷 | 全カテゴリー共通 |
| VS5 | 品質保証・顧客サポート | 品質管理、お客様対応 | 全カテゴリー共通 |

## 事業部軸の特徴

### メリット

1. **事業特性の明確化**: 飲料と酒類の違いを反映
2. **P/L責任との整合**: 事業部ごとの収益管理に適合
3. **専門性の深化**: 各事業の専門知識を活かせる
4. **柔軟性**: 事業ごとの迅速な意思決定

### デメリット

1. **サイロ化リスク**: 事業間連携が弱くなる可能性
2. **重複投資**: 共通機能の重複
3. **シナジー損失**: 事業間学習が進みにくい

### 価値軸との違い

| 観点 | 価値軸 | 事業部軸 |
|------|-------|---------|
| VStr数 | 1本（全社統合） | 3本（事業部別） |
| 抽象度 | 高い（統一重視） | 中程度（事業特性反映） |
| 最適化視点 | 全社最適 | 事業別最適 |
| 意思決定 | HD主導 | 事業会社主導 |
