# サントリー Value Stream設計 - 5軸パターン分析レポート

**作成日**: 2025-12-09
**分析対象**: サントリーホールディングス株式会社
**フレームワーク**: Parasol V5 Lite

---

## エグゼクティブサマリー

本レポートは、サントリーを題材に5つの軸パターンでValue Stream（VStr）設計を並列分析した結果をまとめたものである。各パターンの特徴、適用可能性、推奨組み合わせを提示する。

### 5軸パターン比較一覧

| 軸パターン | 主な特徴 | 適用推奨度 | 最適ケース |
|-----------|---------|-----------|-----------|
| **価値軸** | 「水と生きる」を全VSに貫通 | ★★★★☆ | 事業横断シナジー重視 |
| **事業部軸** | 酒類・飲料・健康食品を独立設計 | ★★★★☆ | 事業部オーナーシップ重視 |
| **ケイパビリティ軸** | 研究開発・ブランドMgt等を統合 | ★★★★☆ | 機能効率化・重複排除重視 |
| **フュージョン型** | 共通層(VS0,6,7)+事業層(VS1-5) | ★★★★☆ | バランス・段階適用重視 |
| **多段VStr** | 顧客+従業員+パートナーの3層 | ★★★★☆ | ESG・ステークホルダー重視 |

---

## 1. 価値軸パターン

### 1.1 概要

「水と生きる」を企業理念としてVS0に据え、全事業を顧客への価値創造フローという単一視点で捉えるアプローチ。

### 1.2 VS0-VS7 定義

| ステージ | 名称 | 定義 | 「水と生きる」との接続 |
|---------|------|------|----------------------|
| **VS0** | ビジョン・戦略立案 | 「水と生きる」を核とした企業理念、長期経営戦略 | **根幹**: 水資源保全・自然との共生が全ての出発点 |
| **VS1** | 市場機会発見 | 消費者ニーズ調査、健康トレンド分析 | 水・自然由来の価値への消費者期待を起点に |
| **VS2** | 製品開発 | 発酵技術、抽出技術、原料研究 | 水の品質・自然素材の活用が競争優位の源泉 |
| **VS3** | ブランディング | ブランド構築、広告宣伝 | 「水と生きる」ストーリーによる情緒的価値 |
| **VS4** | 販売・流通 | 調達、製造、物流、販売チャネル | 水源地保全から消費者への一貫した品質管理 |
| **VS5** | 品質保証 | 品質管理、顧客対応 | 水の安全性・自然由来の信頼を担保 |
| **VS6** | データ活用・DX | 顧客データ統合、需要予測 | 水資源モニタリング、消費行動分析 |
| **VS7** | オペレーション | 製造効率化、持続可能運営 | 水使用量削減、循環型オペレーション |

### 1.4 強み・弱み評価

| 項目 | 評価 | 説明 |

| **技術シナジー** | ◎ | 発酵・抽出・水処理の基盤技術共有 |
| **ブランド統一** | ◎ | 「自然由来」で一貫したストーリー |
| **事業部独立性との摩擦** | △ | 商習慣の違いで調整コスト発生 |
| **抽象度の高さ** | △ | 現場オペレーションとの乖離リスク |

## 2. 事業部軸パターン

### 2.1 概要

既存の事業部構造を尊重し、各事業部ごとに独立したVS1-VS7を定義。全社共通はVS0のみ。

### 2.2 全体構造

```

│                       全社戦略層 (VS0)                           │
│  ・グループ経営戦略  ・ブランドポートフォリオ  ・サステナビリティ    │
└─────────────────────────────────────────────────────────────────┘
              │                    │                    │
              ▼                    ▼                    ▼
    ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐

    │   VS1-VS7       │  │   VS1-VS7       │  │   VS1-VS7       │
    │   独自定義       │  │   独自定義       │  │   独自定義       │
```

## 3. ケイパビリティ軸パターン

### 3.1 概要

事業横断で共通する「能力（ケーパビリティ）」を中心に設計。機能別組織やシェアードサービス化を志向。

### 3.2 ケイパビリティ層構造

```
┌──────────────────────────────────────────────────────────────────────┐
│                     【戦略系ケイパビリティ層】                          │
│    経営企画・戦略  │  M&A・アライアンス  │  サステナビリティ             │
│                         (VS0 対応)                                   │
└──────────────────────────────────────────────────────────────────────┘
                                 │
┌──────────────────────────────────────────────────────────────────────┐
│                     【創造系ケイパビリティ層】                          │
│  研究開発センター  │  商品開発  │  パッケージ開発                       │
│                         (VS2 対応)                                   │
└──────────────────────────────────────────────────────────────────────┘
                                 │
┌──────────────────────────────────────────────────────────────────────┐
│                     【市場系ケイパビリティ層】                          │
│  ブランドマネジメント  │  営業・チャネル  │  デジタルマーケ              │
│                     (VS1, VS3, VS4 対応)                             │
└──────────────────────────────────────────────────────────────────────┘
                                 │
┌──────────────────────────────────────────────────────────────────────┐
│                     【運営系ケイパビリティ層】                          │
│  サプライチェーン  │  製造・品質管理  │  自販機運営                      │
│                      (VS4, VS5 対応)                                 │

│   デジタル基盤  │  人事・組織  │  財務・経理  │  法務・コンプライアンス   │
│                      (VS6, VS7 対応)                                 │
└──────────────────────────────────────────────────────────────────────┘
          │                  │                  │                  │

```

### 3.3 統合判断マトリクス

| ケイパビリティ | 類似度 | 規模 | 判断 | 理由 |
|--------------|-------|------|------|------|
| 研究開発センター | 高 | 大 | **統合◎** | 発酵・醸造技術は共通、健康科学は全事業展開可能 |
| ブランドマネジメント | 高 | 大 | **統合◎** | 「やってみなはれ」「水と生きる」は全社統一 |
| サプライチェーン | 高 | 大 | **統合◎** | 製造・物流の最適化は全事業共通課題 |
| デジタル基盤 | 高 | 中 | **統合○** | データ・IT基盤は全社共通化でスケールメリット |
| 営業・チャネル | 中 | 大 | **部分統合** | 量販店・CVSは統合、飲食店・自販機は事業固有 |
| 酒類規制対応 | 低 | 中 | **事業固有** | 酒類事業固有の法規制対応 |
| 自販機運営 | 低 | 大 | **事業固有** | 飲料事業固有のオペレーション |

### 3.4 強み・弱み評価

| 項目 | 評価 | 説明 |
|------|------|------|
| **重複投資削減** | ◎ | 研究開発・IT等の一元化 |
| **専門性向上** | ◎ | 機能別に深い専門知識蓄積 |
| **シナジー創出** | ◎ | ビール製造技術→ノンアルコール応用 |
| **事業理解低下** | △ | 機能組織が現場から乖離 |
| **調整コスト増大** | △ | 事業と機能間の調整複雑化 |

## 4. フュージョン型パターン

### 4.1 概要

価値軸と事業部軸のハイブリッド。共通層（VS0, VS6, VS7）で骨格を作り、事業層（VS1-VS5）で事業固有の拡張を許容。

### 4.2 層構造

```
┌────────────────────────────────────────────────────────────────────┐
│                        【共通層】                                  │
│  ┌──────────────┬──────────────────┬────────────────────┐         │
│  │ VS0: 戦略    │ VS6: データ基盤  │ VS7: オペレーション │         │
│  │ ・ブランド   │ ・顧客統合DB     │ ・品質管理基準     │         │
│  │  ポートフォリオ│ ・AI/ML基盤     │ ・サステナビリティ  │         │
│  │ ・投資配分   │ ・分析BIツール   │ ・調達プラットフォーム│        │
│  └──────────────┴──────────────────┴────────────────────┘         │
└────────────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌───────────────┐     ┌───────────────┐     ┌───────────────┐
│【酒類事業】    │     │【飲料事業】    │     │【健康食品事業】│
│ VS1:発酵技術  │     │ VS1:健康機能  │     │ VS1:機能性研究│
│ VS2:成熟市場  │     │ VS2:ファミリー │     │ VS2:エビデンス│
│ VS3:酒販店網  │     │ VS3:自販機    │     │ VS3:EC直販   │
│ VS4:蒸溜所ツアー│     │ VS4:工場見学  │     │ VS4:健康相談 │
│ VS5:熟成管理  │     │ VS5:リサイクル │     │ VS5:継続支援 │
└───────────────┘     └───────────────┘     └───────────────┘
                              │
┌────────────────────────────────────────────────────────────────────┐
│                     【連携層】インターフェース                      │
│   顧客ID統合  │  ブランド横断プロモ  │  原料・資材共同調達          │
└────────────────────────────────────────────────────────────────────┘
```

### 4.3 共通化判断

| 機能領域 | 判断 | 理由 |
|---------|------|------|
| ブランド管理 | **共通化** | 「水と生きる」統一メッセージ |
| 品質管理基準 | **共通化** | 食品安全は全社統一必須 |
| 顧客データ基盤 | **共通化** | 一元管理でシナジー創出 |
| 発酵・蒸溜技術 | **固有** | 競争優位の源泉 |
| 自販機オペレーション | **固有** | 事業特性が大きく異なる |
| 機能性エビデンス研究 | **固有** | 顧客接点が独自 |

### 4.4 導入ステップ

| Phase | 期間 | 内容 |
|-------|------|------|
| Phase 1 | 3-6ヶ月 | 共通層の定義、ガバナンス体制構築 |
| Phase 2 | 6-9ヶ月 | 飲料事業でパイロット検証 |
| Phase 3 | 9-18ヶ月 | 酒類・健康食品へ展開 |
| Phase 4 | 18-24ヶ月 | 連携層の強化、クロスセル仕組み化 |

### 4.5 強み・弱み評価

| 項目 | 評価 | 説明 |
|------|------|------|
| **バランス** | ◎ | 酒類の伝統と飲料の革新性を両立 |
| **現実的** | ◎ | 既存の事業会社体制を活かせる |
| **シナジー** | ◎ | 「水」という共通資源で横断価値創出 |
| **設計複雑** | △ | 境界設計が困難 |
| **ガバナンス** | △ | 持株会社と事業会社の二重管理 |

## 5. 多段VStrパターン

### 5.1 概要

顧客VStrを主軸としつつ、従業員VStr・パートナーVStrを階層的に構成。ESG/ステークホルダー資本主義への対応。

### 5.2 3層VStr定義

#### 顧客向けVStr（VS: Value Stage）- Core Domain

| Stage | 状態 | サントリー固有定義 |
|-------|------|-------------------|
| VS0 | 潜在ニーズ | 「喉が渇いた」「リフレッシュしたい」 |
| VS1 | 認知到達 | サントリー製品の存在を知る |
| VS2 | 選好形成 | 「サントリーを選びたい」と思う |
| VS3 | 意思決定 | 「この製品を買う」と決める |
| VS4 | 価値獲得 | 製品を購入し手に入れる |
| VS5 | 価値実感 | 「美味しい」を実感 |
| VS6 | 価値継続 | リピート購入、定番化 |
| VS7 | 価値共創 | SNSでシェア、友人に推奨 |

#### 従業員向けVStr（ES: Employee Stage）- Supporting Domain

| Stage | 状態 | サントリー固有定義 |
|-------|------|-------------------|
| ES0 | 就職潜在 | 飲料・酒類業界で働く可能性を意識していない |
| ES1 | 認知到達 | 「やってみなはれ」文化を知る |
| ES2 | 選考意向 | サントリーで働きたいと思う |
| ES3 | 入社決定 | 内定承諾、入社決定 |
| ES4 | 活躍開始 | 配属後、業務で価値を発揮 |
| ES5 | 成長実感 | スキル向上、やりがいを実感 |
| ES6 | 定着継続 | 長期的にキャリア構築 |
| ES7 | 組織共創 | 後輩育成、文化継承 |

#### パートナー向けVStr（PS: Partner Stage）- Supporting Domain

| Stage | 状態 | サントリー固有定義 |
|-------|------|-------------------|
| PS0 | 協業潜在 | 取引可能性を認識していない |
| PS1 | 認知到達 | サントリーとの取引可能性を知る |
| PS2 | 提携意向 | 取引・協業したいと思う |
| PS3 | 契約決定 | 取引契約を締結 |
| PS4 | 協業開始 | 実際の取引を開始 |
| PS5 | 価値実感 | 協業から利益・成長を実感 |
| PS6 | 拡大継続 | 取引規模拡大 |
| PS7 | 戦略共創 | 戦略的パートナーとして共創 |

## 6. 総合評価と推奨

### 6.1 軸パターン選択マトリクス

| 経営課題 | 推奨パターン | 理由 |
|---------|------------|------|
| 事業横断シナジー最大化 | **価値軸** | 「水と生きる」で全事業を貫通 |
| 事業部オーナーシップ維持 | **事業部軸** | 既存組織との整合性 |
| 機能効率化・コスト削減 | **ケイパビリティ軸** | 重複排除、専門性向上 |
| バランスと段階適用 | **フュージョン型** | 共通/固有の最適配分 |
| ESG・従業員重視 | **多段VStr** | ステークホルダー体験設計 |

### 6.2 サントリーへの推奨組み合わせ

**フュージョン型 + 多段VStr** の組み合わせを推奨：

```
┌────────────────────────────────────────────────────────────┐
│               共通層（Generic + VS0/6/7）                   │
│  戦略・データ基盤・オペレーション・認証・監査              │
└────────────────────────────────────────────────────────────┘
        │               │               │
   ┌────┴────┐    ┌────┴────┐    ┌────┴────┐
   │ 酒類     │    │ 飲料     │    │健康食品  │
   │顧客VStr │    │顧客VStr │    │顧客VStr │
   │従業員VStr│    │従業員VStr│    │従業員VStr│
   │パートナー│    │パートナー│    │パートナー│
   └─────────┘    └─────────┘    └─────────┘
```

**推奨理由**：
1. **「水と生きる」の統一性**: 共通層で企業理念を全事業に貫通
2. **事業固有性の尊重**: 事業層で酒類・飲料・健康食品の特性を維持
3. **ESG対応**: 多段VStrで従業員・パートナー体験を明示的に設計
4. **段階適用**: Phase導入でリスク分散しながら統合推進

### 6.3 実装ロードマップ（推奨）

| Phase | 期間 | 対象 | 内容 |
|-------|------|------|------|
| Phase 1 | 0-9ヶ月 | 共通層 + 飲料事業顧客VStr | 基盤構築、パイロット |
| Phase 2 | 9-15ヶ月 | 酒類・健康食品顧客VStr | 横展開 |
| Phase 3 | 15-21ヶ月 | 従業員VStr（全事業） | EX強化 |
| Phase 4 | 21-27ヶ月 | パートナーVStr（全事業） | サプライチェーン最適化 |
