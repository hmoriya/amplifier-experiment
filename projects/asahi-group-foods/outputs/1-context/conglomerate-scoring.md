# アサヒグループ食品 コングロマリット判定スコアリング

**作成日**: 2025-12-11
**Phase**: 1 - Context
**目的**: 軸パターン自動推奨の入力データ

---

## スコアリング概要

**15点満点のコングロマリット度スコア**を算出し、軸パターン推奨の根拠とする。

---

## 1. 事業ドメイン分散度（0-3点）

| 評価基準 | 点数 | アサヒグループ食品の状況 |
|---------|------|------------------------|
| 単一ドメイン | 0 | - |
| 関連多角化（2-3事業） | 1 | - |
| 関連多角化（4-5事業） | 2 | **該当**: 菓子・FD・ベビー・サプリ・原料（5事業） |
| 非関連多角化 | 3 | - |

**スコア: 2点**

**根拠**:
- 全て食品・ヘルスケア領域内（関連多角化）
- ただし5つの異なる事業セグメントが存在
- 非関連事業（金融、不動産等）は持たない

---

## 2. 顧客セグメント分散度（0-3点）

| 評価基準 | 点数 | アサヒグループ食品の状況 |
|---------|------|------------------------|
| 単一セグメント | 0 | - |
| 2-3セグメント | 1 | - |
| 4-5セグメント | 2 | - |
| 6セグメント以上 | 3 | **該当**: ビジネスパーソン、主婦、乳幼児保護者、高齢者、健康志向層、法人（6+） |

**スコア: 3点**

**根拠**:
- BtoC: 5つ以上の異なるペルソナ
- BtoB: 食品メーカー、外食チェーン
- 各セグメントで購買動機・チャネル・競合が異なる

---

## 3. 競合構造の分散度（0-3点）

| 評価基準 | 点数 | アサヒグループ食品の状況 |
|---------|------|------------------------|
| 同一競合群 | 0 | - |
| 一部重複あり | 1 | - |
| 大半が異なる | 2 | **該当** |
| 完全に異なる | 3 | - |

**スコア: 2点**

**根拠**:
- 菓子: クラシエ、モンデリーズ
- FD: マルコメ、永谷園
- ベビー: キユーピー、ピジョン
- サプリ: DHC、ファンケル
- 一部ドラッグストアチャネルで重複するが、直接競合はほぼ異なる

---

## 4. 技術基盤の共通性（0-3点）※逆転スコア

| 評価基準 | 点数 | アサヒグループ食品の状況 |
|---------|------|------------------------|
| 完全に共通 | 0 | - |
| 大半が共通 | 1 | - |
| 一部共通 | 2 | **該当**: 発酵・FD技術は共通、菓子は別系統 |
| 完全に異なる | 3 | - |

**スコア: 2点**

**根拠**:
- 共通技術: 発酵・酵母技術（サプリ↔原料）、FD技術（FD食品↔原料）
- 別系統: 菓子製造技術
- 技術シナジーは存在するが、全事業統一ではない

---

## 5. ブランドポートフォリオ構造（0-3点）

| 評価基準 | 点数 | アサヒグループ食品の状況 |
|---------|------|------------------------|
| 単一ブランド | 0 | - |
| マスターブランド+サブ | 1 | - |
| 複数独立ブランド（一部連携） | 2 | **該当** |
| 完全独立ブランド群 | 3 | - |

**スコア: 2点**

**根拠**:
- 各事業で独立ブランド: ミンティア、アマノフーズ、和光堂、ディアナチュラ、エビオス
- 「アサヒ」の冠は一部商品のみ
- ブランド間の相互送客は限定的
- ただし「アサヒグループ」としての信頼性は共通資産

---

---

## スコア解釈ガイド

| スコア範囲 | 判定 | 推奨パターン |
|-----------|------|-------------|
| 0-4点 | 低（単一事業型） | 価値軸パターン |
| 5-8点 | 中（関連多角化） | フュージョン型 または ケイパビリティ統合型 |
| 9-12点 | 中〜高 | **事業部軸** または **フュージョン型** |
| 13-15点 | 高（非関連多角化） | 事業部軸パターン |

---

## 軸パターン推奨

### 第1候補: フュージョン型パターン

**推奨理由**:
1. 技術基盤に一定の共通性あり（発酵・FD技術）
2. 顧客セグメントは分散しているが、「健康・食」という上位価値は共通
3. 親会社（アサヒグループジャパン）との連携でGeneric Layerが既存

**適用イメージ**:
```
┌─────────────────────────────────────────────────────────┐
│                    共通バリューストリーム                 │
│    VS0: 「おいしさと健康」統合戦略                       │
│    VS6: グループ共通データ基盤                          │
│    VS7: 品質保証・サステナビリティ                      │
└─────────────────────────────────────────────────────────┘
                          │
          ┌───────────────┼───────────────┐
          ▼               ▼               ▼
    ┌──────────┐    ┌──────────┐    ┌──────────┐
    │ 菓子・FD │    │ベビー    │    │サプリ    │
    │ VS1-VS5  │    │VS1-VS5   │    │VS1-VS5   │
    └──────────┘    └──────────┘    └──────────┘
```

### 第2候補: 事業部軸パターン

**推奨理由**:
1. 顧客セグメントが完全に異なる
2. 競合構造が事業部ごとに独立
3. ブランドが独立して認知されている

**適用イメージ**:
```
┌──────────────────────────────────────────────────────────┐
│                   Generic Layer                          │
│         認証 │ 経理 │ IT基盤 │ 品質管理                  │
└──────────────────────────────────────────────────────────┘
        │           │           │           │
   ┌────┴────┐ ┌────┴────┐ ┌────┴────┐ ┌────┴────┐
   │ ミンティア│ │アマノ   │ │ 和光堂  │ │ディアナチュラ│
   │ VStr    │ │フーズ   │ │ VStr   │ │ VStr    │
   │ VS0-VS7 │ │VStr    │ │ VS0-VS7│ │ VS0-VS7 │
   └─────────┘ └─────────┘ └─────────┘ └─────────┘
```

---

## 推奨判定結果

| 順位 | パターン | 適合度 | 根拠 |
|------|---------|--------|------|
| **1位** | **フュージョン型** | ★★★★☆ | 技術シナジー活用、共通価値（健康・食）の設計可能 |
| 2位 | 事業部軸 | ★★★☆☆ | 各事業の独立性尊重、シンプルな設計 |
| 3位 | ケイパビリティ統合型 | ★★☆☆☆ | R&D統合は可能だが、顧客接点が分散しすぎ |
| - | 価値軸 | ★☆☆☆☆ | 顧客セグメント分散が大きすぎて不適 |
| - | プラットフォーム軸 | ☆☆☆☆☆ | マーケットプレイス型事業なし |

---

## Phase 2への引継ぎ事項

1. **採用パターン決定**: フュージョン型 vs 事業部軸の最終判断
2. **共通層の定義**: 「おいしさと健康」を戦略的価値として設定可能か
3. **事業部グルーピング**:
   - 案A: 4事業独立（菓子、FD、ベビー・シニア、サプリ）
   - 案B: 2グループ（嗜好品系、健康系）
4. **BtoB事業の位置づけ**: 独立VStr vs Supporting Layer

---

## 補足: 多段VStr適用判定

| VStr | 必要性 | 根拠 |
|------|--------|------|
| 顧客VStr | **必須** | 主軸事業 |
| 従業員VStr | オプション | 経営課題レベルではない |
| パートナーVStr | 不要 | 戦略的エコシステムなし |

**結論**: 単一顧客VStr（または事業部別顧客VStr）+ Generic Layer で開始
