# アサヒグループ食品 価値駆動ベース軸パターン選択スコアリング

**作成日**: 2025-12-12
**Phase**: 1 - Context（Phase 2への橋渡し）
**目的**: 価値駆動アプローチに基づく軸パターン選択の根拠

---

## 概要

本ドキュメントは、パラソルの価値駆動アプローチに基づき、アサヒグループ食品に最適な軸パターンを選択するためのスコアリングです。

**従来のコングロマリット判定との違い**:

| 観点 | 従来（組織・市場視点） | 新基準（価値駆動視点） |
|------|----------------------|----------------------|
| 焦点 | 事業の分散度 | 価値体系の統一可能性 |
| 判断軸 | 組織構造、競合、技術 | VL1統一性、ジャーニー共通性 |
| 目的 | コングロマリット度の測定 | 価値実現に最適な軸の選択 |

---

## 1. 価値駆動ベース評価（主要評価軸）

### 【軸1】VL1統一可能性（0-3点）

**評価の焦点**: 全事業で共通する「最上位価値」を定義できるか？

| スコア | 基準 | アサヒグループ食品の状況 |
|--------|------|------------------------|
| 0点 | 全く異なる価値訴求 | - |
| 1点 | 抽象化すれば共通可能 | - |
| **2点** | **自然に共通価値を設定可能** | **「おいしさと健康」で統一可能** |
| 3点 | 既に共通価値が企業理念として浸透 | - |

**スコア: 2点**

**根拠**:
- 親会社アサヒグループの「期待を超えるおいしさ、楽しい生活文化の創造」と整合
- 全事業（菓子、FD、ベビー、サプリ、原料）が「おいしさ」「健康」のいずれか、または両方を価値訴求
- ただし、企業理念として全社員に浸透しているレベルではない（3点未満）

```
VL1候補: 「おいしさと健康で、人々の毎日を豊かにする」

  ┌─────────────┬─────────────┬─────────────┐
  │ 菓子・FD    │ ベビー・シニア│ サプリメント │
  │             │             │             │
  │ 「おいしさ」│ 「おいしさ」 │ 「健康」    │
  │  主軸      │ ＋安心・安全 │  主軸       │
  └─────────────┴─────────────┴─────────────┘
```

---

### 【軸2】顧客価値ジャーニーの共通性（0-3点）

**評価の焦点**: 顧客の「価値実感プロセス」（VS0→VS7）が事業間で共通か？

| スコア | 基準 | アサヒグループ食品の状況 |
|--------|------|------------------------|
| 0点 | 全く異なるジャーニー | - |
| **1点** | **部分的に共通** | **VS1-3は類似、VS4-7は事業で異なる** |
| 2点 | 大半が類似構造 | - |
| 3点 | ほぼ同一構造 | - |

**スコア: 1点**

**根拠**:

```
【共通部分】VS1-VS3（認知→選好→決定）
・全事業でドラッグストア/スーパー/ECでの認知・選択
・「健康・おいしさ」訴求での選好形成

【異なる部分】VS4-VS7（獲得→実感→継続→共創）
┌─────────────┬─────────────────────────────────┐
│ 事業        │ VS4-VS7の特徴                    │
├─────────────┼─────────────────────────────────┤
│ ミンティア  │ 即時消費、衝動購買、高リピート   │
│ アマノフーズ│ 備蓄消費、計画購買、中リピート   │
│ 和光堂      │ 成長段階で卒業、低継続           │
│ ディアナチュラ│ 継続利用、習慣化、高継続       │
│ BtoB原料    │ 契約ベース、長期関係、法人購買   │
└─────────────┴─────────────────────────────────┘
```

---

### 【軸3】価値創出ケイパビリティの共通性（0-3点）

**評価の焦点**: VL1実現に必要な「能力」がどの程度共通か？

| スコア | 基準 | アサヒグループ食品の状況 |
|--------|------|------------------------|
| 0点 | 完全に異なる能力セット | - |
| 1点 | 一部共通 | - |
| **2点** | **大半が共通** | **発酵・FD技術、品質保証が共通** |
| 3点 | ほぼ同一能力セット | - |

**スコア: 2点**

**根拠**:

```
【共通ケイパビリティ】
・発酵・酵母技術: サプリメント ↔ BtoB原料
・フリーズドライ技術: アマノフーズ ↔ BtoB原料
・品質保証・安全管理: 全事業共通（特にベビー事業が厳格）
・研究開発基盤: 親会社との連携

【事業固有ケイパビリティ】
・菓子製造技術: ミンティア固有
・ベビー・シニア向け栄養設計: 和光堂固有
・機能性表示対応: サプリメント事業

価値創出能力として翻訳:
・「おいしさを保つ能力」→ FD技術
・「健康価値を創出する能力」→ 発酵・栄養設計
・「安心・安全を担保する能力」→ 品質保証
```

---

### 【軸4】価値トレーサビリティの複雑度（0-3点）

**評価の焦点**: Operation→VS→VL1への追跡がどの程度複雑になるか？

| スコア | 基準 | アサヒグループ食品の状況 |
|--------|------|------------------------|
| 0点 | シンプルに追跡可能 | - |
| 1点 | 一部複雑だが管理可能 | - |
| **2点** | **複雑で分割管理が必要** | **5事業あり、中程度に複雑** |
| 3点 | 極めて複雑で完全分離が必要 | - |

**スコア: 2点**

**根拠**:
- 5つの事業セグメント（菓子、FD、ベビー・シニア、サプリ、BtoB原料）
- BtoC 4事業 + BtoB 1事業の混在
- 各事業でKPIが異なる（リピート率、継続率、契約継続率）

```
トレーサビリティの検証例:

【ミンティア事業】
Operation: 店頭POP設置
    ↓ 貢献
VS2遷移率: 選好形成（「ミンティアを選びたい」）
    ↓ 貢献
VL3: 「手軽なリフレッシュ価値」
    ↓ 貢献
VL1: 「おいしさと健康」

→ 追跡可能だが、5事業分の管理は複雑
→ 分割管理（事業層）+共通管理（共通層）が現実的
```

---

### 【軸5】価値貢献評価の統一可能性（0-3点）

**評価の焦点**: 全事業を同一基準で価値貢献度評価できるか？

| スコア | 基準 | アサヒグループ食品の状況 |
|--------|------|------------------------|
| 0点 | 完全に異なる評価軸が必要 | - |
| **1点** | **部分的に共通評価可能** | **VS遷移率は共通、詳細KPIは事業別** |
| 2点 | 大半を共通評価可能 | - |
| 3点 | 同一評価体系で運用可能 | - |

**スコア: 1点**

**根拠**:

```
【共通評価可能な指標】
・VS遷移率（全事業で適用可能）
・NPS（顧客推奨度）
・ブランド認知度

【事業固有の指標】
┌─────────────┬─────────────────────────────────┐
│ 事業        │ 固有KPI                          │
├─────────────┼─────────────────────────────────┤
│ ミンティア  │ 購買頻度、店頭シェア             │
│ アマノフーズ│ ギフト比率、備蓄購入率           │
│ 和光堂      │ 利用月齢、卒業後ブランド想起     │
│ ディアナチュラ│ 継続率、併用率                 │
│ BtoB原料    │ 契約継続率、取引単価             │
└─────────────┴─────────────────────────────────┘

→ 完全統一は困難、フュージョン型での階層化が現実的
```

## 5. 最終推奨

### 推奨パターン: フュージョン型

**価値駆動の観点からの推奨理由**:

1. **VL1統一可能性が高い（2点）**
   - 「おいしさと健康」で全事業を統合可能
   - 共通層で戦略的価値を設計できる

2. **ジャーニー共通性が中程度（1点）**
   - VS1-3は共通化可能 → 共通層で設計
   - VS4-7は事業固有 → 事業層で設計
   - フュージョン型の構造に自然にフィット

3. **価値創出Capが共通（2点）**
   - 発酵・FD技術を共通層で活用
   - 技術シナジーを価値創出に変換

4. **トレーサビリティ管理に適切（2点）**
   - 共通層: VL1→共通VL2の追跡
   - 事業層: VL2→VL3→VSの追跡
   - 階層化で複雑度を管理

5. **評価体系の階層化が可能（1点）**
   - 共通層: VL1貢献度で評価
   - 事業層: VS遷移率+事業固有KPIで評価

### 推奨構造

```
┌─────────────────────────────────────────────────────────┐
│                    共通層（VL1）                        │
│    「おいしさと健康で、人々の毎日を豊かにする」        │
│                                                         │
│    VS0: 統合戦略  VS6: データ基盤  VS7: 品質保証       │
└─────────────────────────────────────────────────────────┘
                          │
          ┌───────────────┼───────────────┐
          ▼               ▼               ▼
    ┌──────────┐    ┌──────────┐    ┌──────────┐
    │ 嗜好品   │    │ 健康・   │    │ BtoB     │
    │ グループ │    │ ライフ   │    │ 原料     │
    │          │    │ グループ │    │          │
    │・ミンティア│    │・ディアナチュラ│    │・酵母原料│
    │・アマノ  │    │・和光堂  │    │・FD原料  │
    │          │    │          │    │          │
    │ VS1-VS5  │    │ VS1-VS5  │    │ VS1-VS5  │
    └──────────┘    └──────────┘    └──────────┘
```

## 6. Phase 2への引継ぎ事項

1. **共通層の詳細定義**
   - VL1「おいしさと健康」の具体化
   - VL2への分解（おいしさ価値領域、健康価値領域）
   - VS0/VS6/VS7の設計

2. **事業グループのVStr設計**
   - 嗜好品グループ（ミンティア、アマノ）のVL3→VS1-VS5
   - 健康・ライフグループ（ディアナチュラ、和光堂）のVL3→VS1-VS5
   - BtoB原料事業のVL3→VS1-VS5

3. **価値貢献度ガバナンスの設計**
   - 共通層のイネーブラー評価方式
   - 事業層の事業KPI+VL1貢献度評価
   - 参照: `_value-contribution-governance.md`
