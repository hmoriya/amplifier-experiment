# フュージョン型における価値貢献度ガバナンス

**作成日**: 2025-12-11
**Phase**: 2 - Value Stream Design
**テーマ**: 共通価値への貢献度をどのようにコントロール・推進するか

---

## 1. 問題の本質

### パラソルの価値駆動設計

パラソルは「価値定義に対して企業活動が同じベクトルになるようにバックキャスティング」する方法論です。

```
【価値駆動のバックキャスティング】

VL1（最上位価値）
    ↓ 「この価値を実現するには？」
VL2（価値グループ）
    ↓ 「この価値グループを実現するには？」
VL3（詳細価値）
    ↓ 「この詳細価値を実現するCapabilityは？」
Capability → Operation → システム
```

### フュージョン型の構造的課題

フュージョン型では「共通層」と「事業層」が分離するため、**価値への貢献度が曖昧になるリスク**があります。

```
┌─────────────────────────────────────────────────────────┐
│                    共通価値: VL1                         │
│              「おいしさと健康」                           │
└─────────────────────────────────────────────────────────┘
        ↓ どのように貢献を測定・推進する？
┌─────────────────────────────────────────────────────────┐
│                    共通層                                │
│    VS0: 戦略  VS6: データ基盤  VS7: オペレーション       │
│    ↑                                                    │
│    ここの活動は「VL1への貢献」をどう測定？               │
└─────────────────────────────────────────────────────────┘
          │               │               │
    ┌─────┴─────┐   ┌─────┴─────┐   ┌─────┴─────┐
    │  事業部A   │   │  事業部B   │   │  事業部C   │
    │ VS1-VS5   │   │ VS1-VS5   │   │ VS1-VS5   │
    │    ↑       │   │    ↑       │   │    ↑       │
    │ 独自KPI    │   │ 独自KPI    │   │ 独自KPI    │
    └───────────┘   └───────────┘   └───────────┘
```

**課題**:
- 事業層は「売上」「シェア」等の明確なKPIを持つ
- 共通層は「VL1への貢献」が曖昧になりやすい
- 事業層が共通価値を「お題目」として形骸化させるリスク

## 2. 価値貢献度ガバナンスの設計原則

### 原則1: 価値は「状態遷移」で測定する

**バリューストリームの本質は顧客状態遷移**です。価値貢献度は「どれだけ顧客を次の状態に遷移させたか」で測定します。

```
【価値貢献の測定原則】

❌ 誤った測定: 活動量を測る
   「品質管理活動を100件実施した」← 価値との因果が不明

✅ 正しい測定: 状態遷移を測る
   「VS5→VS6の遷移率が85%から92%に改善」← 価値実現を直接測定
```

### 原則2: 共通層は「イネーブラー」として測定する

共通層は直接顧客に価値を届けるのではなく、**事業層が価値を届ける能力を高める**役割です。

```
【共通層の貢献モデル】

共通層の貢献 = 事業層の「価値創出能力」向上度

例: データ基盤（VS6）の貢献測定
  - 事業層Aの顧客理解精度: 60% → 80%
  - 事業層Bの製品開発サイクル: 12ヶ月 → 8ヶ月
  - 事業層Cの在庫最適化率: 75% → 90%
```

### 原則3: バックキャスティングの整合性を検証する

**全ての活動がVL1にトレース可能**であることを定期的に検証します。

```
【トレーサビリティ検証】

Operation（活動）
    ↑ 「この活動はどのCapabilityを発現させるか？」
Capability（能力）
    ↑ 「この能力はどのVSを実現するか？」
Value Stage（価値ステージ）
    ↑ 「このVSはどのVL3に貢献するか？」
VL3 → VL2 → VL1

検証不能な活動 = 価値に貢献していない可能性
```

### 3.2 共通層の具体的貢献指標

#### VS0: 戦略（共通価値の浸透・整合）

| 指標カテゴリ | 指標 | 測定方法 | 目標例 |
|------------|------|---------|--------|
| **浸透度** | 価値理解度 | 従業員サーベイ | 全従業員の80%が「おいしさと健康」を説明可能 |
| **整合度** | 事業計画整合率 | 事業計画レビュー | 全事業計画がVL1に明示的にリンク |
| **実行度** | 価値起点意思決定率 | 意思決定ログ | 重要意思決定の70%が価値基準で判断 |

#### VS6: データ基盤（分析・インサイト提供）

| 指標カテゴリ | 指標 | 測定方法 | 目標例 |
|------------|------|---------|--------|
| **活用度** | データ活用率 | 事業部アクセスログ | 全事業部が週次でデータ活用 |
| **精度** | 予測精度向上 | A/Bテスト | 需要予測精度 +15% |
| **効率** | 意思決定時間短縮 | プロセス測定 | 製品開発の市場分析工数 -30% |

#### VS7: オペレーション・品質（品質保証・サステナビリティ）

| 指標カテゴリ | 指標 | 測定方法 | 目標例 |
|------------|------|---------|--------|
| **品質** | 品質基準達成率 | 検査結果 | 全製品99.9%以上 |
| **安全** | 重大インシデント | インシデント報告 | ゼロ維持 |
| **持続性** | サステナビリティスコア | 外部評価 | 業界上位10% |

## 5. 価値貢献度の推進メカニズム

### 5.1 インセンティブ設計

**共通層と事業層のインセンティブを「価値」で連結**します。

```
【インセンティブ連結モデル】

┌─────────────────────────────────────────────────────────────┐
│  共通層の評価                                                │
│                                                             │
│  評価 = 自部門KPI(40%) + 事業層支援成果(40%) + VL1貢献(20%) │
│                                                             │
│  例: データ基盤チームの評価                                  │
│  ・自部門KPI: データ品質、システム稼働率                     │
│  ・事業層支援: 「事業部からの満足度」「活用による成果」      │
│  ・VL1貢献: 全社NPS、リピート率への寄与                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  事業層の評価                                                │
│                                                             │
│  評価 = 事業KPI(50%) + VL1貢献(30%) + 共通層活用(20%)       │
│                                                             │
│  例: サプリメント事業部の評価                                │
│  ・事業KPI: 売上、利益、シェア                               │
│  ・VL1貢献: 「健康価値」認知向上への寄与                     │
│  ・共通層活用: データ活用度、品質基準遵守                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 6. アサヒグループ食品への適用案

### 6.1 VL1-VL3の定義（案）

```yaml
VL1: おいしさと健康
  定義: 「食を通じて、人々においしさと健康を届ける」

VL2-1: おいしさ価値
  VL3-1-1: 素材のおいしさ（フリーズドライ技術）
  VL3-1-2: 手軽なおいしさ（ミンティア、即席食品）
  VL3-1-3: 安心のおいしさ（和光堂の品質）

VL2-2: 健康価値
  VL3-2-1: 栄養補給価値（ディアナチュラ）
  VL3-2-2: 消化・整腸価値（エビオス）
  VL3-2-3: 乳幼児栄養価値（和光堂ベビー）

VL2-3: 信頼価値
  VL3-3-1: 品質への信頼
  VL3-3-2: 安全への信頼
  VL3-3-3: 持続可能性への信頼
```

### 6.2 共通層の価値貢献定義

| 共通層 | VL1への貢献経路 | 測定指標 |
|--------|---------------|---------|
| **VS0: 戦略** | 全事業の「おいしさと健康」整合性確保 | 事業計画整合率、従業員価値理解度 |
| **VS6: データ基盤** | 顧客理解→製品開発→おいしさ向上 | 製品開発精度、顧客満足度向上 |
| **VS7: 品質** | 安全・品質→信頼価値向上 | 品質達成率、クレーム率、認証取得 |

### 6.3 事業層のVL1貢献責任

| 事業グループ | 主要貢献VL2 | 貢献指標 |
|------------|-----------|---------|
| **嗜好品G** | おいしさ価値 | おいしさNPS、リピート率 |
| **健康・ライフG** | 健康価値、信頼価値 | 健康効果実感率、安心感スコア |
| **BtoB原料** | おいしさ価値（間接） | 顧客製品の品質向上寄与 |

## 8. まとめ

### フュージョン型の価値貢献度コントロール3原則

1. **測定原則**: 価値は「顧客状態遷移」で測定する
2. **連結原則**: 共通層は「イネーブラー」として事業層の価値創出能力向上で評価
3. **整合原則**: 全活動がVL1にトレース可能であることを検証

### 推進メカニズム3本柱

1. **可視化**: ダッシュボードによる価値貢献度の常時可視化
2. **インセンティブ**: 共通層・事業層のインセンティブを「価値」で連結
3. **意思決定**: 価値ROIによる投資・施策の優先順位決定
