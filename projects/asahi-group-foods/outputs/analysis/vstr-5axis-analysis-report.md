### 1.2 価値駆動評価軸（5軸15点満点）

| 評価軸 | 評価の焦点 | 配点 |
|--------|-----------|------|
| **VL1統一可能性** | 全事業で共通する最上位価値を定義できるか | 0-3点 |
| **顧客価値ジャーニー共通性** | VS0→VS7が事業間で共通か | 0-3点 |
| **価値創出Cap共通性** | VL1実現に必要な能力の共通度 | 0-3点 |
| **価値トレーサビリティ** | Operation→VS→VL1の追跡複雑度 | 0-3点 |
| **価値貢献評価統一** | 全事業を同一基準で評価できるか | 0-3点 |

│  → 3パターンの「いいとこ取り」が可能な唯一の構造                           │
│                                                                             │
│  【論点2】事業部軸ではなくフュージョン型を選ぶ根拠は？                      │
│  ─────────────────────────────────────────────────────────                  │
│  討議結果:                                                                  │
│  ・事業部軸: 短期的には導入容易だが、技術シナジーを逃す                    │
│  ・発酵技術・FD技術はアサヒグループ食品の競争優位の源泉                    │
│  ・これらをサイロ化すると、長期的な価値創出力が低下                        │
│  → 技術資産の活用を考慮すると、フュージョン型が必須                        │
│                                                                             │
│  【論点3】ケイパビリティ統合型を全面採用しない理由は？                      │
│  ─────────────────────────────────────────────────────────                  │
│  討議結果:                                                                  │
│  ・顧客セグメントが異なりすぎる（6種以上）                                 │
│  ・営業・マーケティングの統合は顧客価値を毀損するリスク                    │
│  ・R&D・品質・調達のみ部分採用が最適解                                     │
│  → 「技術は統合、顧客接点は分離」の原則を採用                              │
│                                                                             │
│  【結論】                                                                   │
│  フュージョン型をベースに、ケイパビリティ統合型から技術系機能のみ          │
│  部分採用する「いいとこ取り」構造を推奨                                    │
│                                                                             │
