# CL1 ドメイン分類

**作成日**: 2024年12月12日
**フェーズ**: Phase 3 - ケーパビリティ分解（CL1）
**対象**: 人材派遣事業のドメイン構造
**設計パターン**: Two-sided Value Stream（両面バリューストリーム）

## 1. ドメイン分類概要

スタッフサービスの人材派遣事業を戦略的重要性に基づいてドメイン分類を実施。

### 1.1 両面バリューストリーム（Two-sided VS）パターン採用

人材派遣事業は典型的な **マルチサイドプラットフォーム** であり、
顧客価値軸（企業側）と人材価値軸（スタッフ側）を同一ステージで設計。

```
│  │  │ ├─ エンゲージメント維持（離脱防止）                │   │   │
│  │  │ └─ キャリアパス提案（長期関係構築）                │   │   │

│  │  ├─ スタッフが「働きたい」意思を持っている               │   │
│  │  ├─ 定期的なコミュニケーション維持                       │   │
│  │  └─ キャリア相談・スキルアップ支援の継続                 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  【Cross-side Value Creation】                                  │
│  ├─ 企業の求人量 ↔ スタッフの登録意欲（Network Effect）       │
│  ├─ マッチング成功 → 双方満足 → 継続利用                     │
│  ├─ 実績データ蓄積 → マッチング精度向上 → 価値向上           │
│  └─ スタッフLTV最大化 → 稼働率向上 → 企業への即戦力供給     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 分類基準

```
┌─────────────────────────────────────────────────────────────────┐
│                    ドメイン分類基準                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【Core Domain】競争優位の源泉                                  │
│  ├─ 差別化に直結する                                           │
│  ├─ 自社でコントロールすべき                                   │
│  └─ 戦略的投資の対象                                           │
│                                                                 │
│  【Supporting Domain】Core を支える                             │
│  ├─ Core Domain の実現に必要                                   │
│  ├─ 自社実装 or パッケージ選択可能                             │
│  └─ 効率化・標準化が重要                                       │
│                                                                 │
│  【Generic Domain】業界共通                                     │
│  ├─ 業界標準のプロセス                                         │
│  ├─ パッケージ・SaaS活用推奨                                   │
│  └─ 差別化要素にならない                                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
