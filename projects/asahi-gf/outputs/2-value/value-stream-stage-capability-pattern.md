# バリューストリーム・ステージ・ケーパビリティ適用パターン

## 1. 構造的課題の解決

### 問題：価値軸と事業軸の直交

```
価値軸（横）: 認知 → 検討 → 購入 → 使用 → 推奨
              ↑
              │ 直交（異なる次元）
              ↓
事業軸（縦）: 和光堂 / ディアナチュラ / アマノ / ミンティア / バランス献立
```

### 解決：顧客セグメントを接点層として設計

**ステージとケーパビリティは「階層」ではなく「マッピング」の関係**

```
ケーパビリティは複数のステージに「寄与」する

例：「商品開発」ケーパビリティ
  → 認知ステージ：差別化訴求の源泉
  → 使用ステージ：満足度の根幹
  → 推奨ステージ：口コミのネタ
```

## 2. 適用パターン：3層ケーパビリティモデル

### 全体構造

```
┌─────────────────────────────────────────────────────────────────────┐
│                  市場接点層（Layer 3）                                │
│              顧客価値ステージ（共通フレーム）                          │
│     認知 → 検討 → 購入 → 初回体験 → 継続 → 推奨                      │
└─────────────────────────────────────────────────────────────────────┘
                              ↓ マッピング
┌─────────────────────────────────────────────────────────────────────┐
│                  顧客セグメント層（接点層）                           │
│   ┌─────────┬─────────┬─────────┬─────────┬─────────┐              │
│   │ 育児世帯 │ 健康意識 │ シニア  │ 時短志向 │リフレッシュ│              │
│   │（和光堂）│（ディア）│（バラ献）│（アマノ）│（ミンティア）│              │
│   └─────────┴─────────┴─────────┴─────────┴─────────┘              │
└─────────────────────────────────────────────────────────────────────┘
                              ↓ 実現
┌─────────────────────────────────────────────────────────────────────┐
│                  ケーパビリティ層（Layer 1-2）                        │
│   L1: 共通ケーパビリティ（事業横断・DX優先）                          │
│   L2: ドメインケーパビリティ（セグメント固有）                        │
│   L3: 製品ケーパビリティ（ブランド固有）                              │
└─────────────────────────────────────────────────────────────────────┘
```

### 3層ケーパビリティの定義

| 層 | 定義 | 粒度 | DX優先度 | 例 |
|---|---|---|---|---|
| **L1: 共通** | 事業横断で再利用可能 | 大 | 最高 | 顧客データ基盤、EC基盤、品質管理 |
| **L2: ドメイン** | 顧客セグメント固有 | 中 | 高 | 育児支援、シニア栄養管理 |
| **L3: 製品** | ブランド固有 | 小 | 中 | 和光堂レシピ提案、ミンティアフレーバー開発 |

## 6. 結論：適用パターン

### 価値と事業の融合方法

1. **融合しない** - 直交する軸として認める
2. **顧客セグメントを接点層**として明示的に設計
3. **ケーパビリティはステージにマッピング**する形で整理
4. **DX進化単位はL1を優先**（最大レバレッジ）

### 推奨アプローチ

```
Step 1: 顧客セグメント定義（市場接点層）
    ↓
Step 2: セグメント別価値ステージ設計（スイムレーン）
    ↓
Step 3: ステージ×ケーパビリティ マッピング
    ↓
Step 4: L1/L2/L3の分類とDX優先順位付け
    ↓
Step 5: DX進化単位としての実装
```

この構造により、**価値軸の顧客視点**と**事業軸のケーパビリティ視点**を、顧客セグメントを介して接続し、DXで進化しやすい単位を導出できます。
