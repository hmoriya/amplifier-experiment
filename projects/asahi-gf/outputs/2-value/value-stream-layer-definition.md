# バリューストリーム3層モデル定義

## 概要

アサヒグループ食品のバリューストリーム設計における3層モデルの概念定義を行う。

## 3層モデルの定義

### 層構造サマリー

| 層 | 名称 | 役割 | バリューストリームとの関係 |
|----|------|------|---------------------------|
| Layer 3 | 市場接点層 | 顧客価値の定義・認知 | バリューストリームの**起点と終点** |
| Layer 2 | 事業推進層 | 価値の流れを運営 | **バリューストリームそのもの**（実行単位） |
| Layer 1 | ケーパビリティ層 | 横断的能力を提供 | バリューストリームを**支える共有基盤** |

## Layer 3: 市場接点層（価値軸）

### 定義
顧客が認識する価値カテゴリの集合。バリューストリームの起点（顧客ニーズ）と終点（価値提供）を定義する。

### 機能
- 顧客視点での価値カテゴリ設定
- 市場セグメントとの接点管理
- 顧客インサイトの収集・分析

### アサヒグループ食品における価値軸

| 価値軸 | 定義 | 対象顧客 |
|--------|------|----------|
| 健康寿命延伸 | シニアの健康・栄養サポート | 高齢者・介護関係者 |
| 子どもの健やかな成長 | 乳幼児の栄養・発達支援 | 乳幼児を持つ親 |
| 手軽なおいしさ | 簡便調理での本格的な味 | 忙しい生活者 |
| 美容・ダイエット | 健康的な体型維持 | 美容・健康意識の高い成人 |
| リフレッシュ・活力 | 気分転換・エネルギー補給 | ビジネスパーソン |

## Layer 2: 事業推進層（バリューストリーム）

### 定義
バリューストリームの実行単位（事業ドメイン）。価値軸からの要求をEnd-to-Endで実現し、顧客に届ける。

### 機能
- バリューストリーム全体のオーナーシップ
- P/L責任
- End-to-Endの価値フロー管理

### 重要な概念整理

| 解釈 | 判定 | 理由 |
|------|------|------|
| ステージ | ✕ | ステージはバリューストリーム内の一工程。事業推進層はEnd-to-Endの流れを持つ |
| 単一のバリューストリーム | ✕ | 複数の事業を含む |
| **バリューストリーム群** | ◎ | 「バリューストリーム・ドメイン」として捉える |

### アサヒグループ食品におけるバリューストリーム


## Layer 1: ケーパビリティ層（共有基盤）

### 定義
バリューストリーム間で共有される横断的能力。各バリューストリームに技術・ノウハウを提供する。

### 機能
- 技術・専門知識の集約
- バリューストリーム間での能力共有
- 規模の経済の実現

### アサヒグループ食品における共有ケーパビリティ

| ケーパビリティ | 内容 | 活用するバリューストリーム |
|---------------|------|---------------------------|
| フリーズドライ技術 | 真空凍結乾燥技術 | アマノフーズ、バランス献立 |
| 酵母・発酵技術 | ビール酵母由来の健康機能 | エビオス、新規健康食品 |
| 栄養科学 | 100年超の育児・栄養研究 | 和光堂、ディアナチュラ、バランス献立 |
| 製剤技術 | タブレット・バー成形技術 | ミンティア、1本満足バー |
| 品質管理基盤 | HACCP、ISO、GMP | 全バリューストリーム |
| デジタル・データ基盤 | EC、CRM、分析 | 全バリューストリーム |

## 3層間の相互作用

```
顧客ニーズ ─────────────────────────────────────> 顧客価値提供
     │                                                  ▲
     ▼                                                  │
┌─────────────────────── Layer 3: 市場接点層 ─────────────────────────┐
│   価値軸A              価値軸B              価値軸C                 │
│  「健康寿命延伸」      「子どもの成長」     「手軽なおいしさ」        │
└────────────────┬───────────────────────────────────┬────────────────┘
                 │ 価値要求                         │ 価値提供
                 ▼                                  ▲
┌─────────────────────── Layer 2: 事業推進層 ─────────────────────────┐
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ バリューストリーム: 各事業のEnd-to-End価値フロー            │   │
│  │   [企画]→[開発]→[調達]→[製造]→[品質]→[物流]→[販売]→[CS]  │   │
│  └─────────────────────────────────────────────────────────────┘   │
└────────────────┬───────────────────────────────────┬────────────────┘
                 │ 能力要求                         │ 能力提供
                 ▼                                  ▲
┌─────────────────────── Layer 1: ケーパビリティ層 ───────────────────┐
│   [FD技術]    [酵母技術]    [栄養科学]    [品質管理]    [IT基盤]    │
└─────────────────────────────────────────────────────────────────────┘
```

## 設計原則

### 1. 層間インターフェースの標準化
- Layer 3 → Layer 2: 価値要求の標準フォーマット
- Layer 2 → Layer 1: 能力要求の標準フォーマット

### 2. 各層の独立進化
- 各層は独立して進化可能
- 層間インターフェースを維持すれば内部変更は自由

### 3. バリューストリームのオーナーシップ
- Layer 2の各事業がバリューストリームのP/L責任を持つ
- End-to-Endの顧客価値提供に責任を持つ

### 4. ケーパビリティの共有最大化
- Layer 1の能力は全バリューストリームで共有
- 技術シナジーを構造的に保証

## まとめ

| 問い | 回答 |
|------|------|
| 価値軸は市場接点を狙うものか？ | **Yes** - Layer 3として顧客との接点で機能 |
| 事業推進層はバリューストリームか？ | **Yes** - バリューストリームそのもの（の集合） |
| 事業推進層はステージか？ | **No** - ステージはバリューストリーム内の一工程 |

この定義により、3層モデルにおける各層の役割とバリューストリームとの関係が明確になる。
