## 2. ステークホルダーマトリックス

### 影響力 × 関心度マトリックス

```yaml
高影響力・高関心:
  - 経営層（CEO、CFO、CIO）
  - 株主・投資家
  - 主要顧客セグメント
  - 規制当局
  対応: 密接な関与、定期的コミュニケーション

高影響力・低関心:
  - 一部機能部門
  - 業界団体
  対応: 満足度維持、必要時の情報提供

低影響力・高関心:
  - 現場スタッフ
  - 一部NGO
  - 地域コミュニティ
  対応: 情報提供、懸念への対応

低影響力・低関心:
  - 一般メディア（非専門）
  対応: 最小限の監視
```

### Power-Interest Grid

```
         高関心
           ↑
           |
    管理   |   重点管理
  (Manage) | (Manage Closely)
  ────────┼────────→ 高影響力
           |
   監視    |   情報提供
 (Monitor) | (Keep Informed)
           |
```

**重点管理（最優先）**:
- 経営層
- 株主・投資家
- 主要事業部門CEO
- 規制当局

**情報提供（定期報告）**:
- IT・デジタル部門
- 主要機能部門
- 主要取引先

**管理（適宜対応）**:
- 現場スタッフ
- 地域コミュニティ
- NGO

**監視（必要時対応）**:
- 一般メディア

## 4. 主要リスクと機会

### リスク
```yaml
経営層:
  - 投資判断の遅延
  - 戦略的優先順位の変更
  対策: 明確なビジネスケース、継続的価値実証

事業部門:
  - 変革抵抗
  - リソース不足
  対策: 早期巻き込み、クイックウィン創出

現場スタッフ:
  - ユーザー受容性低下
  - 生産性一時的低下
  対策: 丁寧なトレーニング、サポート体制

株主:
  - 短期的利益圧力
  - 投資回収期待
  対策: 段階的価値実現、透明な進捗報告

規制当局:
  - 規制強化
  - コンプライアンス違反リスク
  対策: 先行的対応、継続的対話
```

### 機会
```yaml
経営層:
```

## 5. コミュニケーション計画

### 頻度と形式
```yaml
日次:
  - 開発チーム（スタンドアップ）

週次:
  - プロジェクトチーム（進捗会議）
  - 技術チーム（レビュー）

月次:
  - 経営報告
  - 事業部門レビュー

四半期:
  - ステアリングコミッティ
  - 株主・投資家向け報告

年次:
  - 株主総会
  - 戦略レビュー
```

### エスカレーションパス
```yaml
レベル1: プロジェクトチーム内
  - 日常的な問題解決

レベル2: プロジェクトマネージャー→事業部門責任者
  - 事業影響のある問題

レベル3: プロジェクトスポンサー→経営層
  - 戦略的判断が必要な問題

レベル4: 経営会議
  - 企業全体に影響する重大事項
```
