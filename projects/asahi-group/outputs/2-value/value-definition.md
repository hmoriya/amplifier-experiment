# 価値定義 - アサヒグループ

## 1. 最上位価値（VL1）

### 定義

**VL1: 期待を超えるおいしさと楽しい生活文化の創造**

### VL1選択の設計ストーリー

```
問い: アサヒグループの究極的価値は何か？

分析プロセス:
┌─────────────────────────────────────────────────────────────┐
│ 入力: ミッション「期待を超えるおいしさ、楽しい生活文化の創造」│
│                                                              │
│ Insight #1（発酵プラットフォーム思考）:                      │
│   発酵技術 = プラットフォーム（単なる製造手段ではない）      │
│   → ビールは「出力の一つ」に過ぎない                        │
│   → 発酵エンジンが生み出す価値は多様（食品、健康、素材）    │
│                                                              │
│ Insight #2（予期せぬ価値）:                                  │
│   「期待を超える」= 予測を裏切る驚き                        │
│   → 継続的イノベーションが本質                              │
│   → 消費者が想像しなかった価値を提供し続ける                │
└─────────────────────────────────────────────────────────────┘

結論:
VL1の本質 = 「発酵技術による予期せぬ価値創造」
表現 = 「期待を超えるおいしさと楽しい生活文化の創造」

選択理由:
1. 1889年からの発酵技術蓄積がコア・コンピタンス
2. 「期待を超える」は継続的イノベーションを内包
3. 「楽しい生活文化」は酒類だけでなく飲料・食品・健康に拡張可能
```

## 2. ミッションステートメント（MS）バックキャスティング

### MS設計の論理

```
VL1「発酵による予期せぬ価値創造」を実現するには？

バックキャスティング:

    VL1（究極の姿）
         ↑
    MS5（18ヶ月後）: イノベーションリーダー確立
         ↑
    MS4（12ヶ月後）: 製品パイプライン充実
         ↑
    MS3（9ヶ月後）: 研究知見・技術の蓄積
         ↑
    MS2（6ヶ月後）: イノベーションプロセス稼働
         ↑
    MS1（3ヶ月後）: DX・データ基盤整備
         ↑
    現在
```

### ミッションステートメント一覧

| MS | 時期 | ステートメント | VL1への貢献 |
|----|------|---------------|-------------|
| MS1 | 3ヶ月後 | 「データ駆動の意思決定基盤を確立する」 | イノベーションの基盤 |
| MS2 | 6ヶ月後 | 「発酵技術を活用した製品開発プロセスを高速化する」 | 価値創造の効率化 |
| MS3 | 9ヶ月後 | 「酵母・発酵研究の知見をナレッジベース化する」 | 予期せぬ発見の促進 |
| MS4 | 12ヶ月後 | 「多様な製品パイプラインを構築する」 | 価値の多様化 |
| MS5 | 18ヶ月後 | 「発酵イノベーションのリーディングカンパニーとなる」 | VL1の実現 |

### MS設計の設計ストーリー

```
なぜこの5つのMSか？

理由1: 時間軸の整合性
  - 四半期レビューと整合（3ヶ月刻み）
  - 研究開発の自然なリズムに合致
  - 経営計画のマイルストーンと連動可能

理由2: 依存関係の明確化
  - MS1なくしてMS2なし（基盤→プロセス）
  - MS3はMS2の結果を蓄積（プロセス→知見）
  - 積み上げ型の成長モデル

理由3: VL1との整合性
  - 「予期せぬ価値」を生み出すには多産多死のパイプライン必要
  - MS4（パイプライン充実）がキーマイルストーン
  - MS5でイノベーションリーダーシップを確立
```

## 3. Value Stream（VS）設計

### VS一覧

| VS | 名称 | 説明 | ドメインタイプ | 対応MS |
|----|------|------|---------------|--------|
| VS0 | 企業基盤 | ガバナンス、財務、HR、IT基盤 | Generic | MS1 |
| VS1 | 研究開発 | 発酵技術研究、酵母研究、基礎研究 | Core | MS3 |
| VS2 | 製品開発 | 新製品企画、処方設計、製品化 | Core | MS2, MS4 |
| VS3 | ブランド・マーケティング | ブランド戦略、販促、顧客体験 | Core | MS5 |
| VS4 | サプライチェーン | 調達、製造、物流、品質管理 | Supporting | MS2 |
| VS5 | 販売・顧客関係 | 営業、チャネル管理、顧客サービス | Supporting | MS4 |
| VS6 | 品質・コンプライアンス | 品質保証、規制対応、安全管理 | Supporting | MS1 |
| VS7 | 事業開発 | M&A、新規事業、アライアンス | Supporting | MS5 |

### VS設計の設計ストーリー

```
なぜこのVS構成か？

【Coreドメイン（VS1, VS2, VS3）】

VS1（研究開発）がCoreである理由:
  - 発酵技術はアサヒの競争優位の源泉
  - 50年超の酵母研究が参入障壁
  - VL1「予期せぬ価値」の源泉

VS2（製品開発）がCoreである理由:
  - 「発酵プラットフォーム」の実行部隊
  - 研究成果を製品価値に変換
  - VL1「おいしさ」を具現化

VS3（ブランド・マーケティング）がCoreである理由:
  - 「期待を超える」体験の演出
  - スーパードライ等の強力ブランド資産
  - VL1「楽しい生活文化」を顧客に届ける

【Supportingドメイン（VS4, VS5, VS6, VS7）】

VS4（サプライチェーン）:
  - 「おいしさ」を物理的に届けるインフラ
  - 品質維持に必須だが差別化要因ではない

VS5（販売・顧客関係）:
  - 製品を顧客に届けるチャネル
  - 業界標準の機能

VS6（品質・コンプライアンス）:
  - 法規制対応は必須
  - 酒類業界共通の要件

VS7（事業開発）:
  - 成長機会の探索
  - Core強化の手段

【Genericドメイン（VS0）】

VS0（企業基盤）:
  - 全社共通機能
  - 差別化要因ではないが必須
```

## 4. 価値階層（VL）

### VL分解

```
VL1: 期待を超えるおいしさと楽しい生活文化の創造
├── VL2a: 発酵技術による革新的製品の創出
│   ├── VL3a-1: 酵母研究に基づく新素材開発
│   ├── VL3a-2: 発酵プロセスの最適化
│   └── VL3a-3: 製品イノベーションの加速
│
├── VL2b: グローバルプレミアムブランドの価値最大化
│   ├── VL3b-1: スーパードライのグローバル展開
│   ├── VL3b-2: 欧州ブランド群のシナジー
│   └── VL3b-3: プレミアム体験の提供
│
└── VL2c: 持続可能な事業基盤の構築
    ├── VL3c-1: サプライチェーン効率化
    ├── VL3c-2: 環境負荷削減
    └── VL3c-3: デジタル基盤整備
```

### VL分解の設計ストーリー

```
なぜこの3つのVL2か？

VL2a（発酵技術による革新）:
  - アサヒのコア・コンピタンス直結
  - 「予期せぬ価値」の源泉
  - VS1, VS2に対応

VL2b（グローバルブランド）:
  - 「期待を超える」体験の具現化
  - 5つのグローバルプレミアムブランド活用
  - VS3に対応

VL2c（持続可能な基盤）:
  - VL2a, VL2bを支える土台
  - 長期的な価値創造の前提
  - VS0, VS4, VS5, VS6, VS7に対応
```

## 5. 設計判断サマリー

| 設計要素 | 選択 | 理由（設計ストーリー） |
|----------|------|------------------------|
| VL1 | 期待を超えるおいしさと楽しい生活文化の創造 | 発酵プラットフォーム思考。ビールは出力の一つ |
| Core VS | VS1, VS2, VS3 | VL1への直接貢献度が最高 |
| MS時間軸 | 3ヶ月単位×5段階 | 四半期レビューと整合。研究開発の自然なリズム |
| VS粒度 | 8つ（VS0-VS7） | チーム境界と投資判断単位の両立 |
| VL階層 | 3層（VL1→VL2→VL3） | 戦略→戦術→運用の整合性 |
