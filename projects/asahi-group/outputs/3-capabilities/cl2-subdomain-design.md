# CL2 サブドメイン設計 - アサヒグループ

## 概要

Coreドメイン（VS1, VS2, VS3）のCL1ケーパビリティをCL2サブドメインに分解し、
ユビキタス言語と境界コンテキスト候補を定義する。

---

## VS1 研究開発 - CL2サブドメイン

### VS1-C1: 発酵プラットフォーム研究

#### サブドメイン分解

| ID | サブドメイン | 責務 | ドメインタイプ |
|----|-------------|------|---------------|
| VS1-C1-S1 | 酵母ライブラリ管理 | 酵母株の保存・分類・特性データベース管理 | Core |
| VS1-C1-S2 | 酵母育種・改良 | 目的特性を持つ酵母の選抜・育種 | Core |
| VS1-C1-S3 | 発酵プロセス最適化 | 発酵条件の最適化・スケールアップ | Core |
| VS1-C1-S4 | 微生物学基盤研究 | 酵母の代謝経路・遺伝子研究 | Core |

#### 入出力定義

```
【酵母ライブラリ管理】
  入力: 新規酵母株、特性評価データ
  出力: 酵母カタログ、推奨酵母リスト
  連携: → 酵母育種・改良、→ 製品開発（VS2）

【酵母育種・改良】
  入力: 目標特性要件、親株候補
  出力: 改良酵母株、特性レポート
  連携: ← 酵母ライブラリ、→ 発酵プロセス最適化

【発酵プロセス最適化】
  入力: 酵母株、製品仕様
  出力: 最適発酵条件、スケールアップ手順
  連携: ← 酵母育種、→ 製造（VS4）
```

#### ユビキタス言語

| 用語 | 定義 |
|------|------|
| 酵母株 (Yeast Strain) | 特定の特性を持つ酵母の系統 |
| 発酵プロファイル | 酵母が生成する香味成分の特性パターン |
| 親株 | 育種の基となる酵母株 |
| スケールアップ | 実験室規模から工場規模への移行 |


## 境界コンテキスト候補

### 推奨される境界コンテキスト

| コンテキスト名 | 含まれるサブドメイン | 理由 |
|---------------|---------------------|------|
| **Fermentation Platform** | VS1-C1全体 | 発酵技術の核心。独自言語が多い |
| **Product Recipe** | VS2-C1, VS2-C5 | 処方・品質設計は密結合 |
| **Brand Portfolio** | VS3-C1, VS3-C2 | グローバル・ローカルブランド一体管理 |
| **Innovation Pipeline** | VS1-C4-S3, VS2-C2 | イノベーション創出の横断領域 |
| **Functional Health** | VS1-C2, VS2-C3 | 機能性・健康領域の専門性 |

### 統合・分割の判断基準

```
統合基準:
- 同一チームが運営
- 共通のユビキタス言語
- 密結合な入出力関係
- 変更頻度が同期

分割基準:
- 異なる専門性が必要
- 独立したリリースサイクル
- 異なるステークホルダー
- 言語の衝突がある
```
