# /ddd:1-plan - 価値デザインとビジネスデザイン

## 概要

このフェーズでは、パラソルV4のPhase 1（価値デザイン）とPhase 2（ビジネスデザイン）を統合して実行します。

## 実行コマンド

```bash
/ddd:1-plan
```

## 実行コンテキスト

### 前提条件
- `/ddd:prime`が完了していること
- 価値宣言が定義されていること
- ドメインコンテキストが確立されていること

### 入力
- 価値コンテキスト（0-prime/outputs/）
- ビジネスゴール
- 市場要求

### 出力
- 価値分解構造（VL0→VL1→VL2→VL3）
- ケーパビリティマトリクス（CL1→CL2→CL3）
- ビジネスオペレーション定義
- バリューマイルストーン

---

## Part 1: 価値デザイン [Phase 1]

### Task 1.1: 価値分解

```yaml
価値分解構造:
  VL0: "持続可能な価値創造"
    VL1-A: "顧客価値の最大化"
      VL2-A1: "体験品質の向上"
        VL3-A1a: "レスポンスタイム改善"
        VL3-A1b: "UI/UXの最適化"
      VL2-A2: "サービス可用性"
        VL3-A2a: "99.9%稼働率"
        VL3-A2b: "障害復旧時間短縮"

    VL1-B: "環境価値の実現"
      VL2-B1: "リソース効率化"
        VL3-B1a: "エネルギー消費削減"
        VL3-B1b: "コンピューティング最適化"
```

### Task 1.2: 価値グループ形成

```yaml
価値グループ:
  グループA（個人的価値）:
    - VL2-A1: 体験品質の向上
    - VL2-A2: サービス可用性

  グループB（社会的価値）:
    - VL2-B1: リソース効率化

  グループC（持続的価値）:
    - 長期的な価値創造メカニズム
```

### Task 1.3: バリューマイルストーン設計

```yaml
マイルストーン:
  M1: "基本価値の実現"
    期限: 3ヶ月
    価値グループ: A
    成功指標:
      - 基本機能の提供
      - 初期ユーザー獲得

  M2: "価値の拡大"
    期限: 6ヶ月
    価値グループ: A, B
    成功指標:
      - 機能拡張
      - 効率化実現

  M3: "持続的価値創造"
    期限: 12ヶ月
    価値グループ: A, B, C
    成功指標:
      - エコシステム確立
      - 継続的改善
```

## Part 2: ビジネスデザイン [Phase 2]

### Task 2.1: バリューストリーム設計

```yaml
バリューストリーム:
  ステージ:
    S1: "発見"
      活動: ニーズの特定

    S2: "評価"
      活動: ソリューション検討

    S3: "取得"
      活動: サービス契約

    S4: "実装"
      活動: システム構築

    S5: "利用"
      活動: 価値享受

    S6: "最適化"
      活動: 継続的改善

    S7: "更新"
      活動: 次世代への移行
```

### Task 2.2: ケーパビリティ抽出（WHAT）

```yaml
ケーパビリティマトリクス:
  CL1（戦略）:
    CL1-001: "顧客体験革新"
      目的: 卓越した顧客体験の提供

    CL1-002: "オペレーショナルエクセレンス"
      目的: 効率的な運用

    CL1-003: "持続可能性"
      目的: 長期的価値創造

  CL2（戦術）:
    CL2-001: "パーソナライゼーション"
      親: CL1-001

    CL2-002: "リアルタイム処理"
      親: CL1-001

    CL2-003: "自動化"
      親: CL1-002

    CL2-004: "最適化"
      親: CL1-002

  CL3（運用）:
    CL3-001: "ユーザープロファイル管理"
      親: CL2-001

    CL3-002: "推薦エンジン"
      親: CL2-001

    CL3-003: "イベント処理"
      親: CL2-002
```

### Task 2.3: ビジネスオペレーション定義（HOW）

```yaml
ビジネスオペレーション:
  CL3-001_ユーザープロファイル管理:
    パターン: CRUD
    操作:
      - Create: プロファイル作成
      - Read: プロファイル参照
      - Update: プロファイル更新
      - Delete: プロファイル削除

  CL3-002_推薦エンジン:
    パターン: Analytics
    操作:
      - 分析: ユーザー行動分析
      - 予測: 嗜好予測
      - 生成: レコメンド生成

  CL3-003_イベント処理:
    パターン: Workflow
    操作:
      - 受信: イベント受信
      - 処理: ビジネスロジック実行
      - 通知: 結果通知
```

### Task 2.4: WHAT-HOW ZIGZAG構造の適用

```yaml
ZIGZAG構造:
  CL1 WHAT: "顧客体験革新"
    ↓
  HOW: パーソナライゼーション、リアルタイム処理
    ↓
  CL2 WHAT: "パーソナライゼーション"
    ↓
  HOW: プロファイル管理、推薦エンジン
    ↓
  CL3 WHAT: "ユーザープロファイル管理"
    ↓
  HOW: CRUD操作
    ↓
  L4 WHAT: 詳細ユースケース（次フェーズで定義）
```
